# 报告模板

## 问题定位规范

所有报告中的问题定位必须遵循以下规范：

- **必须**使用 `scripts/doc-link.sh` 生成 GitHub 可跳转链接（如 `https://github.com/ZEGOCLOUD/docs_all/blob/main/docs/ai-agent/web/quick-start.mdx?plain=1#L328`），方便飞书报告读者直接点击定位
- 用法：`bash .claude/skills/make-integration-test/scripts/doc-link.sh docs/ai-agent/web/quick-start.mdx 328`
- 也可补充**章节路径**（如"前提条件 > 集成 SDK"）作为辅助定位
- **必须**用代码块格式摘录文档原文，包含前后各 3 行上下文
- 如果引用的 MDX 内容来自 import 的 snippet 文件，要指出 snippet 文件的路径和行号
- 不能只说"前提条件章节"这种模糊定位

## 类别标记说明

报告中所有类别标记必须同时使用文字和图标，格式为 `图标 + 文字描述`：

| 标记 | 含义 |
|------|------|
| 📄 文档问题 | 文档本身有误、缺失、模糊、矛盾 |
| 🔧 代码实现问题 | 文档正确但编码偏离文档要求 |
| 🏗️ 环境问题 | 开发环境、依赖版本、工具链问题 |
| 🔗 链接问题 | 超链接失效、指向错误、锚点缺失 |
| 📖 概念缺失 | 前置知识未解释，行业术语未说明 |

## 汇总报告重写要点

在 Step 7 中，根据集成测试结果重写 doc-eval 生成的汇总报告。需要增补以下内容：

### 问题验证结果标注格式

对每个文档评测发现的问题，在汇总报告中追加验证结果列：

```markdown
| 问题 | 严重度（评测） | 严重度（验证后） | 集成测试验证 | 备注 |
|------|---------------|----------------|-------------|------|
| [问题] | 🔴 阻塞 | 🔴 阻塞 | ✅ 确认阻塞 | [说明] |
| [问题] | 🔴 阻塞 | 🟡 体验问题 | ⚠️ 部分阻塞 | [说明] |
| [问题] | 🟡 体验问题 | 🔴 阻塞 | 🆕 集成测试发现 | [说明] |
```

### 综合评分调整说明格式

如果集成测试结果影响综合评分，在汇总报告中添加：

```markdown
## 综合评分调整说明

原综合评分：X/5
建议调整后：Y/5

调整原因：
1. [集成测试发现的重大问题对评分的影响]
2. [文档评测高估/低估的问题说明]
```
