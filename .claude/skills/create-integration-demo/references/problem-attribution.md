# 问题归类详细定义

## 归类前置检查

在将问题归为 📄 文档问题 之前，必须确认以下三点：

1. **文档确实说了这个内容吗？** — 引用原文证明。如果文档没提到这个技术细节，说明文档没给错误信息，只是没写。
2. **文档是否有链接指向这个信息？** — 检查文档中的所有链接（@短链、相对路径、URL path、完整 URL），用 `api-short-link-to-mdx` 或 `url-to-mdx` 跟随链接后再判断。只有确认所有链接都无法提供所需信息后，才能归为"缺失"。
3. **这个信息是我从外部知识引入的吗？** — 如果这个技术细节不是文档中任何地方提到的，而是我从自己的知识库推断的，那这是 🔧 代码实现问题，不能归咎于文档。

## 归类决策树

遇到问题时，按以下顺序判断：

```
问题出现时：
  ├─ 是外部链接失效？ → 🔗 链接问题
  ├─ 是环境/依赖/工具链问题？ → 🏗️ 环境问题
  ├─ 是概念/术语未解释导致无法理解？ → 📖 概念缺失
  ├─ 严格按文档执行，文档是否有错/漏/模糊？ → 📄 文档问题
  └─ 按文档执行，文档看似正确但代码仍报错？ → 🔧 代码实现问题
```

## 类别定义

### 📄 文档问题

问题源于文档本身的缺失、错误或模糊。

子类型：
- **缺失**：文档缺少关键步骤、参数说明、错误处理
- **错误**：文档中的代码、命令、参数值有误
- **模糊**：文档描述不清楚，需要猜测

注意：如果用户明确说明某些参数、产物、链接等处于测试阶段就是跟mdx文档描述不一致的，不算问题。（比如测试期间SDK是通过离线压缩包形式集成而mdx文档说是用包管理）

示例：
- 文档说"调用此方法"但未说明参数填什么
- 代码示例中变量名拼写错误
- 文档说"配置环境"但未说明如何配置
- server 参数值为 'xxx' 占位符，未说明填写什么
- Token 获取只写了 Api.getToken() 未说明实现

### 🔧 代码实现问题

严格按文档实现，但代码仍有问题。

子类型：
- **API 变更**：文档中的 API 已废弃或参数变化
- **类型错误**：代码示例类型与实际 API 不匹配
- **逻辑错误**：按文档写的代码逻辑有问题
- **SDK 问题**：SDK 本身的 bug

示例：
- 按文档调用 API，但返回 "method not found"
- 文档说传 string，实际需要 number
- 按文档写的代码运行时崩溃

### 🏗️ 环境问题

环境配置、依赖版本、工具链导致的问题。

子类型：
- **依赖冲突**：文档指定的依赖版本与当前环境不兼容
- **工具链**：编译器、CLI 工具版本不匹配
- **系统要求**：操作系统、Node.js 版本等要求

示例：
- 文档要求 Node.js 18，当前使用 Node.js 20 导致不兼容
- 文档中的命令在 Windows 上不可用
- 依赖包已删除，无法安装

### 🔗 链接问题

文档中的外部链接失效。

示例：
- 文档中的控制台链接 404
- 引用的外部 API 文档链接失效
- 下载链接无法访问

### 📖 概念缺失

文档未解释关键概念，导致零经验开发者无法理解。

子类型：
- **行业术语**：如"推流"、"拉流"、"连麦"
- **ZEGO 概念**：如"房间"、"StreamID"、"用户 ID"
- **技术概念**：如"Token"、"鉴权"、"回调"

示例：
- 文档直接使用"Token"但未解释是什么、从哪获取
- 文档说"加入房间"但未说明"房间"概念
- 文档使用"鉴权"但未解释鉴权流程

## 严重度定义

| 级别 | 标记 | 含义 |
|------|------|------|
| 阻塞 | 🔴 | 导致无法继续，必须修复 |
| 体验差 | 🟡 | 可以完成但体验糟糕，建议修复 |
| 优化建议 | 🟢 | 锦上添花，可选择性改进 |
