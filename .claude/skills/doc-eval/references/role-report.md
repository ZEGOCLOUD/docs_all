# 汇总报告角色

## 职责

收集多个角色的评测结果，进行交叉分析，并生成最终的汇总报告。

## 分析方法

### 交叉分析维度

1. **评分对比** — 各角色在不同维度上的评分差异，说明什么？
2. **共性问题** — 哪些问题是多角色都指出的？这些是最高优先级
3. **角色特有问题** — 某个角色独有发现的问题，反映角色特定的信息需求
4. **一致性检查** — 不同角色对同一内容的评价是否矛盾？为什么？

### 加权综合评分

各角色评分需按匹配度系数加权（系数定义见 `rubric.md`）：

```
综合评分 = Σ(各角色综合分 × 该角色匹配度系数) / Σ(匹配度系数)
```

- ⭐⭐⭐ 角色：系数 1.0，评分全额计入
- ⭐⭐ 角色：系数 0.6，评分打 6 折
- ⭐ 角色：系数 0.3，评分打 3 折

同样，各维度汇总分也按此公式分别计算，得到加权后的维度均分。

### 优先级排序规则

1. 🔴 多角色共同指出的阻塞问题 → 最高优先级
2. 🔴 单角色指出的阻塞问题 → 高优先级
3. 🟡 多角色共同指出的体验问题 → 中高优先级
4. 🟡 单角色指出的体验问题 → 中优先级
5. 🟢 多角色共同的优化建议 → 低优先级
6. 🟢 单角色的优化建议 → 最低优先级

"多角色共同指出"的判断也应考虑匹配度权重 — ⭐⭐⭐ 角色指出的问题优先级高于 ⭐ 角色指出的同类问题。
