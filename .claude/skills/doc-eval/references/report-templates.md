# 文档评测汇总报告输出格式

本模板用于 `doc-eval` 的阶段摘要和文档评测汇总报告。

## 输出位置

`doc-eval` 阶段至少输出：

```text
doc-test-reports/{run-name}/doc-eval/doc-eval-summary.md
doc-test-reports/{run-name}/doc-eval/role-*.md
```

`run-name` 固定格式：

```text
{product}-{platform}-{scope}-{date}
```

## 飞书写入策略

在本地文件创建后，默认还要写入飞书。

如果用户明确要求“不写飞书”“只输出本地 Markdown”“不要上传”，则只生成本地 Markdown，不写飞书。


## doc-eval-summary.md 模板

```markdown

## 基本信息

| 字段 | 值 |
|------|----|
| 产品 | {product} |
| 平台 | {platform} |
| 范围 | {scope} |
| 文档类型 | {doc_type} |
| 目标文档 | {doc_path_1}<br>{doc_path_2}<br>{...} |
| 评测日期 | {date} |

## 阶段结论

| 字段 | 值 |
|------|----|
| 状态 | completed / failed / blocked |
| 综合评分 | X/5 |
| 参与角色 | {角色列表} |
| 阻塞问题数 | N |
| 体验问题数 | N |
| 优化建议数 | N |
| 一句话结论 | {一句话概括文档质量} |

## 角色评分

| 角色 | 综合评分 | 最强维度 | 最弱维度 | 报告 |
|------|----------|----------|----------|------|
| 客户端开发 | X/5 | 信息架构 | 前提条件 | role-client-dev.md |

## 问题摘要

| 问题 ID | 角色 | 定位 | 类别 | 严重度 | 问题 | 建议 |
|---------|------|------|------|--------|------|------|
| DOC-001 | 客户端开发 | [docs/xxx.mdx:123](https://github.com/ZEGOCLOUD/docs_all/blob/main/docs/xxx.mdx?plain=1#L123) | 📄 文档问题 | 🔴 阻塞 | {一句话问题} | {一句话建议} |

## 共性问题

| 问题 | 指出角色 | 严重度 | 定位 | 建议 |
|------|----------|--------|------|------|
| {共性问题} | 客户端开发、全栈开发 | 🔴 阻塞 | [docs/xxx.mdx:123](https://github.com/...) | {建议} |

## 改进建议排序

改进建议必须能对应到具体角色报告中的问题详情，不能只写泛泛建议。

本地 Markdown 中使用 `文件名 + 标题锚点` 指向角色报告的问题详情，例如：

- `role-client-dev.md#doc-001-appid-获取路径缺失`

飞书写入后，如果暂时没有脚本生成标题 block 链接，则在飞书报告中写清楚：

- 角色报告文档链接
- 问题 ID
- 问题详情章节标题

后续如果通过飞书 API 获取到标题 block_id，可以再升级为脚本生成章节级链接。
```
