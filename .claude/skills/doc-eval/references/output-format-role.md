# 角色评测报告输出格式

每个角色评测 agent 严格按以下格式输出：

```markdown
## [角色名] 评测报告

### 角色画像
- 角色: [角色名]
- 目标: [该角色阅读文档的目标]
- 关注路径: [该角色典型阅读路径]
- 匹配度: [⭐⭐⭐/⭐⭐/⭐]（[核心/重要/可选]角色，系数 [1.0/0.6/0.3]）

### 评分总览
| 维度 | 评分 | 说明 |
|------|------|------|
| 信息架构 | X/5 | [一句话说明，引用具体章节] |
| 内容完整性 | X/5 | [一句话说明，列出缺失项] |
| 前提条件 | X/5 | [一句话说明] |
| 参数/配置说明 | X/5 | [一句话说明] |
| 可发现性 | X/5 | [一句话说明] |
| 准确性 | X/5 | [一句话说明] |
| **综合评分** | **X/5** | [加权平均计算过程] |

### 详细问题说明

按严重程度分组，每个问题必须包含完整的定位和修复建议。

#### 🔴 阻塞问题

**问题 N: [问题标题]**
- **定位**: [GitHub 链接](使用 `scripts/doc-link.sh <相对路径> <行号>` 生成) 或 [章节名 > 子章节名]
- **原文内容**:
```markdown
引用文档中导致问题的具体内容（原文摘录）。
从有问题的文档内容行/段落摘出至少包含前后各3行内容。必要时保持组件完整性摘出来。
 **这段内容非常重要！！绝对不能省略！方便人工快速review评价**
```
- **为什么是问题**: 从该角色视角解释为什么这会导致阻塞（如"作为普通开发者，看到两种矛盾的说法，不知道该按哪个操作，无法继续集成"）
- **改进建议**: 具体的修复方案，包括建议的替换文案或补充内容

#### 🟡 体验问题

[同上格式]

#### 🟢 优化建议

[同上格式]

### 各阶段发现

#### Phase 1: [阶段名]
- ✅ 找到了: ...
- ❌ 没找到: ...
- ⚠️ 困惑点: ...

[覆盖该角色的所有 Phase]

### 集成测试简报

本节为集成测试提供输入，提炼从该角色视角的关键验证点。

#### 预期测试步骤

1. [步骤 1] - 源自 Phase X: [具体操作]
2. [步骤 2] - 源自 Phase Y: [具体操作]
...

#### 阻塞点汇总

| 问题 | 严重度 | 影响 |
|------|--------|------|
| [问题标题] | 🔴/🟡 | 导致无法完成 [具体步骤] |
...

#### 环境前置条件

- [条件 1]
- [条件 2]
...

### 亮点
- ...（引用具体内容）

### 改进建议（按优先级）
1. ...
```

## 关键要求

### 问题定位必须具体
- 必须在文档工作区根目录下使用 `scripts/doc-link.sh` 生成 GitHub 可跳转链接（如 `https://github.com/ZEGOCLOUD/docs_all/blob/main/docs/ai-agent/web/quick-start.mdx#L328`），方便飞书报告读者直接点击定位
- 用法：`bash .claude/skills/doc-eval/scripts/doc-link.sh docs/ai-agent/web/quick-start.mdx 328`
- 也可补充**章节路径**（如"前提条件 > 集成 SDK"）作为辅助定位
- 必须用代码块格式，从有问题的文档内容行/段落摘出至少包含前后各3行内容。必要时保持组件完整性摘出来。
- 如果引用的 MDX 内容来自 import 的 snippet 文件，要指出 snippet 文件的路径和行号
- 不能只说"前提条件章节"这种模糊定位

### 原文内容必须引用
- 每个问题必须摘录文档中的原文（或代码片段），让读者不用去翻原文就能看到问题所在
- 如果问题是通过对比发现的（如两处说法矛盾），两处原文都要引用

### "为什么是问题"必须从角色视角解释
- 不能只说"缺少说明"，要解释**为什么从这个角色的视角看这会阻塞或影响体验**
- 例如：CTO 视角要说"没有这个信息，CTO 无法评估技术风险"；客户端开发视角要说"作为普通开发者，不知道这个参数填什么，无法继续"

### 改进建议必须可操作
- 不能只说"补充说明"，要给出**建议的具体文案**或**建议的补充内容结构**
- 例如：不能说"补充 server 参数说明"，要说"建议将 `const server = 'xxx'` 改为 `const server = '<your_server_url>'  // 从 ZEGO 控制台 > 项目详情 > 服务器地址 获取`"

### 技术支持角色的额外输出
除标准格式外，还需输出"模拟用户问题测试"表格：

```markdown
### 模拟用户问题测试
| 用户问题 | 文档能否解答 | 定位 | 说明 |
|----------|-------------|------|------|
| [典型用户问题] | ✅/❌ | [章节位置] | ... |
```
