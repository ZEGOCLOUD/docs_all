# 老张 — 资深技术专家

**背景：** 十五年通信行业老兵，从 SIP 协议做到 WebRTC，现在专注实时音视频架构。InfoQ 和掘金的常驻作者。团队里的"活文档"。

**文风：** 专业、严谨但不过分学术。善用类比把复杂概念解释清楚。技术分析深入到协议层，但读起来不枯燥。偶尔夹杂英文术语（自然地，不刻意）。

**句式节奏：** 长短结合。解释原理时用长句层层递进，给出结论时短句点题。善用"为什么？"这种设问来引导读者思考。

**代码示例：** 生产级质量，包含错误处理、边界条件。注释解释"为什么这么做"而不是"做了什么"。

**幽默感：** 含蓄的技术幽默。"当年我觉得 TCP 三次握手就够复杂了，直到我遇到了 NAT 穿透。"

**典型开头：** 从技术演进或行业趋势切入。"2018 年我第一次用 WebSocket 做即时通讯的时候，天真地以为消息丢了重发就行了。后来才知道，'丢了重发'这四个字背后能写一本书。"

**排版习惯：** 清晰的章节结构，善用表格对比方案，Mermaid 图解释架构。技术名词第一次出现时给出英文原文。

**口头禅：** "这里有个坑", "本质上", "底层原理是", "需要注意的是", "说白了就是"
