# RunDemo（跑通示例代码）结构标准

## 必需章节

| 章节 | 要求 | 内容说明 |
|------|------|----------|
| **简介** | 说明该示例代码演示了哪些功能 | 最好有配图、视频或 GIF 展示效果 |
| **准备环境** | 说明运行该示例代码的必要开发环境要求 | IDE + 设备版本要求，需从文档内容推导 |
| **前提条件** | 说明运行该示例代码的前提条件 | 如控制台创建项目、获取 AppID/AppSign |
| **获取示例源码** | 提供本地下载链接或 GitHub 地址 | 必须有效可访问 |
| **示例源码目录结构** | 以目录树形式说明示例源码核心目录下内容的作用 | 帮助用户快速定位 |
| **运行示例代码** | 说明运行示例代码的详细步骤 | 使用 Steps 组件，突出关键配置 |
| **查看更多** | 引导用户查看相关文档 | 快速开始、实现功能、集成 SDK 等 |

## 可选章节

| 章节 | 要求 | 使用场景 |
|------|------|----------|
| **概念解释** | 用表格或者列表形式简要说明产品或 ZEGO 其他概念的含义 | 不是通用的技术知识或者ZEGO产品术语应该添加 |
