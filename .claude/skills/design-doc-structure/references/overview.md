# Overview（产品概述）结构标准

## 必需章节

| 章节 | 要求 | 内容说明 |
|------|------|----------|
| **产品简介** | 简要介绍产品是什么，解决什么问题，核心价值主张 | 1-2段话，开篇点明产品定位 |
| **核心功能** | 列出产品的主要功能特性，通常以列表形式呈现 | 配合简短说明 |
| **应用场景** | 说明产品的典型应用场景，帮助用户理解何时使用该产品 | 3-5个真实场景，可配图说明 |
| **产品架构** | 用图示说明产品的核心架构、主要组件及其关系 | 必须包含架构图 |
| **计费说明** | 说明计费模式（按分钟/按路数/免费额度等），引导至计费文档 | 简要说明 + 链接到详细计费文档 |

## 可选章节

| 章节 | 要求 | 使用场景 |
|------|------|----------|
| **产品优势** | 对比竞品或同类方案的优势（性能、稳定性、功能等） | 需要突出差异化时添加 |
| **版本说明** | 说明当前 SDK 版本对应的文档版本 | 有多版本并存时添加 |
| **相关产品** | 推荐相关的 ZEGO 产品 | 产品有关联性时添加 |

## 写作模板

```markdown
# [产品名称]

[一句话简介：产品是什么，解决什么问题]

## 产品简介

[2-3段话详细介绍产品]
- 产品的定位和目标用户
- 解决的核心问题
- 提供的价值

## 核心功能

| 功能 | 说明 |
|------|------|
| 功能1 | 功能1的简要说明 |
| 功能2 | 功能2的简要说明 |
| ... | ... |

## 应用场景

### 场景1：[场景名称]

[场景描述 + 配图/视频（如果需要）]

### 场景2：[场景名称]

[场景描述 + 配图/视频（如果需要）]

## 产品架构

<Frame width="auto" height="auto" caption="架构图"> <img src="images/architecture.png" /></Frame>

[架构说明文字]

## 计费说明

[计费模式简要说明]，详见 [计费文档](计费链接)。
```

## 检查清单

- [ ] 产品简介是否在一句话内说清楚产品是什么
- [ ] 核心功能是否覆盖产品主要特性
- [ ] 应用场景是否贴近实际业务
- [ ] 是否有清晰的架构图
- [ ] 计费说明是否准确引导至详细文档
- [ ] 文档结构是否符合模板要求
