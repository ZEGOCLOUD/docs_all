# QuickStart（快速开始）结构标准

## 必需章节

| 章节 | 要求 | 内容说明 |
|------|------|----------|
| **简介** | 说明本文档将指导用户完成什么 | 如"5 分钟实现音视频通话" |
| **应用场景** | 说明本快速开始适用的场景 | 明确场景边界，避免用户误用 |
| **概念解释** | 简要说明产品或 ZEGO 其他概念的含义 | 不是通用的技术知识或者ZEGO产品术语应该添加 |
| **前提条件** | 说明开始前需要准备的账号、权限、设备等 | 必须完整列出所有前置条件 |
| **示例代码** | 提供可下载的完整示例代码 | 方便用户直接运行体验 |
| **集成 SDK/调用服务端 API** | 说明如何集成 SDK/（CocoaPods/Maven/Gradle/NPM 等）/调用服务端 API | 提供完整集成命令/给出“API 调用方式”文档链接 |
| **实现核心功能** | 分步骤说明实现核心功能的代码流程 | 包含完整可运行的代码示例 |
| **运行项目** | 说明如何编译运行，验证功能是否正常 | 提供运行命令和预期效果 |
| **下一步** | 引导用户查看进阶功能文档 | 链接到相关功能文档 |

## 文档中代码片段要求

### 实现核心功能章节的代码片段

1. **简洁性** - 代码应尽量精简，展示核心流程。某些通用性代码可以省略，但需提及。
2. **注释** - 关键步骤必须有注释说明
3. **错误处理** - 可省略复杂错误处理，但需提及

### 示例代码章节的要求

1. **可下载** - 提供压缩包或 GitHub 链接
2. **可运行** - 下载后可直接编译运行
3. **有效果** - 包含运行效果展示

## 检查清单

### 内容完整性
- [ ] 简介是否说明目标
- [ ] 应用场景是否明确（1-3 个场景）
- [ ] 前置条件是否完整（账号、权限、环境）
- [ ] 示例代码是否可下载
- [ ] SDK 集成步骤是否准确
- [ ] 运行步骤是否清晰
- [ ] 预期效果是否说明
- [ ] 下一步引导是否合理

### 场景适配
- [ ] 是否明确说明适用的场景
- [ ] 是否引导不适用场景的用户到正确文档
- [ ] 代码示例是否符合场景需求
