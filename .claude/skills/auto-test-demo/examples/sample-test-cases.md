# 示例：ZIM 简单聊天 - 核心测试用例

## test-cases.md 示例

以下是 ZIM 聊天 demo 的核心测试用例示例。注意只覆盖 **核心 happy-path 流程**，跳过空输入验证、重复联系人、消息持久化等边界场景。

```markdown
# 测试用例 - ZIM 简单聊天 (Web)

> 平台: Web | 用户: 2 (Alice, Bob) | 仅覆盖核心功能

## 测试用例

| 编号 | 设备/标签 | 用例名称 | 操作步骤 | 预期结果 |
|------|----------|---------|---------|---------|
| TC-01 | 标签页: alice | 登录 - Alice | 在用户ID输入框输入 `user_alice`，用户名输入 `Alice`，点击登录按钮 | 页面跳转到主界面，显示聊天/联系人/我的标签 |
| TC-02 | 标签页: bob | 登录 - Bob | 在用户ID输入框输入 `user_bob`，用户名输入 `Bob`，点击登录按钮 | 页面跳转到主界面，显示聊天/联系人/我的标签 |
| TC-03 | 标签页: alice | 添加联系人 | 切换到联系人标签，输入用户ID `user_bob`，名称 `Bob`，点击添加 | Bob 出现在联系人列表中 |
| TC-04 | 标签页: alice | 发送消息 | 点击联系人 Bob 打开聊天，输入 `你好 Bob!`，按回车发送 | 消息 `你好 Bob!` 以绿色气泡显示在右侧 |
| TC-05 | 标签页: bob | 接收消息 | 切换到聊天标签 | 与 `user_alice` 的会话出现，最后一条消息为 `你好 Bob!` |
| TC-06 | 标签页: bob | 回复消息 | 打开与 Alice 的会话，输入 `你好 Alice!`，按回车发送 | 消息 `你好 Alice!` 以绿色气泡显示在右侧 |
| TC-07 | 标签页: alice | 接收回复 | 查看当前聊天窗口 | 消息 `你好 Alice!` 以白色气泡显示在左侧 |
```

## 设计要点

1. **7 个测试用例**覆盖核心流程：登录 → 添加联系人 → 发送 → 接收 → 回复 → 接收。对于示例项目来说足够。

2. **故意跳过的内容**（非核心）：
   - 空字段验证
   - 重复联系人防护
   - 登出/重新登录
   - 消息历史持久化
   - 空消息防护
   - 导航边界情况

3. **多用户流程**：TC-04 和 TC-05 是发送-接收配对，TC-06 和 TC-07 是另一对。验证核心实时消息能力。

4. **设备/标签列**：明确指定每个操作在哪个标签/设备上执行，多用户场景必须。
