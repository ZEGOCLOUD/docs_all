<Title>When the camera supports it, the preview is set to 1920 * 1080, but the actual published stream is only 640 * 480. How should this be handled?</Title>

<ArticleMetadata language="en" product="Video Call" platform="Web" />



- - -


## Problem Cause

Before calling the [createStream](https://docs.zegocloud.com/article/api?doc=Express_Video_SDK_API~javascript_web~class~ZegoExpressEngine#create-stream) interface to create and publish a stream, the previous preview video stream was not destroyed. Therefore, when users check the current published stream resolution through the `getusermedia` interface, it will be reduced.

## Solution

Before calling the SDK interface [createStream](https://docs.zegocloud.com/article/api?doc=Express_Video_SDK_API~javascript_web~class~ZegoExpressEngine#create-stream) to create a stream, please first call the [destroyStream](https://docs.zegocloud.com/article/api?doc=Express_Video_SDK_API~javascript_web~class~ZegoExpressEngine#destroy-stream) interface to destroy the previous preview video stream.
