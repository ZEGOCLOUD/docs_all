<Title>How to handle black screen or no audio during live streaming on the Web platform?</Title>

<ArticleMetadata language="en" product="Video Call / Audio Call / Live streaming" platform="Web" />



- - -

When the publishing end discovers that the local preview image suddenly goes black or the captured sound suddenly disappears, it is likely due to device loosening, poor contact, or disconnection, causing the audio or video track to stop.

When this problem occurs, you need to recreate the stream and republish after inserting the device. The specific steps are as follows:

1. Listen to the device error callback event [deviceError](https://docs.zegocloud.com/article/api?doc=Express_Video_SDK_API~javascript_web~interface~ZegoRTCEvent&jumpType=route#device-error).

<Note title="Note">


SDK version 1.16.0 and above support the [deviceError](https://docs.zegocloud.com/article/api?doc=Express_Video_SDK_API~javascript_web~interface~ZegoRTCEvent&jumpType=route#device-error) callback. If your current SDK version is lower than 1.16.0, please upgrade first.

</Note>



```javascript
const zg = zg = new ZegoExpressEngine(appID, server);
zg.on('deviceError', (errorCode, deviceName) => {
        // Prompt the user to confirm whether the device is working properly
});
```

2. After the SDK triggers the [deviceError](https://docs.zegocloud.com/article/api?doc=Express_Video_SDK_API~javascript_web~interface~ZegoRTCEvent#device-error) event, prompt the user to confirm whether the device is working properly.

- If the device is normal, end the operation.
- If the device is abnormal, please reinsert the device and perform the following 3 steps.

3. Call the [stopPublishingStream](https://docs.zegocloud.com/article/api?doc=Express_Video_SDK_API~javascript_web~class~ZegoExpressEngine&jumpType=route#stop-publishing-stream) interface to stop the current publishing.

```javascript
zg.stopPublishingStream(streamID);
```

4. Call the [createStream](https://docs.zegocloud.com/article/api?doc=Express_Video_SDK_API~javascript_web~class~ZegoExpressEngine#create-stream) interface to recreate the stream.

```javascript
const localStream = await zg.createStream();
```

5. Call the [startPublishingStream](https://docs.zegocloud.com/article/api?doc=Express_Video_SDK_API~javascript_web~class~ZegoExpressEngine#start-publishing-stream) interface to republish.

```javascript
zg.startPublishingStream(streamID, localStream)
```
