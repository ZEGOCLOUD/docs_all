<Title>After integrating Express uni-app SDK on the Windows platform, when using dev certificate cloud packaging to run iOS applications, an error occurs. How should this be handled?</Title>

<ArticleMetadata language="en" product="Video Call" platform="uni-app" />



- - -
  
## Problem Description

After integrating Express uni-app SDK on the Windows platform, when using dev certificate cloud packaging to run iOS applications, you may encounter an error message "Failed to decompress file":

<Frame width="512" height="auto" caption=""><img src="https://doc-media.zego.im/sdk-doc/Pics/QuickStart/uniapp_dev_error.png" /></Frame>
  

## Solution

- If you use adhoc certificate packaging, it runs directly without any exceptions.

- If you use dev certificate packaging and encounter the above error:

    Please manually install the custom base, base path: unpackage > debug > iOS_debug.ipa, and right-click to select "Install to Phone", as shown in the figure below:

    <Frame width="512" height="auto" caption=""><img src="https://doc-media.zego.im/sdk-doc/Pics/QuickStart/uniapp_dev_error_solution.png" /></Frame>
    <Frame width="512" height="auto" caption=""><img src="https://doc-media.zego.im/sdk-doc/Pics/QuickStart/uniapp_dev_error_solution1.png" /></Frame>

<Note title="Note">


    When recognizing Apple devices on the Windows platform, you need to install the iTunes application in advance.
    
</Note>
