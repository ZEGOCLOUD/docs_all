### When integrating through "jCenter" on the Android platform, the "jCenter" dependency fails with the error: "Failed to resolve:im.zego:express-audio:x.x.x"?

Please confirm whether the SDK version you are depending on is the latest version. Check the current SDK version through [SDK Release History](https://doc-zh.zego.im/real-time-video-android-java/client-sdk/download-sdk). If it is not the latest version, the dependency will fail.




### Why does the Electron platform fail to load the SDK when using the "Electron-Vue" framework?

The "Vue" framework needs to use "Webpack", and change the "require" parameter of "Webpack" to "window.require".



### How to determine hardware support and permission authorization on the Web platform?

Hardware support can be determined through the [checkSystemRequirements](https://www.zegocloud.com/article/api?doc=Express_Video_SDK_API~javascript_web_web~class~ZegoExpressEngine#check-system-requirements) interface by obtaining "CapabilityDetection".


### When calling the createEngine interface to create an engine, how to set the application scenario "scenario" that the App belongs to?

When calling the [createEngine](https://docs.zegocloud.com/article/api?doc=Express_Video_SDK_API~Java_android~class~im-zego-zegoexpress-zego-express-engine&jumpType=route#create-engine) interface to create an engine, you need to set the application scenario through the "scenario" parameter. ZEGO Express SDK provides the following three scenarios. The preset values of [ZegoAudioConfig](https://docs.zegocloud.com/article/api?doc=Express_Video_SDK_API~Java~class~im-zego-zegoexpress-entity-zego-audio-config) class and [ZegoEngineConfig](https://docs.zegocloud.com/article/api?doc=Express_Video_SDK_API~Java~class~im-zego-zegoexpress-entity-zego-engine-config) class are different in different scenarios. For details, please refer to the table below. Developers can choose based on the actual needs of the App they are developing.

<Warning title="Note">

If developers have customized scenario requirements, they can call the [setAudioConfig](https://docs.zegocloud.com/article/api?doc=Express_Video_SDK_API~Java_android~class~im-zego-zegoexpress-zego-express-engine&jumpType=route#set-audio-config) interface to modify the [CodecID](https://docs.zegocloud.com/article/api?doc=Express_Video_SDK_API~Java~class~im-zego-zegoexpress-entity-zego-audio-config#public-func-lists) property in the [ZegoAudioConfig](https://docs.zegocloud.com/article/api?doc=Express_Video_SDK_API~Java~class~im-zego-zegoexpress-entity-zego-audio-config) class, and contact ZEGOCLOUD Technical Support to modify the [advancedConfig](https://docs.zegocloud.com/article/api?doc=Express_Video_SDK_API~Java~class~im-zego-zegoexpress-entity-zego-engine-config#advanced-config) property in the [ZegoEngineConfig](https://docs.zegocloud.com/article/api?doc=Express_Video_SDK_API~Java~class~im-zego-zegoexpress-entity-zego-engine-config) class of the [setEngineConfig
](https://docs.zegocloud.com/article/api?doc=Express_Video_SDK_API~Java~class~im-zego-zegoexpress-zego-express-engine#set-engine-config) interface.


</Warning>
