<Title>What is the difference between the onConnectionStateChanged callback and the onRoomStateChanged callback in ZIM SDK? When are they triggered respectively?</Title>

<ArticleMetadata language="en" product="In-app chat" platform="iOS / Android / macOS / Windows / Web / Mini Program / Flutter / Electron / ReactNative / Unity3D" />



- - -

- [onConnectionStateChanged](https://docs.zegocloud.com/article/api?doc=zim_API~java_android~class~ZIMEventHandler#on-connection-state-changed) is a callback triggered when `the user's online connection state changes`, and [onRoomStateChanged](https://docs.zegocloud.com/article/api?doc=zim_API~java_android~class~ZIMEventHandler#on-room-state-changed) is a callback triggered when `the user is in a room and the room state changes`.
- If the user only calls the [login](https://docs.zegocloud.com/article/api?doc=zim_API~java_android~class~ZIM#login-1) interface to log in to ZIM, only the [onConnectionStateChanged](https://docs.zegocloud.com/article/api?doc=zim_API~java_android~class~ZIMEventHandler#on-connection-state-changed) callback is returned.
- If the user calls both the [login](https://docs.zegocloud.com/article/api?doc=zim_API~java_android~class~ZIM#login-1) interface to log in to ZIM and calls [createRoom](https://docs.zegocloud.com/article/api?doc=zim_API~java_android~class~ZIM#create-room-1) or [enterRoom](https://docs.zegocloud.com/article/api?doc=zim_API~java_android~class~ZIM#enter-room) to enter a room, both [onConnectionStateChanged](https://docs.zegocloud.com/article/api?doc=zim_API~java_android~class~ZIMEventHandler#on-connection-state-changed) and [onRoomStateChanged](https://docs.zegocloud.com/article/api?doc=zim_API~java_android~class~ZIMEventHandler#on-room-state-changed) callbacks will be returned.
