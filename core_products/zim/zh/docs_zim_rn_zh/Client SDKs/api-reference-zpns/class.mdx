<ParamField
  name="setServerBadge"
  prototype="setServerBadge(badge: number): void"
  desc="通过该接口向 ZPNs 服务器上报当前 App 的角标数。"
  parent_name="ZPNsManager"
  parent_type="class">
**参数**

| 名称 | 类型 | 描述 |
| --- | --- | --- |
| badge | number | 需要设置的值。 |

**详情**

通过该接口向 ZPNs 服务器上报当前 App 的角标数。当 App 离线后，ZPNs 将基于之前上报的角标数进行变更。

<Note title="">
- **业务场景**：当开发者本地角标发生变更时，调用该接口上报。
- **默认值**：若接收端不上报当前角标数，则接收推送时，ZPNs 不会自动修改角标。
- **调用时机**：任意时机可调用， 但实际向后台上报时机在 pushID 注册成功并且用户登录后，登出前。
</Note>

<Warning title="">
- **支持版本**：ZPNs 2.6.0 及以上版本。
- **使用限制**：暂时没有单接口限频。目前仅 iOS 端需要设置本接口。uni-app SDK 暂不支持该接口。
- **注意事项**：badge 不可小于 0。
</Warning>

</ParamField>

## ZPNsMessageReceiver

<ParamField
  name="notificationClicked"
  prototype="onNotificationClicked(): void"
  desc="SDK 离线推送点击通知事件。"
  parent_name="ZPNsMessageReceiver"
  parent_type="class">
当用户通过离线推送点击打开应用后，可以通过该回调获取到离线推送里的自定义内容。

<Note title="">
- **业务场景**：便于开发者在点击推送打开应用后获取自定义内容做业务页面跳转等操作。
</Note>

<Warning title="">

- **使用限制**：FCM 不直接支持，可以通过 data message 自行实现；Android 国内厂商需在官网控制台配置“自定义点击跳转”后生效。
</Warning>

</ParamField>
