# ZIM 升级指南

- - -

本文介绍 ZIM Flutter 平台 SDK 版本升级时的一些说明和注意事项。

## 3.0.0 升级指南

<Warning title="注意">

本次版本（3.0.0）移除了废弃超过 1 年的接口及相关枚举值和字段，同时对部分接口新增了废弃标记。请参考以下说明完成迁移。
</Warning>

### 废弃接口删除

#### ZIM 初始化接口（create）

废弃的旧版 `create(int appID)` 接口已被移除，请使用接受 [ZIMAppConfig](https://pub.dev/documentation/zego_zim/latest/zego_zim/ZIMAppConfig-class.html) 参数的 [create](https://pub.dev/documentation/zego_zim/latest/zego_zim/ZIM/create.html) 接口代替。

<CodeGroup>
```dart 新版本用法
ZIMAppConfig appConfig = ZIMAppConfig();
appConfig.appID = 12345678;
appConfig.appSign = 'appSign';
ZIM.create(appConfig);
```

```dart 旧版本用法
// 旧版接口已移除
ZIM.create(12345678);
```
</CodeGroup>

#### ZIM 登录接口（login）

废弃的旧版 `login(ZIMUserInfo, String token)` 接口已被移除，请使用接受 `userID` 和 [ZIMLoginConfig](https://pub.dev/documentation/zego_zim/latest/zego_zim/ZIMLoginConfig-class.html) 参数的 [login](https://pub.dev/documentation/zego_zim/latest/zego_zim/ZIM/login.html) 接口代替。

<CodeGroup>
```dart 新版本用法
ZIMLoginConfig loginConfig = ZIMLoginConfig();
loginConfig.userName = 'userName';
loginConfig.token = ''; // 若使用 Token 鉴权，请填写 Token
loginConfig.isOfflineLogin = false;
await ZIM.getInstance()!.login('userID', loginConfig);
```

```dart 旧版本用法
// 旧版接口已移除
ZIMUserInfo userInfo = ZIMUserInfo();
userInfo.userID = 'userID';
userInfo.userName = 'userName';
await ZIM.getInstance()!.login(userInfo, '');
```
</CodeGroup>

#### ZIM 消息发送接口（sendMessage）

废弃的 `sendPeerMessage`、`sendRoomMessage`、`sendGroupMessage` 接口已被移除，请统一使用含 `notification` 参数的 [sendMessage](https://pub.dev/documentation/zego_zim/latest/zego_zim/ZIM/sendMessage.html) 接口代替。

<CodeGroup>
```dart 新版本用法
ZIMTextMessage textMessage = ZIMTextMessage(message: 'Hello');
ZIMMessageSendConfig config = ZIMMessageSendConfig();
ZIMMessageSendNotification notification = ZIMMessageSendNotification();
notification.onMessageAttached = (ZIMMessage message) {
    // 消息发送前的业务逻辑
};
await ZIM.getInstance()!.sendMessage(
    textMessage,
    'toConversationID',
    ZIMConversationType.peer,
    config,
    notification,
);
```

```dart 旧版本用法
// 旧版接口已移除，请迁移到 sendMessage 接口
ZIMTextMessage textMessage = ZIMTextMessage(message: 'Hello');
ZIMMessageSendConfig config = ZIMMessageSendConfig();
// 旧版单聊消息发送
await ZIM.getInstance()!.sendPeerMessage(textMessage, 'toUserID', config);
// 旧版群聊消息发送
await ZIM.getInstance()!.sendGroupMessage(textMessage, 'toGroupID', config);
```
</CodeGroup>

#### ZIM 消息接收回调接口

在 2.18.0 中已废弃的 `onReceivePeerMessage`、`onReceiveRoomMessage`、`onReceiveGroupMessage` 回调在本版本中正式移除，建议迁移至 [onMessageReceived](https://pub.dev/documentation/zego_zim/latest/zego_zim/ZIMEventHandler/onMessageReceived.html)。

<CodeGroup>
```dart 新版本用法
ZIMEventHandler.onMessageReceived = (ZIM zim,
    ZIMMessageReceivedEventResult result) {
    List<ZIMMessage> messageList = result.messageList;
    ZIMMessageReceivedInfo info = result.info;
    String conversationID = result.conversationID;
    ZIMConversationType conversationType = result.conversationType;
    // 根据 conversationType 处理不同类型会话的消息
};
```

```dart 旧版本用法（已移除）
ZIMEventHandler.onReceivePeerMessage = (ZIM zim,
    List<ZIMMessage> messageList, String fromUserID) {};
ZIMEventHandler.onReceiveRoomMessage = (ZIM zim,
    List<ZIMMessage> messageList, String fromRoomID) {};
ZIMEventHandler.onReceiveGroupMessage = (ZIM zim,
    List<ZIMMessage> messageList, String fromGroupID) {};
```
</CodeGroup>

#### ZIM 呼叫邀请回调接口

旧版 `onCallInvitationTimeout(zim, callID)`（仅 callID 参数）、`onCallInvitationRejected`、`onCallInvitationAccepted`、`onCallInviteesAnsweredTimeout` 回调已被移除，请使用新版 [onCallInvitationTimeout](https://pub.dev/documentation/zego_zim/latest/zego_zim/ZIMEventHandler/onCallInvitationTimeout.html) 和 [onCallUserStateChanged](https://pub.dev/documentation/zego_zim/latest/zego_zim/ZIMEventHandler/onCallUserStateChanged.html) 代替。

<CodeGroup>
```dart 新版本用法
ZIMEventHandler.onCallInvitationTimeout = (ZIM zim,
    ZIMCallInvitationTimeoutInfo info,
    String callID) {
    // info.mode 可区分普通模式/进阶模式
};

ZIMEventHandler.onCallUserStateChanged = (ZIM zim,
    ZIMCallUserStateChangeInfo info,
    String callID) {
    // info.callUserList 包含状态变更的用户列表
    // 可统一处理接受、拒绝、超时等状态变更
};
```

```dart 旧版本用法
// 旧版回调已移除，请迁移到 onCallInvitationTimeout 和 onCallUserStateChanged
ZIMEventHandler.onCallInvitationTimeout = (ZIM zim, String callID) {};
ZIMEventHandler.onCallInvitationRejected = (ZIM zim,
    ZIMCallInvitationRejectedInfo info, String callID) {};
ZIMEventHandler.onCallInvitationAccepted = (ZIM zim,
    ZIMCallInvitationAcceptedInfo info, String callID) {};
ZIMEventHandler.onCallInviteesAnsweredTimeout = (ZIM zim,
    List<String> invitees, String callID) {};
```
</CodeGroup>

#### importLocalMessages / exportLocalMessages 接口删除

`importLocalMessages` 和 `exportLocalMessages` 接口已在 3.0.0 版本中暂时移除，且暂无替代接口。后续版本将重新开放该功能，请关注 SDK 更新日志。

### 新增废弃接口（建议迁移）

#### ZIM 消息接收回调接口

`onPeerMessageReceived`、`onRoomMessageReceived`、`onGroupMessageReceived` 回调在本版本中新增废弃标记，当前仍可正常使用，但建议尽快迁移至 [onMessageReceived](https://pub.dev/documentation/zego_zim/latest/zego_zim/ZIMEventHandler/onMessageReceived.html)。

<CodeGroup>
```dart 建议使用（新接口）
ZIMEventHandler.onMessageReceived = (ZIM zim,
    ZIMMessageReceivedEventResult result) {
    List<ZIMMessage> messageList = result.messageList;
    ZIMMessageReceivedInfo info = result.info;
    String conversationID = result.conversationID;
    ZIMConversationType conversationType = result.conversationType;
    // 根据 conversationType 处理不同类型会话的消息
};
```

```dart 废弃接口（仍可用，建议迁移）
ZIMEventHandler.onPeerMessageReceived = (ZIM zim,
    List<ZIMMessage> messageList, ZIMMessageReceivedInfo info,
    String fromUserID) {};
ZIMEventHandler.onRoomMessageReceived = (ZIM zim,
    List<ZIMMessage> messageList, ZIMMessageReceivedInfo info,
    String fromRoomID) {};
ZIMEventHandler.onGroupMessageReceived = (ZIM zim,
    List<ZIMMessage> messageList, ZIMMessageReceivedInfo info,
    String fromGroupID) {};
```
</CodeGroup>

#### ZIMGroupConversation 废弃

`ZIMGroupConversation` 类在 3.0.0 中已被废弃，其中的 `isDisabled` 和 `mutedExpiredTime` 字段请通过基类 `ZIMConversation` 的对应属性替代：

| ZIMGroupConversation（已废弃） | ZIMConversation（替代属性） |
|---|---|
| `isDisabled` | `isConversationDisabled` |
| `mutedExpiredTime` | `selfMutedExpiredTime` |

<CodeGroup>
```dart 新版本用法
bool isDisabled = conversation.isConversationDisabled;
int mutedExpiredTime = conversation.selfMutedExpiredTime;
```

```dart 旧版本用法（已废弃）
ZIMGroupConversation groupConversation = conversation as ZIMGroupConversation;
bool isDisabled = groupConversation.isDisabled;
int mutedExpiredTime = groupConversation.mutedExpiredTime;
```
</CodeGroup>

### 枚举值移除

#### ZIMMessageType 枚举值移除

`ZIMMessageType.system`（值为 30）已从枚举中移除，同时 `ZIMSystemMessage` 类也已被移除。如需发送系统级指令，请使用 [ZIMCommandMessage](https://pub.dev/documentation/zego_zim/latest/zego_zim/ZIMCommandMessage-class.html) 代替。

<CodeGroup>
```dart 新版本用法
// 使用 ZIMCommandMessage 代替 ZIMSystemMessage
ZIMCommandMessage commandMessage = ZIMCommandMessage(
    message: Uint8List.fromList('payload'.codeUnits),
);
await ZIM.getInstance()!.sendMessage(
    commandMessage,
    'toConversationID',
    ZIMConversationType.peer,
    ZIMMessageSendConfig(),
    null,
);
```

```dart 旧版本用法
// ZIMMessageType.system 和 ZIMSystemMessage 已被移除
ZIMSystemMessage systemMessage = ZIMSystemMessage();
// ...
```
</CodeGroup>

#### ZIMCallUserState 枚举值移除

`ZIMCallUserState.offline`（值为 4）已从枚举中移除，请检查代码中是否有对该枚举值的判断并将其删除。

```dart
// 以下枚举值已移除，请检查并删除代码中对该值的引用
// ZIMCallUserState.offline  （值为 4）
```

### 字段变更

#### ZIMUserFullInfo.userAvatarUrl 废弃

`ZIMUserFullInfo.userAvatarUrl` 已废弃，请使用 `ZIMUserFullInfo.baseInfo.userAvatarUrl` 代替。

<CodeGroup>
```dart 新版本用法
String avatarUrl = userFullInfo.baseInfo.userAvatarUrl;
```

```dart 旧版本用法
String avatarUrl = userFullInfo.userAvatarUrl; // 已废弃
```
</CodeGroup>

#### ZIMMessage.conversationSeq 替换为 messageSeq

`ZIMMessage.conversationSeq` 已被移除，请使用 `ZIMMessage.messageSeq` 代替。

<CodeGroup>
```dart 新版本用法
int seq = message.messageSeq;
```

```dart 旧版本用法
int seq = message.conversationSeq; // 已移除
```
</CodeGroup>

#### ZIMMessageDeletedInfo.isDeleteConversationAllMessage 移除

`ZIMMessageDeletedInfo.isDeleteConversationAllMessage` 已被移除，请使用 `ZIMMessageDeletedInfo.messageDeleteType` 代替。

<CodeGroup>
```dart 新版本用法
ZIMMessageDeleteType deleteType = deletedInfo.messageDeleteType;
// 通过 deleteType 判断是单条删除还是全量删除
```

```dart 旧版本用法
bool isDeleteAll = deletedInfo.isDeleteConversationAllMessage; // 已移除
```
</CodeGroup>

#### ZIMGroupMemberInfo.memberAvatarUrl 移除

`ZIMGroupMemberInfo.memberAvatarUrl` 已被移除，请使用 `ZIMGroupMemberInfo.userAvatarUrl` 代替。

<CodeGroup>
```dart 新版本用法
String avatarUrl = groupMemberInfo.userAvatarUrl;
```

```dart 旧版本用法
String avatarUrl = groupMemberInfo.memberAvatarUrl; // 已移除
```
</CodeGroup>

#### ZIMGroupOperatedInfo 结构变更

`ZIMGroupOperatedInfo.operatedUserInfo` 字段已被移除，其内部字段已平铺到 `ZIMGroupOperatedInfo`，可直接从 `ZIMGroupOperatedInfo` 中获取对应字段。

<CodeGroup>
```dart 新版本用法
// 直接从 ZIMGroupOperatedInfo 获取字段
String operatorUserID = groupOperatedInfo.userID;
String operatorUserName = groupOperatedInfo.userName;
String operatorAvatarUrl = groupOperatedInfo.userAvatarUrl;
```

```dart 旧版本用法
// operatedUserInfo 已移除
String operatorUserID = groupOperatedInfo.operatedUserInfo.userID;
String operatorUserName = groupOperatedInfo.operatedUserInfo.userName;
```
</CodeGroup>

#### ZIMCallInvitationSentInfo.errorInvitees 替换

`ZIMCallInvitationSentInfo.errorInvitees` 已被移除，请使用 `ZIMCallInvitationSentInfo.errorUserList` 代替，类型由 `List<ZIMCallUserInfo>` 变更为 `List<ZIMErrorUserInfo>`。

<CodeGroup>
```dart 新版本用法
final result = await ZIM.getInstance()!.callInvite(invitees, config);
for (final errorUser in result.info.errorUserList) {
    // errorUser.userID, errorUser.reason
}
```

```dart 旧版本用法
// errorInvitees 已移除，请使用 errorUserList
final result = await ZIM.getInstance()!.callInvite(invitees, config);
for (final errorInvitee in result.info.errorInvitees) {
    // errorInvitee.userID, errorInvitee.state
}
```
</CodeGroup>

#### ZIMConversationChangeInfo 字段变更

`ZIMConversationChangeInfo` 中的 `event` 属性（类型 `ZIMConversationEvent`）已被移除，请使用同结构下的 `action` 属性（类型 `ZIMConversationChangeAction`）代替。

<CodeGroup>
```dart 新版本用法
ZIMEventHandler.onConversationChanged = (ZIM zim,
    ZIMConversationChangeEventResult result) {
    for (ZIMConversationChangeInfo changeInfo in result.infoList) {
        ZIMConversationChangeAction action = changeInfo.action;
        // 根据 action 处理会话变更
    }
};
```

```dart 旧版本用法
ZIMEventHandler.onConversationChanged = (ZIM zim,
    ZIMConversationChangeEventResult result) {
    for (ZIMConversationChangeInfo changeInfo in result.infoList) {
        ZIMConversationEvent event = changeInfo.event; // 已移除
    }
};
```
</CodeGroup>

## 2.28.0 升级指南

<Warning title="注意">

ZIM SDK 2.28.0 版本调整了部分接口的返回参数和事件回调触发的时机。从旧版本升级到 2.28.0 版本时，请您阅读以下指南。
</Warning>

### 接口定义变更

<Steps>
<Step title="addMessageReaction">

<CodeGroup>
```dart 新版本定义 {2-6}
Future<ZIMMessageReactionAddedResult> addMessageReaction(
  {
    required String reactionType,
    required ZIMMessage message,
    ZIMMessageReactionAddConfig? config,
  }
);
```

```dart 旧版本定义 {2,3}
Future<ZIMMessageReactionAddedResult> addMessageReaction(
  String reactionType, 
  ZIMMessage message
);
```
</CodeGroup>

<CodeGroup>
```dart 新版本用法 {2,3}
final result = await zim.addMessageReaction(
```

```dart 旧版本用法 {2,3}
final result = await zim.addMessageReaction(
```

</CodeGroup>
</Step>

<Step title="deleteMessageReaction">

```dart 新版本定义 {2-5}
Future<ZIMMessageReactionDeletedResult> deleteMessageReaction(
```

### 事件回调时机变更

<Warning title="注意">

以下回调自 2.28.0 版本起触发时机有变化，建议开发者检查相应业务逻辑在升级 SDK 后是否受到影响。
</Warning>

<Steps>
<Step title="onMessageReactionsChanged">
[onMessageReactionsChanged](https://pub.dev/documentation/zego_zim/latest/zego_zim/ZIMEventHandler/onMessageReactionsChanged.html) 回调会在客户端调用 `addMessageReaction`、`deleteMessageReaction` 或 `deleteMessageAllReactions` 成功后触发。
</Step>

<Step title="onConversationChanged">
[onConversationChanged](https://pub.dev/documentation/zego_zim/latest/zego_zim/ZIMEventHandler/onConversationChanged.html) 回调会在客户端调用 `deleteConversation` 成功后触发。
</Step>

<Step title="onConversationsAllDeleted">
[onConversationsAllDeleted](https://pub.dev/documentation/zego_zim/latest/zego_zim/ZIMEventHandler/onConversationsAllDeleted.html) 回调会在客户端调用 `deleteAllConversations` 成功后触发。
</Step>

</Steps>


## 2.23.0 升级指南

<Warning title="注意">

ZIM Flutter SDK 2.23.0 版本兼容修复了 Native 和 Web 平台的部分字段的类型错误的问题。从旧版本升级到该版本时，请您阅读以下指南。
</Warning>


### 接口类型变更

| 字段              | 老版本类型 | 新版本类型 | 实际类型处理 |
|------------------|------------|------------|------------|
| `messageID`      | `int`      | `dynamic`  | Web 平台: String，其他平台：int
| `localMessageID` | `int`      | `dynamic`  | Web 平台: String，其他平台：int
| `fileLocalPath`  | `String`   | `dynamic`  | Web 平台: UInt8List，其他平台：String

### 代码调用示例

```dart 新版本用法 
dynamic messageID = msg.messageID;
dynamic localMessageID = msg.localMessageID;
if (kIsWeb) {
  String myMessageID = messageID as String;
  String myLocalMessageID = localMessageID as String;
} else {
  int myMessageID = messageID as int;
  int myLocalMessageID = localMessageID as int;
}

var fileLocalPath;
if (kIsWeb) {
  //  Web 平台：本地文件内容  
  fileLocalPath = Uint8List.fromList([1, 2, 3]);
} else {
  //  其他平台：本地文件绝对路径  
  fileLocalPath = '';
}
ZIMImageMessage imageMessage = ZIMImageMessage(fileLocalPath);
```


## 2.21.1 升级指南



```dart 2.13.0版本用法
    loginConfig.isOfflineLogin = false;
```


#### 2. [getInstance](https://pub.dev/documentation/zego_zim/latest/zego_zim/ZIM/getInstance.html) 方法变更

将 [getInstance](https://pub.dev/documentation/zego_zim/latest/zego_zim/ZIM/getInstance.html) 方法的返回值从 `ZIM` 改为 `ZIM?`，因此，请您注意处理空安全问题。

每一个 API 接口都需要调整，下面展示其中一个 API 用法调整。

<CodeGroup>
```dart 2.3.0版本用法
await ZIM.getInstance()!.login(userInfo, token);
```

```dart 旧版本用法
await ZIM.getInstance().login(userInfo, token);
```
</CodeGroup>

#### 4. 删除不必要的 Future 返回值

我们删除了一些 API 接口中不必要的 Future 返回值，这样就不需要 `await` 返回值了。

涉及到的接口有：[destroy](https://pub.dev/documentation/zego_zim/latest/zego_zim/ZIM/destroy.html)、[logout](https://pub.dev/documentation/zego_zim/latest/zego_zim/ZIM/logout.html)、[setLogConfig](https://pub.dev/documentation/zego_zim/latest/zego_zim/ZIM/setLogConfig.html)、[setCacheConfig](https://pub.dev/documentation/zego_zim/latest/zego_zim/ZIM/setCacheConfig.html)、[beginRoomAttributesBatchOperation](https://pub.dev/documentation/zego_zim/latest/zego_zim/ZIM/beginRoomAttributesBatchOperation.html)。

<CodeGroup>
```dart 2.3.0版本用法
ZIM.setLogConfig(config);
......

ZIM.setCacheConfig(config);
......

ZIM.getInstance()!.beginRoomAttributesBatchOperation(roomID, config);
......

ZIM.getInstance()!.logout();
......

ZIM.getInstance()!.destroy();
......
```

```dart 旧版本用法
await ZIM.getInstance().setLogConfig(config);
......

await ZIM.getInstance().setCacheConfig(config);
......

await ZIM.getInstance().beginRoomAttributesBatchOperation(roomID, config);
......

await ZIM.getInstance().logout();
......

await ZIM.getInstance().destroy();
......
```
</CodeGroup>
