# 呼叫中邀请回调

---

## 描述

**呼叫中邀请** 是指处于呼叫中的用户再邀请其他用户进入此次呼叫的行为。当处于呼叫中的用户在客户端发送邀请后，业务后台会收到 ZIM 服务端的呼叫中邀请回调，用于获取本次邀请的详情信息。

**使用场景：** 处于呼叫中的用户再邀请其他用户进入此次呼叫时，业务后台会收到 ZIM 服务端的呼叫中邀请回调，可用于确认是否成功邀请其他用户。

## 回调说明
- 请求方法：POST。
<Note title="说明">
回调数据格式为 JSON，您需要对其进行 UrlDecode 解码。
</Note>
- 请求地址：请在 [ZEGO 控制台](https://console.zego.im/) 上配置回调地址，配置流程请参考控制台文档 [ZIM 相关回调配置](/console/service-configuration/im/call-callback-config)。
- 传输协议：HTTPS/HTTP，建议使用 HTTPS。
## 回调参数


| 参数         | 类型    | 说明                                                                                                                                                  |
| ------------ | ------- | ----------------------------------------------------------------------------------------------------------------------------------------------------- |
| appid        | String  | APP 的唯一标识。                                                                                                                                      |
| event        | String  | 回调事件，此回调返回值为 `call_invite`。                                                                                                              |
| timestamp    | Int     | 服务器当前时间， Unix 时间戳。                                                                                                                          |
| nonce        | String  | 随机数。                                                                                                                                              |
| signature    | String  | 检验串，详情见 [检验说明](./authenticating-server-to-server-callbacks.mdx)。                                                                                                                           |
| call_id      | String  | 呼叫 ID。                                                                                                                                             |
| user_ids     | Array of String   | 被邀请成功的用户 ID 列表。<Warning title="注意">不包含邀请失败的用户，比如已经邀请过某个用户，且该用户已经接受邀请。</Warning>                                               |
| inviter      | String  | 邀请发起者。                                                                                                                                          |
| invite_time  | Int     | 邀请的时间。服务器当前时间，Unix 时间戳。单位：毫秒。                                                                                                 |

<Note title="说明">
建议您将 `appid` 和 `nonce` 参数转换为 Int 进行逻辑处理。
</Note>

## 数据示例

```json
{
    "appid": "1",
    "event": "call_invite",
    "nonce": "350176",
    "signature": "signature",
    "timestamp": 1499676978,
    "call_id": "3501761173612493269",
    "user_ids": ["abc","def"],
    "inviter": "aaa",
    "invite_time": 1499676978000
}

```

## 返回响应
返回 HTTP status code 为 2XX （例如 200）表示成功，其他响应都表示失败。

## 回调重试策略

如果 ZEGO 服务器没有收到响应，或收到的 HTTP status code 不为 2XX（例如 200），都会尝试重试 (**最多进行 5 次重试**)。   
每次重试请求与**上一次请求**的间隔时间分别为 2s、4s、8s、16s、32s。若第 5 次重试后仍然失败，将不再重试，该回调丢失。
