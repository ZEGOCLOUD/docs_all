
# API 概览

---

ZIM 服务端提供以下相关 API 接口及回调，可参考 [调用方式](./accessing-server-apis.mdx) 调用以下接口。

|接口名称|接口功能| 默认调用频率限制|
|-|-|-|

|[批量注册用户](./user/batch-register-users.mdx)| 批量帮用户注册 IM 账号。| 20 次/秒 |

|[批量添加好友](./user/batch-add-friends.mdx)| 为用户直接批量添加至多 20 名好友，无需对方同意。| 20 次/秒 |
|[批量发送好友申请](./user/batch-send-friend-requests.mdx)| 为一名用户批量向至多 20 名其他用户发送好友申请。| 20 次/秒 |
|[批量删除好友](./user/batch-delete-friends.mdx)| 为用户批量单向或双向删除至多 20 名好友。| 20 次/秒 |

|[查询好友列表](./user/query-the-friend-list.mdx)| 根据用户 ID 分页拉取其好友列表，获取备注、好友关系建立时间等信息。| 20 次/秒 |

|[更新好友属性](./user/modify-the-attributes-of-a-friend.mdx)| 为用户修改其好友的好友属性。| 20 次/秒 |
|[批量拉黑用户](./user/batch-block-users.mdx)| 为用户批量拉黑至多 20 名用户，不再接收相关用户消息。| 20 次/秒 |
|[批量移除黑名单](./user/batch-unblock-users.mdx)| 为用户批量将 20 名用户从黑名单移除。| 20 次/秒 |
|[查询黑名单](./user/query-the-blocklist.mdx)| 根据用户 ID 分页拉取其全量黑名单数据。| 20 次/秒 |
|[检查黑名单关系](./user/check-blockships.mdx)| 为一名用户批量检查其与至多 20 名其他用户的黑名单关系（是否已拉黑这些用户）。| 20 次/秒 |

### 呼叫相关

| 回调名称 | 回调说明 |
|---------|---------|
| [呼叫创建回调](./callbacks/call-invitation-sent.mdx) | 用户在客户端成功创建呼叫后，业务后台可以接收 ZIM 服务端的呼叫创建回调，用于确定呼叫创建成功请求。 |
| [呼叫取消回调](./callbacks/call-invitation-canceled.mdx) | 当出现以下情况时，业务后台可以接收 ZIM 服务端的呼叫取消回调，用于确定呼叫状态：<ul><li>呼叫发起用户成功取消呼叫</li><li>呼叫发起用户在成功创建呼叫后主动登出或心跳超时后为登出态</li></ul> |
| [呼叫接受回调](./callbacks/call-invitation-accepted.mdx) | 当呼叫对象接受呼叫后，业务后台可以接收 ZIM 服务端的呼叫接受回调，用于确定呼叫接受成功请求。 |
| [呼叫拒绝回调](./callbacks/call-invitation-rejected.mdx) | 当呼叫对象拒绝呼叫后，业务后台可以接收 ZIM 服务端的呼叫拒绝回调，用于确定呼叫拒绝成功请求。 |
| [呼叫超时回调](./callbacks/call-invitation-timed-out.mdx) | 呼叫创建成功后，当出现以下情况时，业务后台可以接收 ZIM 服务端的呼叫超时回调，用于确定呼叫超时用户：<ul><li>呼叫对象一直未应答，直到满足后台配置的超时时间</li><li>呼叫对象一直未应答，直到满足创建呼叫时设置的超时时间</li></ul> |

### 消息相关

| 回调名称 | 回调说明 |
|---------|---------|
| [消息发送前回调](./callbacks/message-not-sent-yet.mdx) | 设置此回调后，当用户发送单聊、群聊或房间聊天消息时，ZIM 会向您的业务后台发起请求，您可以通过应答，对消息进行实时操作。 |
