# 频道消息管理

---

## 功能简介

社群（Community）是 ZIM 提供的类 Discord 实时互动社区能力，采用"社群 - 频道"的两层组织结构。频道是社群内聊天互动的基本载体，功能类似传统群组，支持发送文本、图片、语音、视频等各类消息。

本文介绍如何在社群频道中收发消息。频道消息的发送与接收基于 ZIM 标准消息接口，统一使用 `ZIMConversationType.COMMUNITY_CHANNEL` 作为会话类型。

<Note title="说明">
- 社群功能为旗舰版功能，使用前请联系 ZEGO 技术支持开通。
- 支持的消息类型与群组一致，包括文本、富媒体、自定义、组合等，详见 [收发消息 - 消息类型](../Messaging/Send%20and%20receive%20messages.mdx#消息类型)。
</Note>

## 前提条件

发送频道消息前，需要先让用户加入（或创建）一个社群，并获取目标频道的 `conversationID`。

### 步骤一：创建或加入社群

**创建社群**

调用 {getPlatformData(props,createCommunityMap)} 接口创建社群。创建成功后，系统会自动为该社群创建一个类型为 `GENERAL` 的默认频道。

:::if{props.platform=undefined}
```java
// 构造社群基本信息
ZIMCommunityInfo communityInfo = new ZIMCommunityInfo();
communityInfo.setCommunityID("community_001");   // 社群 ID，由开发者自定义
communityInfo.setCommunityName("我的社群");
communityInfo.setCommunityAvatarUrl("https://example.com/avatar.png");

ZIMCommunityCreateConfig config = new ZIMCommunityCreateConfig();
// 可选：设置社群公告
config.setCommunityNotice("欢迎加入！");

zim.createCommunity(communityInfo, config, (ZIMCommunityFullInfo fullInfo, ZIMError error) -> {
    if (error.code == ZIMErrorCode.SUCCESS) {
        // 创建成功，fullInfo 包含完整社群信息
    }
});
```
:::

**加入已有社群**

已知 `communityID` 时，调用 {getPlatformData(props,joinCommunityMap)} 接口加入社群。

:::if{props.platform=undefined}
```java
String communityID = "community_001";

zim.joinCommunity(communityID, (ZIMCommunityFullInfo fullInfo, ZIMError error) -> {
    if (error.code == ZIMErrorCode.SUCCESS) {
        // 加入成功，fullInfo 包含完整社群信息
    }
});
```
:::

### 步骤二：获取频道列表

加入社群后，调用 {getPlatformData(props,queryCommunityChannelListMap)} 接口查询频道列表，获取频道对象 {getPlatformData(props,ZIMCommunityChannelMap)}。

频道对象的 `conversationID` 字段即是后续收发消息所需的会话 ID。

:::if{props.platform=undefined}
```java
String communityID = "community_001";
int count = 100; // 每次最多拉取的频道数量

ZIMCommunityChannelListQueryConfig config = new ZIMCommunityChannelListQueryConfig();
// config.nextFlag 用于分页，首次查询无需设置，下次查询填入上次返回的 nextFlag

zim.queryCommunityChannelList(communityID, count, config,
    (ArrayList<ZIMCommunityChannel> channelList, long nextFlag, ZIMError error) -> {
        if (error.code == ZIMErrorCode.SUCCESS) {
            for (ZIMCommunityChannel channel : channelList) {
                // channel.baseInfo.channelID   — 频道 ID
                // channel.baseInfo.channelName — 频道名称
                // channel.conversationID        — 发送/接收消息时使用的会话 ID
            }
        }
    });
```
:::

<Note title="说明">
- `ZIMCommunityChannel.conversationID` 与 `ZIMCommunityChannelInfo.channelID` 不同，发送消息时请使用 `conversationID`。
- 如果社群频道较多，可根据返回的 `nextFlag` 多次分页拉取，直到返回的 `nextFlag` 为 0 或列表为空。
</Note>

## 发送消息

获取到目标频道的 `conversationID` 后，调用 {getPlatformData(props,sendMessageMap)} 接口发送消息。与其他会话类型的区别在于：需要将 `conversationType` 设置为 `ZIMConversationType.COMMUNITY_CHANNEL`。

<Warning title="注意">
- 发送消息的接口与单聊、群组、房间共用同一个 {getPlatformData(props,sendMessageMap)}，通过 `conversationType` 区分目标会话类型。
- 两次发送消息之间的间隔须大于 100ms。
</Warning>

### 普通消息

:::if{props.platform=undefined}
```java
// 获取到目标频道后，用其 conversationID 发送消息
String conversationID = channel.conversationID; // 来自 queryCommunityChannelList 返回结果

ZIMTextMessage textMessage = new ZIMTextMessage();
textMessage.setMessage("Hello, Channel!");

ZIMMessageSendConfig config = new ZIMMessageSendConfig();
// 设置消息优先级：LOW(1) / MEDIUM(2) / HIGH(3)
config.setPriority(ZIMMessagePriority.LOW);

zim.sendMessage(
    textMessage,
    conversationID,
    ZIMConversationType.COMMUNITY_CHANNEL,   // 会话类型：社群频道
    config,
    new ZIMMessageSentFullCallback() {
        @Override
        public void onMessageAttached(ZIMMessage message) {
            // 消息已通过本地校验，准备发送；可在此更新 UI（如展示发送中状态）
        }

        @Override
        public void onMessageSent(ZIMMessage message, ZIMError error) {
            if (error.code == ZIMErrorCode.SUCCESS) {
                // 消息发送成功
            } else {
                // 消息发送失败，可根据 error.code 处理
            }
        }

        @Override
        public void onMediaUploadingProgress(ZIMMediaMessage message, long currentFileSize, long totalFileSize) {
        }

        @Override
        public void onMultipleMediaUploadingProgress(ZIMMediaMessage message, long currentFileSize, long totalFileSize, int index) {
        }
    }
);
```
:::

### 富媒体消息

发送图片、文件、音频、视频等富媒体消息时，同样将 `conversationType` 设置为 `ZIMConversationType.COMMUNITY_CHANNEL`，其余用法与群组消息一致。

:::if{props.platform=undefined}
```java
// 以发送图片消息为例
ZIMImageMessage imageMessage = new ZIMImageMessage(
    "/storage/emulated/0/Android/data/com.example/pictures/photo.jpg"
);

ZIMMessageSendConfig config = new ZIMMessageSendConfig();
config.setPriority(ZIMMessagePriority.MEDIUM);

zim.sendMessage(
    imageMessage,
    conversationID,
    ZIMConversationType.COMMUNITY_CHANNEL,
    config,
    new ZIMMessageSentFullCallback() {
        @Override
        public void onMessageAttached(ZIMMessage message) {
            // 可在此提前展示图片占位 UI
        }

        @Override
        public void onMessageSent(ZIMMessage message, ZIMError error) {
            if (error.code == ZIMErrorCode.SUCCESS) {
                // 图片发送成功，message 中包含服务端生成的 fileDownloadUrl
            }
        }

        @Override
        public void onMediaUploadingProgress(
            ZIMMediaMessage message, long currentFileSize, long totalFileSize) {
            // 可在此更新上传进度条
        }
    }
);
```
:::

<Note title="说明">
各消息类型（自定义消息、组合消息等）的构造方式与发送参数，请参考 [收发消息](../Messaging/Send%20and%20receive%20messages.mdx)。将对应示例代码中的 `conversationType` 替换为 `ZIMConversationType.COMMUNITY_CHANNEL`，`toConversationID` 替换为频道的 `conversationID` 即可。
</Note>

## 接收消息

ZIM SDK 3.0.0 起，通过统一的 {getPlatformData(props,onMessageReceivedMap)} 回调接收来自单聊、群组、房间和社群频道的所有在线消息。回调返回 {getPlatformData(props,ZIMMessageReceivedEventResultMap)}，其中 `receivedInfo.conversationType` 字段用于判断消息所属的会话类型。

:::if{props.platform=undefined}
```java
zim.setEventHandler(new ZIMEventHandler() {
    @Override
    public void onMessageReceived(ZIM zim, ZIMMessageReceivedEventResult result) {
        ZIMMessageReceivedInfo info = result.receivedInfo;

        // 过滤社群频道消息
        if (info.conversationType == ZIMConversationType.COMMUNITY_CHANNEL) {
            String conversationID = info.conversationID; // 消息所属频道的 conversationID

            for (ZIMMessage message : result.messageList) {
                // 根据消息类型处理
                switch (message.type) {
                    case TEXT:
                        ZIMTextMessage textMsg = (ZIMTextMessage) message;
                        // 处理文本消息
                        break;
                    case IMAGE:
                        ZIMImageMessage imageMsg = (ZIMImageMessage) message;
                        // 处理图片消息，可调用 downloadMediaFile 下载原图
                        break;
                    default:
                        break;
                }
            }
        }
    }
});
```
:::

<Note title="说明">
- 用户重新登录后，可通过此回调收到离线期间积压的频道消息（需服务端开启频道离线消息存储）。
- 如需主动拉取历史消息，请参考 [查询历史消息](../Messaging/Get%20message%20history.mdx)，传入频道的 `conversationID` 和 `ZIMConversationType.COMMUNITY_CHANNEL`。
</Note>

## 进阶操作

在基础收发消息之外，社群频道还支持以下进阶消息操作。各功能的 `conversationID` 传入频道的 `conversationID`，`conversationType` 传入 `ZIMConversationType.COMMUNITY_CHANNEL` 即可，详细用法参见对应文档。

| 功能 | 说明 |
|------|------|
| [获取历史消息](../Messaging/Get%20message%20history.mdx) | 分页拉取频道内的历史消息，支持按时间范围、消息序号等条件查询。 |
| [插入本地消息](../Messaging/Insert%20local%20messages.mdx) | 向频道本地数据库插入一条仅本端可见的提示消息，不会发送给其他成员。 |
| [撤回消息](../Messaging/Recall%20messages.mdx) | 撤回频道内自己发送的消息；管理员或群主可撤回他人消息。 |
| [设置消息拓展字段](../Messaging/Set%20message%20extension%20field.mdx) | 为频道消息附加对端可见或仅本端可见的拓展字段。 |
| [搜索本地消息](../Messaging/Search%20for%20local%20messages.mdx) | 按关键字、用户 ID 等条件搜索频道内的本地历史消息。 |
| [消息表态](../Messaging/Respond%20to%20messages%20with%20emoticons.mdx) | 对频道消息添加或删除 Emoji 表态，支持群组投票等场景。 |
| [接收 Tips 消息](../Messaging/Receive%20tip%20messages.mdx) | 接收频道内由系统自动产生的操作提示消息（如成员加入/离开等）。 |
| [编辑消息](../Messaging/Edit%20messages.mdx) | 修改频道内已发送的消息，变更内容实时同步给所有频道成员。 |
