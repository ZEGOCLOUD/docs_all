# API overview


- - -

ZEGOCLOUD's In-app Chat provides you with the following Server APIs, please refer to [Accessing Server APIs](./Accessing%20Server%20APIs.mdx) to call the following interface.

## Callback

### User callbacks

| Callback | Description |
|----------|-------------|
| [Login and logout callback](./Callbacks/Login%20and%20logout.mdx) | Listen for user login and logout and implement the business logic, for example, counting online users. |

### Room callbacks

| Callback | Description |
| -------- | -------- |
| [Room created callback](./Callbacks/room/room-created.mdx) | Real-time monitoring of user room creation behavior to view information about rooms created by users. |
| [Room entered callback](./Callbacks/room/room-entered.mdx)| Real-time monitoring of user joining specific room information, which can be used to notify the backend when members enter a room. |
| [Room left callback](./Callbacks/room/room-left.mdx)| Real-time monitoring of user leaving specific room information, which can be used to notify the backend when members leave a room. |
| [Room destroyed callback](./Callbacks/room/room-destroyed.mdx) | Real-time monitoring of user room destruction behavior, which can be used to record logs of users destroying rooms. |

### Group callbacks

| Callback | Description |
| -------- | -------- |
| [Group created callback](./Callbacks/group/group-created.mdx) | Real-time monitoring of user group creation behavior to obtain information about the group. |
| [Group joined callback](./Callbacks/group/group-joined.mdx) | Real-time viewing of user joining specific group information, which can be used to notify the backend when members join a group. |
| [Group left callback](./Callbacks/group/group-left.mdx) | Real-time monitoring of user leaving specific group information, which can be used to notify the backend when members leave a group. |
| [Group destroyed callback](./Callbacks/group/group-destroyed.mdx) | Real-time monitoring of user group dissolution behavior, which can be used to record logs of users dissolving groups. |

### Call callbacks

| Callback | Description |
|---------|---------|
| [Callback on call invitation sent](./Callbacks/Call%20invitation%20sent.mdx) | After a call is created on the client, the business backend receives the call creation callback from the ZIM server to check whether the call is created successfully. |
| [Callback on call invitation canceled](./Callbacks/Call%20invitation%20canceled.mdx) | When the following situations occur, the business backend can receive a callback from the ZIM server to cancel a call, which helps determine the call status:<br/>- The initiating user successfully cancels the call.<br/>- The initiating user proactively logs out or enters a logged-out state due to a timeout after successfully creating the call. |
| [Callback on call invitation accepted](./Callbacks/Call%20invitation%20accepted.mdx) | After the callee accepts the call invitation, the business server can receive the callback on the call invitation accepted by the In-app Chat Server to determine the success of the call invitation acceptance request. |
| [Callback on call invitation rejected](./Callbacks/Call%20invitation%20rejected.mdx) | After the callee rejects the call invitation, the business server can receive the callback on the call invitation rejected by the In-app Chat Server to determine the success of the call invitation request. |
| [Callback on call invitation timed out](./Callbacks/Call%20invitation%20timed%20out.mdx) | After the call invitation is sent successfully, when the following situations occur, the business server can receive the call timeout callback from the In-app Chat Server to know the target user:<br/>- The callee has not answered until the timeout period configured in the business server is met.<br/>- The callee does not answer until the timeout period set when the call was created is met. |

### Message callbacks

| Callback | Description |
|---------|---------|
| [Callback on message not sent yet](./Callbacks/Message%20not%20sent%20yet.mdx) | After setting this callback, when a user sends a message to a one-on-one chat, group chat or room chat, In-app Chat will initiate a request to your business server, and you can perform real-time operations on the message through the response. |
| [Callback on message sent](./Callbacks/Message%20sent.mdx) | After the user sends a message to one-on-one chat, group chat, or room successfully or fails, the business server can receive the message sending callback from the In-app Chat server, and synchronize the message sent by the user to the business server in real time, and store it in the business server. |
| [Callback on message sent by a one-to-one conversation bot](./Callbacks/One%20to%20one%20conversation%20bot%20message%20sent.mdx) | After the one-to-one conversation bot sends a message successfully or fails, your server can receive the message sending callback from the In-app Chat server, and synchronize the message sent by the one-to-one conversation bot to your server in real time, and store it in your server. |
