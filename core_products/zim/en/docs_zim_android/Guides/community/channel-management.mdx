# Channel management

- - -

## Feature overview

A Channel is the basic carrier for chat interaction within a community. It functions similarly to a traditional group, and each community can contain multiple channels. ZIM SDK provides comprehensive channel management capabilities, supporting the following features:

- Create/dismiss a channel
- Update channel name, avatar, and notice
- Set/delete channel custom attributes
- Query channel information and list
- Set channel mute status
- Listen for channel list changes and information updates

## Prerequisites

- Please refer to [Implement basic message sending and receiving](./Messaging/Send%20and%20receive%20messages.mdx) to complete ZIM SDK initialization and user login.
- Please refer to [Authenticate with Token](./Users/Authentication.mdx) to implement user authentication login.
- Community features require ZIM SDK 3.0.0 or later.
- Community features are part of the Enterprise plan. Please contact ZEGOCLOUD Technical Support for activation before use.
- The Token generation method for community features is the same as other ZIM features; no additional permission declaration is required.

### Channel type description

| Type | Description | Join method | Visibility |
|------|------|----------|--------|
| GENERAL | Default channel, automatically created when a community is created. Community-level Tips and notifications are sent to this channel | Auto join | Visible to all members |
| PUBLIC | Public channel, needs to be created after the community is created | Auto join | Visible to all members |


<Note title="Description">
`GENERAL` and `PUBLIC` function identically; the difference lies only in the timing of creation: `GENERAL` is automatically created when the community is created, while `PUBLIC` needs to be created manually after the community.
</Note>

## Create a channel

After joining a community, call the {getPlatformData(props,createCommunityChannelMap)} API, passing in the channel basic information {getPlatformData(props,ZIMCommunityChannelInfoMap)} and creation configuration {getPlatformData(props,ZIMCommunityChannelCreateConfigMap)} to create a channel in the specified community.

<Note title="Description">

- Only community **owners** and **admins** have permission to create channels.
- When a community is created, the system automatically creates a default channel of type `GENERAL`. Developers can create additional public channels (`PUBLIC`) on this basis.
- The `channelID` supports custom values and only allows digits, English characters, and the following special characters: `! # $ % & ( ) + - : ; < = . > ? @ [ ] ^ _ { } | ~`, but it cannot start with `#`. If left empty, the ZIM server will create a channel with a channelID starting with `#C`.
- After successful creation, you are automatically in the channel and do not need to call the join API again.
</Note>

```java title="Example code"
import im.zego.zim.callback.ZIMCommunityChannelCreatedCallback;

import im.zego.zim.entity.ZIMCommunityChannelInfo;
import im.zego.zim.entity.ZIMError;
import im.zego.zim.enums.ZIMErrorCode;

ZIMCommunityChannelInfo channelInfo = new ZIMCommunityChannelInfo();
channelInfo.setChannelID("channel_001");    // Channel ID; can be customized
channelInfo.setChannelName("Announcement channel"); //Max 300 characters, configurable
channelInfo.setChannelAvatarUrl("https://example.com/channel_avatar.png"); //Max 100 characters, configurable
channelInfo.setCommunityID(communityID);    // Community ID

ZIMCommunityChannelCreateConfig config = new ZIMCommunityChannelCreateConfig();
config.setChannelNotice("This is Channel notice"); //Max 500 characters, configurable
HashMap<String, String> channelAttributes = new HashMap<>();
channelAttributes.put("type", "announcement");
config.setChannelAttributes(channelAttributes);

zim.createCommunityChannel(channelInfo, config, new ZIMCommunityChannelCreatedCallback() {
    @Override
    public void onCommunityChannelCreated(ZIMCommunityChannelFullInfo channelFullInfo, ZIMError errorInfo) {
        if (errorInfo.code == ZIMErrorCode.SUCCESS) {
            // Created successfully, channelFullInfo contains full channel info
            // Get the conversation ID for sending/receiving messages via channelFullInfo.conversationID
        }
    }
});
```

## Query channel information

Call the {getPlatformData(props,queryCommunityChannelsInfoMap)} API to batch query the complete information of specified channels, including channel name, avatar, notice, mute status, etc. Channel IDs that failed the query will be returned through `errorChannelIDs`.

:::if{props.platform=undefined}
<CodeGroup>
```java title="Example code"

channelIDs.add("channel_001");
channelIDs.add("channel_002");

zim.queryCommunityChannelsInfo(channelIDs, communityID,
```

## Query channel list

Call the {getPlatformData(props,queryCommunityChannelListMap)} API to retrieve all channels in a specified community with pagination. The `conversationID` field in the returned `ZIMCommunityChannel` object is the conversation ID needed for sending and receiving messages.

Pagination rules are the same as querying the community list: set `config.nextFlag` to `0` for the first query, pass the returned `nextFlag` to the next request, and continue until `0` is returned. The `count` range is 1-100, with a recommended value of 20.

:::if{props.platform=undefined}
<CodeGroup>
```java title="Example code"
ZIMCommunityChannelListQueryConfig config = new ZIMCommunityChannelListQueryConfig();
config.setNextFlag(0);

zim.queryCommunityChannelList(communityID, 100, config,
    new ZIMCommunityChannelListQueriedCallback() {
        @Override
        public void onCommunityChannelListQueried(String communityID, ArrayList<ZIMCommunityChannel> channelList, long nextFlag, ZIMError errorInfo) {
            if (errorInfo.code == ZIMErrorCode.SUCCESS) {
                for (ZIMCommunityChannel channel : channelList) {
                    // channel.getBaseInfo().getChannelID(): channel ID
                    // channel.getBaseInfo().getChannelName()：Channel name
                    // channel.getConversationID(): conversation ID for sending/receiving messages
                }
            }
        }
    });
```
</CodeGroup>

## Set channel mute

Call the {getPlatformData(props,muteCommunityChannelsMap)} API to batch set the mute status of specified channels. The mute configuration is passed through {getPlatformData(props,ZIMCommunityChannelMuteConfigMap)} .

`channelIDs` and `communityID` are direct parameters of the API method, not config fields.

| Config field | Description |
|---------|------|
| `mode` | Mute scope. Enum values: `NONE`(0) = no mute, `NORMAL`(1) = mute regular members, `ALL`(2) = mute all, `CUSTOM`(3) = custom role scope |
| `duration` | Mute duration (seconds); range 1-2592001 (approximately 30 days), or `-1` for permanent mute |
| `roles` | When `mode` is `CUSTOM`, specifies the list of muted roles |

Channel IDs that failed the operation will be returned through `errorChannelIDs`. For details, please refer to [Community mute](./community-mute.mdx)。

:::if{props.platform=undefined}
<CodeGroup>
```java title="Example code"
ArrayList<String> channelIDs = new ArrayList<>();
channelIDs.add("channel_001");
channelIDs.add("channel_002");

ZIMCommunityChannelMuteConfig config = new ZIMCommunityChannelMuteConfig();
config.setDuration(3600);  // Mute for 1 hour; -1 for permanent mute
config.setMode(ZIMCommunityChannelMuteMode.ALL);  // Mute all users

// isMute = true means muted, false means unmuted
zim.muteCommunityChannels(true, channelIDs, communityID, config,
    new ZIMCommunityChannelsMutedCallback() {
        @Override
        public void onCommunityChannelsMuted(String communityID, boolean isMute, ArrayList<String> errorChannelIDs, ZIMError errorInfo) {
            if (errorInfo.code == ZIMErrorCode.SUCCESS) {
                // MuteSet successfully
            }
        }
    });
```
</CodeGroup>

## Listen for channel changes

### Channel list changes

When a channel is created or dismissed in a community, causing the channel list to change, the SDK triggers the {getPlatformData(props,onCommunityChannelListChangedMap)} callback. `changeInfoList` contains the change type and channel information for each changed channel.

:::if{props.platform=undefined}
<CodeGroup>
```java title="Example code"
zim.setEventHandler(new ZIMEventHandler() {
    @Override
    public void onCommunityChannelListChanged(ZIM zim, ZIMCommunityChannelListChangedEventResult result) {
        String communityID = result.getCommunityID();
        for (ZIMCommunityChannelChangeInfo changeInfo : result.getChangeInfoList()) {
            // changeInfo.action: change type (create / disband)
            // changeInfo.channel：Changed channel information
        }
    }
});
```
</CodeGroup>
:::
