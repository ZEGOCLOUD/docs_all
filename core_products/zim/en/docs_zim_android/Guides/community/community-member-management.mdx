# Community member management

- - -

## Feature overview

ZIM SDK provides comprehensive community member management capabilities, supporting the following features:

- Invite users to join a community
- Kick community members
- Update member roles
- Transfer community ownership
- Query community member list
- Query specific member information
- Set member mute status
- Listen for member status and information changes

## Prerequisites

- Please refer to [Implement basic message sending and receiving](../Messaging/Send%20and%20receive%20messages.mdx) to complete ZIM SDK initialization and user login.
- Please refer to [Authenticate with Token](../Users/Authentication.mdx) to implement user authentication login.
- Community features require ZIM SDK 3.0.0 or later.
- Community features are part of the Enterprise plan. Please contact ZEGOCLOUD Technical Support for activation before use.
- The Token generation method for community features is the same as other ZIM features; no additional permission declaration is required.

## Invite users to join a community

Call the {getPlatformData(props,inviteUsersIntoCommunityMap)} API to batch invite users to join a community. Invitations do not require the invitee's consent; invited users will directly become community members. The invited users must already be registered in the system.
A single API call can invite up to 100 users to join a community.

User information of failed invitations will be returned through `errorUserList`.

:::if{props.platform=undefined}
<CodeGroup>
```java title="Example code"
import im.zego.zim.ZIM;
import im.zego.zim.callback.ZIMCommunityUsersInvitedCallback;
import im.zego.zim.entity.ZIMError;
import im.zego.zim.entity.ZIMErrorUserInfo;
import im.zego.zim.enums.ZIMErrorCode;

ArrayList<String> userIDs = new ArrayList<>();
userIDs.add("user_1");
userIDs.add("user_2");

zim.inviteUsersIntoCommunity(userIDs, communityID, new ZIMCommunityUsersInvitedCallback() {
    @Override
    public void onCommunityUsersInvited(String communityID, ArrayList<ZIMErrorUserInfo> errorUserList, ZIMError errorInfo) {
        if (errorInfo.code == ZIMErrorCode.SUCCESS) {
            // Invitation completed; errorUserList lists users for whom the invitation failed
        }
    }
});
```
</CodeGroup>
:::
:::if{props.platform="iOS|mac"}
<CodeGroup>
```objc title="Example code"

[zim inviteUsersIntoCommunity:userIDs communityID:communityID callback:^(NSString * _Nonnull communityID, NSArray<ZIMErrorUserInfo *> * _Nonnull errorUserList, ZIMError * _Nonnull errorInfo) {
```

## Query community member list

Pagination rules are the same as querying the community list: set `config.nextFlag` to `0` for the first query, pass the returned `nextFlag` to the next request, and continue until `nextFlag` returns `0`.

```java title="Example code"
zim.queryCommunityMemberList(communityID, 100, config, new ZIMCommunityMemberListQueriedCallback() {
    @Override
    public void onCommunityMemberListQueried(String communityID, ArrayList<ZIMCommunityMemberInfo> memberList, long nextFlag, ZIMError errorInfo) {
        if (errorInfo.code == ZIMErrorCode.SUCCESS) {

});
```

## Listen for member changes

### Member status changes

When members join or leave the community, the SDK triggers the {getPlatformData(props,onCommunityMemberStateChangedMap)} callback. You can use `updateInfo.getState()` to determine whether a member has entered (`ENTERED`) or exited (`EXITED`).

:::if{props.platform=undefined}
<CodeGroup>
```java title="Example code"
zim.setEventHandler(new ZIMEventHandler() {
    @Override
    public void onCommunityMemberStateChanged(ZIM zim, ZIMCommunityMemberStateChangedEventResult result) {
        for (ZIMCommunityMemberStateChangeInfo updateInfo : result.getUpdateInfoList()) {
            if (updateInfo.getState() == ZIMCommunityMemberChangeState.ENTERED) {
                // Member joined the community
            } else if (updateInfo.getState() == ZIMCommunityMemberChangeState.EXITED) {
                // Member left the community
            }
        }
    }
});
```
</CodeGroup>
:::
:::if{props.platform="iOS|mac"}
<CodeGroup>
```objc title="Example code"
- (void)zim:(ZIM *)zim communityMemberStateChangedWithResult:(ZIMCommunityMemberStateChangedEventResult *)result {
```

### Member information update

When a community member's nickname, role, mute status, or other information changes, the SDK triggers the {getPlatformData(props,onCommunityMemberInfoUpdatedMap)} callback.

```java title="Example code"
            ZIMCommunityMemberInfo memberInfo = updateInfo.getMemberInfo();
```
