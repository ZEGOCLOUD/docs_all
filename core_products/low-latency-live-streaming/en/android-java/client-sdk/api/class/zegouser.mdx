# ZegoUser

## ZegoUser

User object.

**Details**

Configure user ID and username to identify users in the room.
  that the userID must be unique under the same appID, otherwise, there will be mutual kicks when logging in to the room.
 It is strongly recommended that userID corresponds to the user ID of the business APP, that is, a userID and a real user are fixed and unique, and should not be passed to the SDK in a random userID. Because the unique and fixed userID allows ZEGO technicians to quickly locate online problems.

*Declared in `entity/ZegoUser.java`*
