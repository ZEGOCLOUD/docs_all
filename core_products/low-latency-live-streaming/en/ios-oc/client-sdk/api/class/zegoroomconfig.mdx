# ZegoRoomConfig

## ZegoRoomConfig

Advanced room configuration.

**Details**

Configure maximum number of users in the room and authentication token, etc.

*Declared in `ZegoExpressDefines.h`*

### Properties
<ParamField
  name="maxMemberCount"
  prototype="nonatomic, assign unsigned int maxMemberCount"
  parent_name="ZegoRoomConfig"
  parent_type="class">
The maximum number of users in the room, Passing 0 means unlimited, the default is unlimited.
</ParamField>
<ParamField
  name="isUserStatusNotify"
  prototype="nonatomic, assign BOOL isUserStatusNotify"
  parent_name="ZegoRoomConfig"
  parent_type="class">
Whether to enable the user in and out of the room callback notification [onRoomUserUpdate], the default is off. If developers need to use ZEGO Room user notifications, make sure that each user who login sets this flag to true
</ParamField>
<ParamField
  name="token"
  prototype="nonatomic, copy NSString * token"
  parent_name="ZegoRoomConfig"
  parent_type="class">
The token issued by the developer's business server is used to ensure security. For the generation rules, please refer to [Using Token Authentication](https://doc-zh.zego.im/article/10360), the default is an empty string, that is, no authentication. In versions 2.17.0 and above, if appSign is not passed in when calling the [createEngine] API to create an engine, or if appSign is empty, this parameter must be set for authentication when logging in to a room.
</ParamField>
<ParamField
  name="capabilityNegotiationTypes"
  prototype="nonatomic, assign unsigned int capabilityNegotiationTypes"
  parent_name="ZegoRoomConfig"
  parent_type="class">
The bitmask marker for capability negotiation, refer to enum [ZegoRoomCapabilityNegotiationTypesBitMask], when this param converted to binary, 0b01 that means 1 &lt;&lt; 0 for enable the capability negotiation of all user in the room, 0x10 that means 1 &lt;&lt; 1 for enable the capability negotiation of publisher in the room. The masks can be combined to allow different types of capability negotiation.
</ParamField>
<ParamField
  name="roomType"
  prototype="nonatomic, assign unsigned int roomType"
  parent_name="ZegoRoomConfig"
  parent_type="class">
The type of the room, generally, it can be ignored and set to 0.
</ParamField>
