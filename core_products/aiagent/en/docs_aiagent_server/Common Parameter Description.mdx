# Common Parameter Description

---

## LLM

| Parameter       | Type            | Required | Description                                                                                                                                                  |
|-----------------|-----------------|----------|--------------------------------------------------------------------------------------------------------------------------------------------------------------|
| Url             | String          | Yes      | The endpoint that receives requests (can be your own service or a service provided by any LLM service provider), and must be compatible with the [OpenAI Chat Completions API](https://platform.openai.com/docs/api-reference/chat).<br/>For example: https://api.openai.com/v1/chat/completions                                                                  |
| ApiKey          | String          | No       | The parameter used by the LLM service provider for authentication. Default is empty, but it must be provided in a production environment.             |
| Model           | String          | Yes      | The LLM model. Different LLM service providers support different models, please refer to their official documentation to choose an appropriate model. |
| SystemPrompt    | String          | No       | The system prompt for the intelligent agent. Predefined information attached at the beginning when calling the LLM, used to control the output of the LLM. It can include role settings, prompts, and response examples.   |
| Temperature     | Float           | No       | Higher values will make the output more random, while lower values will make the output more focused and deterministic.    |
| TopP            | Float           | No       | Sampling method, smaller values result in stronger determinism; larger values result in more randomness.    |
| Params          | Object          | No       | Other parameters supported by the LLM service provider, such as maximum Token number limit, etc. Different LLM providers support different parameters, please refer to their official documentation and fill in as needed. |

```json title="Params Example"
"Params": {
    "max_tokens": 1024
}
```

## TTS
| Parameter | Type | Required | Description   |
| ----------------------- | --------------- | -------- | ----- |
| Vendor                    | String          | Yes      | Text-to-speech (TTS) service provider. Options:<ul><li>`Aliyun`: Alibaba Cloud.</li><li>`Bytedance`: Volcano Engine.</li><li>`Minimax`: MiniMax.</li></ul> |
| Params         | Object          | No       | TTS configuration parameters, formatted as a JSON object. The structure of this parameter varies depending on the TTS provider, for more details refer to the following:<ul><li>Alibaba Cloud: [Alibaba Cloud Speech Synthesis Interface Documentation_Intelligent Speech Interaction(ISI)-Alibaba Cloud Help Center](https://help.aliyun.com/zh/isi/developer-reference/overview-of-speech-synthesis?spm=a2c4g.11186623.help-menu-30413.d_3_1_0_0.9e5b1f01YvXYPa&scm=20140722.H_84435._.OR_help-T_cn~zh-V_1#section-ljp-l5z-efu)</li><li>Volcano Engine: [Large Model Speech Synthesis API--Speech Technology-Volcano Engine](https://www.volcengine.com/docs/6561/1257584#%E8%AF%B7%E6%B1%82%E5%8F%82%E6%95%B0)</li><li>MiniMax: [MiniMax - Speech Model - T2A v2 (Speech Generation) - HTTP](https://platform.minimaxi.com/document/T2A%20V2?key=66719005a427f0c8a5701643#YqSh1KAoyms1WH4XJrdeIrrb)</li></ul> |
| FilterText       | Array of Object | No       | <p>Filters the text within specified punctuation marks from the content returned by LLM before performing speech synthesis. For the structure of this parameter, please refer to [FilterText](#filtertext).</p><Note title="Note"><ul><li>You need to define in [LLM > SystemPrompt](#LLM) which content should be placed within the specified punctuation marks.</li><li>This parameter cannot be updated when [modifying an intelligent agent instance](./Agent%20Instance%20Management/Update%20Agent%20instance.mdx).</li></ul></Note> |

### Params Example Code

<Note title="Note">
- Parameters under app are related to TTS interface authentication and vary for each provider.
- Parameters other than app will be directly passed through to third-party TTS.
</Note>

<CodeGroup>
```json title="For Alibaba Cloud"

    "Vendor": "Aliyun",
    "Params": {
        "app":{
            "appid": "your_appid",
            "app_secret": "aaaafdsfsdf",
            "app_secret2": "fdsfdsrwrxdsf"
        },
        "voice": "zhitian_emo",
        "volume": 50,
        "speech_rate": 0,
        "pitch_rate": 0
    }
}
```
```json title="Applicable to Volcano Engine"

    "Vendor": "Bytedance",
    "Params": {
        "app": {
            "appid": "your_appid",
            "token": "your_token",
            "cluster": "volcano_tts"
        },
        "audio": {
            "voice_type": "your_voice_type",
            "emotion": "your_emotion",
            "enable_emotion": true,
            "speed_ratio": 1.0
        }
    }
}
```
```json title="Applicable to MiniMax"

```
</CodeGroup>

### FilterText

| Parameter            | Type   | Required | Description   |
|---------------------|--------|----------|---------------|
| BeginCharacters     | String | Yes      | Start punctuation for filtering text. For example, if you want to filter out the content in `()`, set this to `(`. |
| EndCharacters       | String | Yes      | End punctuation for filtering text. For example, if you want to filter out the content in `()`, set this to `)`. |

```json
"FilterText": [
    {
        "BeginCharacters": "(",
        "EndCharacters": ")"
    },
    {
        "BeginCharacters": "（",
        "EndCharacters": "）"
    }
]
```

## ASR

| Parameter | Type | Required | Description |
|-----------|------|----------|-------------|
| HotWord   | String | No | A hotword list used to improve recognition accuracy. Format: `Hotword1|Weight1,Hotword2|Weight2,Hotword3|Weight3`.<p />Each hotword should not exceed 30 characters (up to 10 Chinese characters), no spaces allowed, weight range: [-1, 11].<p />Supports up to 128 hotwords.<Note title="Note">When the weight is 11, it indicates that the word is a super hotword. It is recommended to set only important and necessary hotwords to 11, as too many hotwords with a weight of 11 may affect recognition performance.</Note>|
| Params    | Object | No | Extended parameters, please contact ZEGO Technical Support for details. |

```json
"ASR": {
    "HotWord": "History|10, Physics|10, Sports|10"
}
```
