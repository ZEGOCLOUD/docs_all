# Controlling AI Agent Speech Emotion

## Scenario Description

In real-time voice interaction scenarios with AI, the default TTS voice is often flat and lacks emotional variation. By controlling parameters such as emotion and tone during TTS synthesis, you can make the AI's voice more expressive and immersive.

For example, in a customer service scenario, the AI can automatically switch to a warm, patient tone based on the user's questions; in a gaming companion scenario, the AI can express excitement, surprise, and other emotions as the conversation progresses. These capabilities are controlled through TTS synthesis parameters.

There are two main control approaches:
- **Specific emotion enums**: Control voice emotion through predefined emotion tags (such as `happy`, `sad`, `angry`). This approach uses emotion lists defined by the TTS vendor, and developers select from them.  
<span style={{color: '#999', fontStyle: 'italic'}}>For example, MiniMax's Speech series supports specifying emotions such as happy, sad, and angry for speech synthesis.</span>

- **Natural language descriptions**: Control voice style through natural language instructions (such as "Speak in a gentle tone"). This approach is more flexible and can describe subtler emotional variations.  
<span style={{color: '#999', fontStyle: 'italic'}}>For example, Volcengine Doubao TTS 2.0 allows entering instructions like "Speak in a gentle tone" to control the synthesis effect.</span>

## Feature Overview

### Supported Models and Capabilities

ZEGO AI Agent currently supports the following TTS vendors' emotion control capabilities. Different vendors and models support different control methods: MiniMax and Volcengine Doubao TTS 1.0 support specifying predefined emotion enums (such as `happy`, `sad`, `angry`) through the `emotion` tag; Volcengine Doubao TTS 2.0 and CosyVoice support describing the desired voice style through natural language instructions (such as `instruction` or `context_texts`).

## How to Use

### Prerequisites

1. Enable AI Agent service.
2. Confirm that the TTS model or timbre used supports the corresponding synthesis parameters (see [Supported Models and Capabilities](#supported-models-and-capabilities)).
3. ZEGO AI Agent service supports the corresponding TTS vendor (see [Supported Vendors and Parameters](#supported-vendors-and-parameters)).

### Supported Vendors and Parameters

The table below lists the pass-through parameters supported by each TTS vendor. The "TTS Parameter Path" column contains the `value` (dot-notation path of the vendor request body) needed for the `TTSParamPaths` mapping table in [Step 2: Configure TTSParamPaths Mapping](#configure-ttsparampaths-mapping), while the short parameter names used in tags correspond to the `key` of `TTSParamPaths`.

### Usage Steps

<Steps>
<Step title="Configure LLMMetaInfo Extraction Rules"><span id="configure-llmmetainfo-extraction-rules"></span>

Configure the `AdvancedConfig.LLMMetaInfo` parameter of the [Create Agent Instance](/aiagent-server/api-reference/agent-instance-management/create-agent-instance) API to specify how to extract metadata from LLM text. For example:
```json
"AdvancedConfig": {
  "LLMMetaInfo": {
    "BeginCharacters": "[[",
    "EndCharacters": "]]"
  }
}
```
<Note title="Note">No need to configure the FilterText.BeginCharacters and FilterText.EndCharacters parameters again when registering the AI agent or creating the AI agent instance, because the metadata and marker symbols will be removed from the LLM output text.</Note>
</Step>
<Step title="Configure TTSParamPaths Mapping"><span id="configure-ttsparampaths-mapping"></span>

`TTSParamPaths` is a mapping table passed through `AdvancedConfig` when creating an agent instance, where the `key` is the short parameter name used in tags, and the `value` is the dot-notation path of the vendor request body (i.e., the "TTS Parameter Path" column in the [Supported Vendors and Parameters](#supported-vendors-and-parameters) table above).

For example, for Volcengine TTS (Unidirectional Streaming), configure as follows when creating an agent instance:

```json
"AdvancedConfig": {
  "TTSParamPaths": {
    "context_texts": "req_params.additions.context_texts",
    "pitch": "req_params.additions.post_process.pitch"
  }
}
```

After configuration, simply use short parameter names in the tags, and the system will automatically expand and merge them into the vendor request body according to the paths.

<Note title="Note">Parameters not configured in `TTSParamPaths` are silently ignored by the TTS layer, without affecting other parameters and normal synthesis. Parameter validity is verified by the vendor API, and errors are returned by the vendor.</Note>
</Step>
<Step title="Have the LLM Output Content in the Specified Format"><span id="have-the-llm-output-content-in-the-specified-format"></span>

In the `LLM.SystemPrompt` of the [Register Agent](/aiagent-server/api-reference/agent-configuration-management/register-agent) or [Create Agent Instance](/aiagent-server/api-reference/agent-instance-management/create-agent-instance) API, prompt the LLM to output tags containing native vendor parameters in its responses.

<Tabs>
<Tab title="CosyVoice (instruction)">
```markdown
# Role
You are an intelligent voice assistant capable of dynamically adjusting your voice tone based on the scenario.

## Format Requirements
- You can insert `[[{...}]]` tags at the beginning of sentences to control TTS synthesis parameters.
- Supported parameters:
  - "instruction" (optional): Natural language instruction, such as "Speak in a gentle tone."

## Example
- [[{"instruction":"Speak in a gentle tone"}]] Hello, I'm happy to serve you.
```
</Tab>
<Tab title="MiniMax (emotion)">
```markdown
# Role
You are a smart voice assistant that can dynamically adjust the reply tone based on the user's mood and add emotional tags and speed control when necessary.

## Format Requirements
- **LLMMetaInfo** is a JSON string for emotion and speed control, **must be strictly wrapped in [[ and ]]**.
- The JSON key-value pair syntax must be complete, and the following keys are supported (one or more can be selected):
  - "emotion" (optional): emotion type, limited to ["happy","sad","angry","fearful","surprised","neutral"].
- **LLMMetaInfo must be placed at the beginning of the corresponding sentence**, and cannot appear in the middle or at the end of the sentence.
- If the emotions or speeds of adjacent sentences are different, a new LLMMetaInfo must be inserted at the beginning of the changing sentence.

## Example
- User: Today's weather is good.
  Assistant:
  [[{"emotion":"happy"}]] Today is indeed a good day.
  [[{"emotion":"sad"}]] The regret of yesterday still lingers in my heart, but I just want to enjoy the sunshine now.
```
</Tab>
<Tab title="Doubao TTS 1.0 (emotion)">
```markdown
# Role
You are a smart voice assistant that can dynamically adjust the reply tone based on the user's mood and add emotional tags and speed control when necessary.

## Format Requirements
- **LLMMetaInfo** is a JSON string for emotion and speed control, **must be strictly wrapped in [[ and ]]**.
- The JSON key-value pair syntax must be complete, and the following keys are supported (one or more can be selected):
  - "emotion" (optional): emotion type, limited to ["happy","sad","angry","fearful","surprised","neutral"].
  - "emotion_scale" (optional): emotion intensity, a float number between 1 and 5 (e.g. 4); the intensity is non-linearly increasing, and 3 and 5 may have similar perception.
- **LLMMetaInfo must be placed at the beginning of the corresponding sentence**, and cannot appear in the middle or at the end of the sentence.
- If the emotions or speeds of adjacent sentences are different, a new LLMMetaInfo must be inserted at the beginning of the changing sentence.

## Content Requirements
- The emotion and speed adjustments must be highly consistent with the user's current emotion and dialogue context.

## Example
- User: Today's weather is good.
  Assistant:
  [[{"emotion":"happy","emotion_scale":3}]] Today is indeed a good day.
  [[{"emotion":"sad","emotion_scale":4}]] The regret of yesterday still lingers in my heart, but I just want to enjoy the sunshine now.
```
</Tab>

<Step title="Direct TTS API Calls (Optional)"><span id="direct-tts-api-calls-optional"></span>

In addition to automatic tag output through the LLM, you can also directly include tags at the very beginning of the `text` field when calling APIs such as [sendAgentInstanceTTS](/aiagent-server/api-reference/agent-instance-control/send-agent-instance-tts). The tag content will not be read aloud:

```json
{
 "text": "[[{\"context_texts\":[\"Speak in a happy tone\"],\"pitch\":5}]]Today's weather is really nice"
}
```

<Note title="Note">The tag must be placed at the very beginning of the text.</Note>
</Step>
</Steps>

### Special Notes

Customers are now responsible for managing the validity of their input parameters. For `emotion`, if the LLM generates or `sendAgentInstanceTTS` passes an unsupported emotion value, the generated TTS may have no audio output.
