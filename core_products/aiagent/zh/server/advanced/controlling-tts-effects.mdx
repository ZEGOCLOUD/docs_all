# 控制智能体语音情绪

## 场景说明

在 AI 实时语音互动场景中，默认的 TTS 语音往往较为平淡，缺乏情感变化。通过控制 TTS 合成时的情绪、语调等参数，可以让 AI 的声音更富有表现力和沉浸感。

例如，在客服场景中，AI 可以根据用户的问题自动切换为温和、耐心的语气；在游戏陪玩场景中，AI 可以随着对话内容表现出兴奋、惊讶等情绪。这些能力都需要通过 TTS 合成参数来控制。

目前主要有两种控制方式：
- **特定情绪枚举**：通过预定义的情绪标签（如 `happy`、`sad`、`angry`）控制语音情绪。这种方式由 TTS 厂商定义可用的情绪列表，开发者从中选择。  
<span style={{color: '#999', fontStyle: 'italic'}}>例如，MiniMax 的 Speech 系列支持高兴、悲伤、愤怒等指定情绪进行语音合成。</span>

- **自然语言描述**：通过自然语言指令（如"用温柔的语气说话"）控制语音风格。这种方式更加灵活，可以描述更细腻的情感变化。  
<span style={{color: '#999', fontStyle: 'italic'}}>例如火山豆包语音2.0，可以填写如"用温柔的语气说话"的指令来控制合成效果。</span>

## 功能简介

### 支持的模型及能力

目前 ZEGO AI Agent 支持以下 TTS 厂商的情绪控制能力。不同厂商和模型支持的控制方式不同：MiniMax 和火山豆包语音 1.0 支持通过 `emotion` 标签指定预定义的情绪枚举（如 `happy`、`sad`、`angry` 等）；火山豆包语音 2.0 和 CosyVoice 支持通过自然语言指令（如 `instruction` 或 `context_texts`）描述想要的语音风格。

|厂商|TTS 模型|支持的类型|体验方式|
|---|---|---|---|
|MiniMax|Speech 系列|支持特定情绪枚举（emotion 标签），如 `happy`、`sad`、`angry`、`fearful`、`surprised`、`neutral` 等。[查看完整 emotion 列表](https://platform.minimaxi.com/docs/api-reference/speech-t2a-websocket)|[语音调试台](https://platform.minimaxi.com/examination-center/voice-experience-center/t2a_v2)|
|火山引擎（豆包语音 2.0）|2.0 系列|支持自然语言描述（context_texts 标签），如填写"用温柔的语气说话"、"用兴奋的语调说话"等指令控制语音风格。[查看语音指令与标签说明](https://www.volcengine.com/docs/6561/1871062?lang=zh)|[豆包语音合成大模型](https://www.volcengine.com/product/tts)中的多情感音色|
|火山引擎（豆包语音 1.0）|1.0 系列|支持特定情绪枚举（emotion 标签），如 `happy`、`sad`、`angry` 等，同时支持 `emotion_scale` 控制情绪强度。[查看情感参数说明](https://www.volcengine.com/docs/6561/1257544?lang=zh#%E6%83%85%E6%84%9F%E5%8F%82%E6%95%B0%EF%BC%88emotion%EF%BC%89%EF%BC%9A)|[豆包语音合成大模型](https://www.volcengine.com/product/tts)中的多情感音色|
|阿里云（CosyVoice）|CosyVoice|支持自然语言描述（instruction 标签），如填写"用温柔的语气说话"、"用四川方言说话"等指令控制方言、情感、角色等。[查看指令控制说明](https://help.aliyun.com/zh/model-studio/realtime-tts-user-guide?spm=a2c4g.11186623.help-menu-2400256.d_0_3_5_0.19d978d79n2Br4#12884a10929p9)|[CosyVoice 体验](https://help.aliyun.com/zh/model-studio/cosyvoice-voice-list)|

## 使用方式

### 前提条件

1. 开通 AI Agent 服务。
2. 确认所使用的 TTS 模型或音色支持对应的合成参数（详细见[支持的模型及能力](#支持的模型及能力)）。
3. ZEGO AI Agent 服务支持对应的 TTS 厂商（详细见[支持的厂商与参数](#支持的厂商与参数)）。

### 支持的厂商与参数

下表列出了各 TTS 厂商支持透传的参数及其详情。其中"TTS 参数路径"列的值即[步骤 2：配置 TTSParamPaths 映射](#配置-ttsparampaths-映射)中 `TTSParamPaths` 映射表所需填写的 `value`（厂商请求体的点号路径），而标签中使用的短参数名则对应 `TTSParamPaths` 的 `key`。

|厂商（TTS vendor）|参数 / 类型 / 适用说明|TTS 参数路径|LLM 流式输出|`sendAgentInstanceTTS`|
|---|---|---|---|---|
|火山引擎 TTS（单向流式）<br />`ByteDanceV3`|`emotion` (string)<br />特定情绪枚举，仅 1.0 系列支持|`req_params.audio_params.emotion`|✅|✅|
|火山引擎 TTS（单向流式）<br />`ByteDanceV3`|`emotion_scale` (number)<br />情绪强度 1-5，仅 1.0 系列支持|`req_params.audio_params.emotion_scale`|✅|✅|
|火山引擎 TTS（单向流式）<br />`ByteDanceV3`|`context_texts` (string[])<br />自然语言指令，仅 2.0 系列支持|`req_params.additions.context_texts`|✅|✅|
|火山引擎 TTS（单向流式）<br />`ByteDanceV3`|`pitch` (int)<br />音调调整，全部系列支持|`req_params.additions.post_process.pitch`|✅|✅|
|火山引擎 TTS（双向流式）<br />`ByteDanceFlowing`|`context_texts` (string[])<br />自然语言指令|`req_params.additions.context_texts`|❌|✅|
|阿里云 CosyVoice<br />`CosyVoice`|`instruction` (string)<br />自然语言指令|`payload.parameters.instruction`|✅|✅|
|MiniMax<br />`MiniMax`|`emotion` (string)<br />特定情绪枚举，如 `happy`、`sad`、`angry`、`fearful`、`surprised`、`neutral` 等|[查看完整 emotion 列表](https://platform.minimaxi.com/docs/api-reference/speech-t2a-websocket)|✅|❌|

<Warning title="注意">火山引擎 TTS 双向流式不支持 LLM 流式输出场景：LLM 成句后插入标签可能造成断句。如需在双向流式下使用，请通过直接调用 TTS 接口（如 [sendAgentInstanceTTS](/aiagent-server/api-reference/agent-instance-control/send-agent-instance-tts)）传入标签。</Warning>

### 使用步骤

<Steps>
<Step title="配置 LLMMetaInfo 提取规则"><span id="配置-llmmetainfo-提取规则"></span>

通过配置[创建智能体实例](/aiagent-server/api-reference/agent-instance-management/create-agent-instance)接口的 `AdvancedConfig.LLMMetaInfo` 参数，指定如何从 LLM 文本中提取元数据。例如：
```json
"AdvancedConfig": {
  "LLMMetaInfo": {
    "BeginCharacters": "[[",
    "EndCharacters": "]]"
  }
}
```
<Note title="说明">无需在注册智能体或创建智能体实例时再配置 FilterText.BeginCharacters 和 FilterText.EndCharacters 参数，因为元数据及标记符号将会被从 LLM 输出文本中移除。</Note>
</Step>
<Step title="配置 TTSParamPaths 映射"><span id="配置-ttsparampaths-映射"></span>

`TTSParamPaths` 是创建智能体实例时通过 `AdvancedConfig` 传入的映射表，`key` 为标签中使用的短参数名，`value` 为厂商请求体的点号路径（即上方[支持的厂商与参数](#支持的厂商与参数)表格中的"TTS 参数路径"列）。

以火山引擎 TTS（单向流式）为例，在创建智能体实例时配置如下：

```json
"AdvancedConfig": {
  "TTSParamPaths": {
    "context_texts": "req_params.additions.context_texts",
    "pitch": "req_params.additions.post_process.pitch"
  }
}
```

配置完成后，在标签中直接使用短参数名即可，系统会自动按路径展开并合并到厂商请求体中发送。

<Note title="说明">未在 `TTSParamPaths` 中配置的参数会被 TTS 层静默忽略，不影响其他参数与正常合成。参数有效性由厂商 API 自行校验，错误由厂商返回。</Note>
</Step>
<Step title="让 LLM 按照指定格式输出内容"><span id="让-llm-按照指定格式输出内容"></span>

在 [注册智能体](/aiagent-server/api-reference/agent-configuration-management/register-agent) 或 [创建智能体实例](/aiagent-server/api-reference/agent-instance-management/create-agent-instance)接口的 `LLM.SystemPrompt` 中提示 LLM 在回答中输出包含厂商原生参数的标签。

```markdown
## 格式要求
- 回答中可在句首插入 `[[{...}]]` 标签控制语音合成参数。
- 支持的参数：
  - "instruction"（可选）：自然语言指令，如"用温柔的语气说话"。

## 示例
```
```markdown
- 用户：今天天气不错。
  助手：

```

</Step>
<Step title="直接调用 TTS 接口（可选）"><span id="直接调用-tts-接口可选"></span>

除了通过 LLM 自动输出标签外，您也可以在调用 [sendAgentInstanceTTS](/aiagent-server/api-reference/agent-instance-control/send-agent-instance-tts) 等接口时，直接在 `text` 字段最前面携带标签，标签内容不会被朗读：

```json
{
 "text": "[[{\"context_texts\":[\"用开心的语气说话\"],\"pitch\":5}]]今天天气真好"
}
```

<Note title="说明">标签必须置于文本最前面（句首）。</Note>
</Step>
</Steps>

### 特殊说明

现在由客户自己管理传入参数的有效性。对于 `emotion`，如果大模型生成或 sendAgentInstanceTTS 调用传入了不支持的 emotion 值，可能导致生成的 TTS 没有声音。

<Accordion title="过时方案：通过 LLM SystemPrompt 指定情绪（emotion 标签）" defaultOpen="false">

<Note title="说明">以下方案为早期实现方式，推荐使用上述"使用步骤"中基于 TTSParamPaths 透传厂商原生参数的方式。</Note>

要实现该能力主要分为三步：
1. 指定 LLM 文本中控制情绪的内容格式。
2. 让 LLM 按照指定的控制情绪格式输出内容。
3. 让 TTS 厂商根据情绪控制参数合成带情绪的语音（ZEGO AI Agent 自动处理）。

<Note title="说明">
- 音色/情绪列表可能会变更，请以 TTS 厂商提供的最新列表为准。
- 元数据中 JSON 的 key 只能是 emotion 或者 emotion_scale，value 则必须与 TTS 厂商支持的情绪参数值完全相同。
</Note>

<Steps>
<Step title="指定 LLM 文本中控制情绪的内容格式">

<Note title="说明">无需在注册智能体或创建智能体实例时再配置 FilterText.BeginCharacters 和 FilterText.EndCharacters 参数，因为元数据及标记符号将会被从 LLM 输出文本中移除。</Note>

通过配置[创建智能体实例](/aiagent-server/api-reference/agent-instance-management/create-agent-instance)接口的 AdvancedConfig.LLMMetaInfo 参数，指定如何从 LLM 文本中提取控制情绪的元数据。例如：
```json
"LLMMetaInfo" : {
    "BeginCharacters": "[[",
    "EndCharacters": "]]"
}
```
</Step>
<Step title="让 LLM 按照指定的控制情绪格式输出内容">

<Note title="说明">以下示例中，提示的 `emotion` 选值仅仅是示例，实际 TTS 厂商支持哪些情绪就可以让 LLM 输出的文本中包含哪些情绪。但是一般都不会包含所有情绪（比如一个人工客服应用不会让它有悲伤情绪）。</Note>

以下是使用 MiniMax 和豆包语音 TTS 时，调用 [注册智能体](/aiagent-server/api-reference/agent-configuration-management/register-agent) 和 [创建智能体实例](/aiagent-server/api-reference/agent-instance-management/create-agent-instance)接口对应的 `LLM.SystemPrompt` 示例，仅供参考，请根据实际需求调整：
<Tabs>
<Tab title="MiniMax">
```markdown
# 角色
您是一位智能语音助手，能够根据用户情绪动态调整回复语气，并在必要时为每句添加情感标签与语速控制。

## 格式要求
- **LLMMetaInfo** 为情感与语速控制的 JSON 字符串，**必须严格使用 [[ 和 ]] 包裹**。
- JSON 键值对语法需完整，支持以下键（可任选其一或多个）：
  - "emotion"（可选）：情绪类型，取值限于 ["happy","sad","angry","fearful","surprised","neutral"]。
- **LLMMetaInfo 必须置于对应句子的最开头**，不得出现在句中或句尾。
- 若相邻句子情绪或语速不同，需在变化句开头插入新的 LLMMetaInfo。

## 示例
- 用户：今天天气不错。
  助手：
  [[{"emotion":"happy"}]]今天确实是个好日子。
  [[{"emotion":"sad"}]]昨天的遗憾虽仍在心头，但此刻只想好好享受阳光。
```
</Tab>
<Tab title="豆包语音(单向流式)">
```markdown
# 角色
您是一位智能语音助手，能够根据用户情绪动态调整回复语气，并在必要时为每句添加情感标签与语速控制。

## 格式要求
- **LLMMetaInfo** 为情感与语速控制的 JSON 字符串，**必须严格使用 [[ 和 ]] 包裹**。
- JSON 键值对语法需完整，支持以下键（可任选其一或多个）：
  - "emotion"（可选）：情绪类型，取值限于 ["happy","sad","angry","fearful","surprised","neutral","hate","excited","coldness","tender"]。
  - "emotion_scale"（可选）：情绪强度，范围 1–5 的浮点数（如 4）；强度非线性递增，3 与 5 可能感知相近。
- **LLMMetaInfo 必须置于对应句子的最开头**，不得出现在句中或句尾。
- 若相邻句子情绪或语速不同，需在变化句开头插入新的 LLMMetaInfo。

## 内容要求
- 情感与语速调整须与用户当前情绪及对话语境高度一致。

## 示例
- 用户：今天天气不错。
  助手：
  [[{"emotion":"happy","emotion_scale":3}]]今天确实是个好日子。
  [[{"emotion":"sad","emotion_scale":4}]]昨天的遗憾虽仍在心头，但此刻只想好好享受阳光。
```
