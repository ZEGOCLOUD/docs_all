## Prepare Environment

Before running the sample source code, please ensure your development environment meets the following requirements:

* Android Studio 2021.1.1 or above
* Gradle 8.13
* Android Gradle Plugin 8.12.3
* Android 8.0 (API 26) or above, with an Android device or emulator that supports audio/video (real device recommended)
* Android device and development computer connected to the Internet

<Note title="Description">

- When compiling the sample source code for the first time, if there are no corresponding dependency libraries or build tools, Android Studio will automatically download the dependencies, which may take some time. Please wait patiently.
- It is recommended to download specific Gradle versions in advance. Please refer to [Gradle Services](https://services.gradle.org) for downloads.
- The Gradle related versions used in the sample source code can be viewed in the `gradle/wrapper/gradle-wrapper.properties` and `gradle/libs.versions.toml` files.
- For version dependencies between Android Gradle Plugin, Gradle, and SDK Tool, please refer to: [Android Gradle Plugin Release Notes](https://developer.android.com/studio/releases/gradle-plugin)

</Note>

## Run Sample Source Code

<Steps>
<Step title="Clone Source Code">

Open a terminal and execute the following command to clone the repository locally:

```bash
git clone https://github.com/ZEGOCLOUD/digital_human_paas_quick_start.git
```

</Step>
<Step title="Configure Server URL">

Open the `app/src/main/java/com/example/zegodigitalhumanquickstart/ZegoQuickStartConstants.java` file and modify `DEFAULT_SERVER_URL` to your business server API address.

```java title="ZegoQuickStartConstants.java"
public static final String DEFAULT_SERVER_URL = "http://your-server-address/api";
```

<Warning title="Note">

Please ensure the server address is accessible, otherwise the sample source code will not run properly.

</Warning>

</Step>
<Step title="Configure Permissions">

The sample source code has declared the required permissions (network, camera, microphone, etc.) in `AndroidManifest.xml`. When running the application for the first time, it will automatically request runtime permissions. Please allow the relevant permissions.

</Step>
<Step title="Run Project">

1. Open the `Android` directory in Android Studio
2. Connect an Android device or start an emulator
3. Click the **Run** button in Android Studio to install the application on the device

</Step>
</Steps>

## Experience Digital Human Features

Run the project on a real device. After successful execution, you can see the digital human interface. The sample application provides the following features:

1. **Create Task**: Click the "Create Task" button to create a digital human stream task
2. **Drive Digital Human**:
   - **Text Drive**: Input text content to drive the digital human to speak the text
   - **Audio Drive**: Upload or record audio to drive the digital human to speak the audio content
   - **WebSocket Drive**: Drive the digital human in real-time via WebSocket protocol (TTS audio stream), suitable for real-time conversation scenarios
3. **Task Management**: You can stop the current task, interrupt the drive, destroy all tasks, etc.

<Note title="Description">

- When running the application for the first time, you need to grant camera and microphone permissions, otherwise some features may not work properly.
- Ensure the server is correctly deployed and running, otherwise you cannot create tasks and drive the digital human.
- Digital human resources will be automatically preloaded on first use, please wait patiently.

</Note>

## FAQ

**1. When running the sample source code, it shows "Network Error" or "Server Connection Failed". How to handle it?**

Please troubleshoot with the following steps:
- Check if the server URL configured in `ZegoQuickStartConstants.java` is correct
- Ensure the server is correctly deployed and running. You can test if the API is accessible via browser or Postman
- Ensure the Android device and server are on the same network. If testing locally, make sure the device and computer are on the same LAN
- Check if network permissions are declared in AndroidManifest.xml

**2. Task creation failed with "AppID Not Set" or "Token Empty". How to handle it?**

The sample application automatically gets AppID and Token from the server API without manual configuration. Please check:
- Ensure the server API correctly returns task configuration information
- Use Android Studio's Logcat to view network request logs and confirm the data format returned by the server is correct

**3. Digital human cannot display or shows black screen. How to handle it?**

Please troubleshoot with the following steps:
- Ensure the task has been successfully created and logged into the [Video Call Room](/real-time-video-android-java/quick-start/implementing-video-call#createroom)
- Ensure the [Base64Config](/aigc-digital-human-android/quick-start) configuration returned by the server is correct
- Filter by `Zego` tag in Logcat to view SDK logs and confirm if the digital human SDK started normally
