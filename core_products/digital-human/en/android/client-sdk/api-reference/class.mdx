| boolean | Whether the configuration was successfully set, true for success, false for failure. |

**Details**

Set digital human log configuration, customizing log storage path, single file size, and rotation count.

<Warning title="">
- **Note**: This method must be called before calling [create] or [ZegoDigitalHumanResource] methods.
</Warning>

<Note title="">
- **Example**: If `logDirectoryPath` is set to "files/" and `logCount` is set to 3, the system will generate rotating log files: "files/ZaLog1.txt", "files/ZaLog2.txt", and "files/ZaLog3.txt". When "ZaLog1.txt" is full, it will be renamed to "ZaLog2.txt", and a new "ZaLog1.txt" will be created. This process continues, always maintaining a maximum of 3 log files.
</Note>
</ParamField>

  desc="Preload digital human resources."
  parent_file="Declared in \`ZegoDigitalHumanResource.java\`"
  parent_name="ZegoDigitalHumanResource"
  parent_type="class">
**Parameters**

| Name | Type | Description |
| --- | --- | --- |
| context | Context | Android context. |
| auth | [ZegoDigitalMobileAuth](#zegodigitalmobileauth) | Authentication information. |
| digitalHumanID | String | Digital human ID. |
| callback | [PreloadCallback](#zegodigitalhumanresourcepreloadcallback) | Preload progress and result callback. |

**Details**

Asynchronously preload avatar package resources for the specified digital human. After resources are loaded, local cache will be used directly when starting that digital human, greatly improving loading speed.

<Note title="">
- **Recommended practice**: Call in advance during application idle time or before user enters a specific page.
- **Calling timing**: Call after [ZegoExpressEngine.createEngine] and before [ZegoExpressEngine.destroyEngine], i.e., depends on the existence of ZegoExpressEngine instance.
</Note>

<Warning title="">
- **Network requirement**: This method requires network connection to synchronize resource status information.
</Warning>
</ParamField>
