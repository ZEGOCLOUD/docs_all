# Run Example Code

This document describes how to quickly run the Digital Human iOS example code to help you experience Digital Human features.

## Prerequisites

Before running the example code, make sure your development environment meets the following requirements:

* Xcode 15.0 or later
* iOS 12.0 or later with audio/video support on an iOS device or simulator (real device recommended)
* CocoaPods 1.10.0 or later
* The iOS device/simulator and development computer are connected to the Internet

<Note title="Note">

* When compiling the example code for the first time, CocoaPods will automatically download dependencies, which may take a while. Please be patient.
* If CocoaPods installation is slow, consider switching to a domestic mirror source.
* The SDK version used in the example code can be viewed in the `Podfile` file.

</Note>

## Preconditions

1. You have deployed a business server integrated with ZEGOCLOUD Digital Human API. Please first check the server-side [Run Example Code](/aigc-digital-human-server/quick-start/run-example-code) and ensure the server API is accessible.

After downloading the example code, enter the `iOS` directory. All file paths mentioned below are relative to this directory.

```tree

│   ├─ ZegoDigitalHumanQuickStart/                # Project source code directory
│   │   ├─ Models/                                # Data models
│   │   │   ├─ ZegoConfig.h/.m                    # App configuration model
│   │   │   ├─ ZegoDigitalHumanInfo.h/.m          # Digital Human info model
│   │   │   └─ ZegoTask.h/.m                      # Task model
│   │   ├─ Network/                               # Network request related
│   │   │   ├─ ZegoAPIConstants.h                 # API constants configuration (server address configured here)
│   │   │   ├─ ZegoNetworkManager.h/.m            # Network manager
│   │   │   └─ ZegoAPIService.h/.m                # API service
│   │   ├─ ViewControllers/                       # View controllers
│   │   ├─ Views/                                 # View components
│   │   ├─ Utils/                                 # Utility classes

```

## Run the Example Code

<Steps>
<Step title="Clone the source code">

Open a terminal and run the following command to clone the repository locally:

```bash
git clone https://github.com/ZEGOCLOUD/digital_human_paas_quick_start.git
```

</Step>
<Step title="Install dependencies">

Navigate to the `iOS/ZegoDigitalHumanQuickStart` directory and run the following command to install dependencies:

```bash
cd iOS/ZegoDigitalHumanQuickStart
pod install
```

</Step>
<Step title="Configure server URL">

Open the `ZegoDigitalHumanQuickStart/Network/ZegoAPIConstants.h` file and modify `kZegoDefaultServerURL` to your business server API address.

```objc title="ZegoAPIConstants.h"
// Note: The example code runs on iOS devices. Please enter the actual server IP or domain name.
// Do not use 127.0.0.1 or localhost, otherwise the server cannot be accessed.
static NSString * const kZegoDefaultServerURL = @"https://your-server-address/api";
```

<Warning title="Warning">

Please ensure the server address is accessible, otherwise the example code will not run properly.

</Warning>

</Step>
<Step title="Configure permissions">

The example code has already declared the required permissions (microphone, camera, local network, etc.) in `Info.plist`. When running the app for the first time, it will automatically request permissions. Please allow the relevant permissions.

</Step>
<Step title="Run the project">

1. Open the `ZegoDigitalHumanQuickStart.xcworkspace` file with Xcode
2. Connect an iOS device or launch a simulator
3. Click the **Run** button in Xcode to install the app on the device

</Step>
</Steps>

## Experience Digital Human Features

Run the project on a real device. After successful launch, you will see the Digital Human interface. The example app provides the following features:

1. **Create task**: Click "Open Control Panel" → "Create Task" button to create a Digital Human streaming task
2. **Drive the Digital Human**:
   - **Text drive**: Enter text in the drive control area and click "Text Drive", and the Digital Human will broadcast the corresponding content
   - **Audio drive**: Upload or record audio to drive the Digital Human to broadcast the audio content
   - **WebSocket drive**: Drive the Digital Human in real-time streaming (TTS audio stream) via WebSocket protocol, suitable for real-time conversation scenarios
3. **Task management**: Click "Stop Task" to end the current Digital Human service

<Note title="Note">

* When running the app for the first time, microphone and camera permissions must be granted, otherwise some features may not work properly.
* Ensure the server has been correctly deployed and is running, otherwise tasks cannot be created and the Digital Human cannot be driven.
* Digital Human resources will be automatically preloaded on first use. Please be patient.

</Note>

## FAQ

**1. "Network error" or "Server connection failed" when running the example code. How to fix?**

Please troubleshoot with the following steps:
* Check if the server URL configured in `ZegoAPIConstants.h` is correct
* Ensure the server has been correctly deployed and is running. You can test API accessibility via a browser or Postman
* Ensure the iOS device can reach the server network. For local testing, ensure the device and computer are on the same local network
* Check if `Info.plist` has been configured to allow local network access (`NSAppTransportSecurity`)

**2. What to do if `pod install` fails?**

Please troubleshoot with the following steps:
* Confirm CocoaPods is installed. If not, run `sudo gem install cocoapods` to install it
* Try updating the CocoaPods repository: `pod repo update`
* If installation is slow due to network issues, consider switching to a domestic mirror source

**3. Digital Human does not display or shows a black screen. How to fix?**

Please troubleshoot with the following steps:
* Ensure a task has been successfully created and you have joined a [Video Call room](/real-time-video-ios/quick-start/implementing-video-call#createroom)
* Ensure the [Base64Config](/aigc-digital-human-ios/quick-start) returned by the server is configured correctly
* Check the SDK logs in the Xcode Console to confirm the Digital Human SDK has started properly
