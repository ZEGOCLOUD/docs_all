# Integrate SDK
---

This page only describes the SDK components required for integrating Digital Human in an iOS project. For example code and business workflows, please refer to other sections.

## Prerequisites

- Xcode 15.0 or later.
- iOS 12.0 or later device (real device debugging requires network connectivity).
- You have obtained a valid AppID / Token and Digital Human permissions (from ZEGOCLOUD Technical Support).

## Integrate ZEGO Express SDK

Since the current Digital Human feature requires the ZEGO Express SDK optimized for Digital Human, the following integration methods are recommended:

### Method 1: CocoaPods (Recommended)

1. Ensure CocoaPods version >= 1.10.0.
2. Run `pod init` in the project root directory and add the following to the target in the Podfile:
   ```ruby
   target 'MyProject' do
       use_frameworks!
       # ZEGO Express SDK optimized for Digital Human
       pod 'ZegoExpressPrivate', '3.22.0.46788'
   end
   ```
3. Run `pod repo update` followed by `pod install`.

### Method 2: Manual Integration

1. Obtain the ZEGO Express SDK optimized for Digital Human from the [Download page](./introduction/download.mdx).
2. In Xcode, select "File > Add Files to YourProject" and add the extracted `ZegoExpressPrivate.xcframework` (or `ZegoExpressEngine.xcframework`) to the project.
3. In "TARGETS > General > Frameworks, Libraries, and Embedded Content", add the framework and set Embed to "Embed & Sign".

## Integrate Digital Human SDK

The core Digital Human capabilities are provided by `ZegoDigitalMobile`, supporting iOS 12+.

## Permission Configuration

Depending on whether your app captures audio, add at least the microphone usage description in `Info.plist`. It is also recommended to declare background audio mode:

```xml
<key>UIBackgroundModes</key>
<array>
    <string>audio</string>
</array>
<key>NSMicrophoneUsageDescription</key>
<string>Microphone access is required for voice interaction</string>
```

If you only play the Digital Human video without capturing audio, you can remove the microphone permission declaration as needed. Please ensure the corresponding system permissions are enabled when debugging on a real device.
