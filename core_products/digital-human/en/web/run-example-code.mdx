## Prepare Environment

Before running the sample source code, please ensure your development environment meets the following requirements:

### Development Environment Requirements

- **Node.js**: 16.0 or higher
- **Package Manager**: npm, yarn, or pnpm
- **Browser**: Modern browsers such as Chrome, Firefox, Safari, Edge

| Chrome 58 or above| ✔️ | ✔️ | ✔️ | ✔️ |

## Run Sample Source Code

<Steps>
<Step title="Clone Source Code">

Open a terminal and execute the following command to clone the repository locally:

```bash
git clone https://github.com/ZEGOCLOUD/digital_human_paas_quick_start.git
cd digital_human_paas_quick_start/Web
```

</Step>
<Step title="Install Dependencies">

In the Web directory, use one of the following commands to install dependencies:

<CodeGroup>
  ```bash npm
  npm install
  ```
  ```bash yarn
  yarn install
  ```
  ```bash pnpm
  pnpm install
  ```
</CodeGroup>

</Step>
<Step title="Configure Environment Variables">

Create a `.env` file in the Web directory (if it doesn't exist) and configure the server address:

```env
VITE_API_BASE_URL=http://localhost:3000/api
VITE_APP_SERVER=wss://accesshub-wss.zego.im/accesshub
```

Configuration description:
- `VITE_API_BASE_URL`: Business server API address
- `VITE_APP_SERVER`: ZEGO Server address, obtained from [ZEGOCLOUD Console](https://console.zego.im/)

<Warning title="Note">

- Ensure the server is started and running at the configured address
- For production environment, please use HTTPS protocol

</Warning>

</Step>
<Step title="Start Development Server">

Run the following command to start the development server:

<CodeGroup>
```bash npm
npm run dev
```
```bash yarn
yarn dev
```
```bash pnpm
pnpm dev
```
</CodeGroup>


After successful startup, the browser will automatically open `http://localhost:5173` (or the address shown in the console).

</Step>
</Steps>

## Experience Digital Human Features

After opening the sample application in your browser, you can experience the following digital human features:

1. **Get Digital Human Information**: After the page loads, the system will automatically retrieve available digital human resource information from the server.

2. **Create Task**: Click the "Create Task" button to create a digital human task. After successful task creation, the digital human screen will automatically display on the page.

3. **Drive Digital Human**: In the control panel, you can experience the following driving methods:
   - **Text Drive**: Input text content to drive the digital human to speak the text
   - **Audio Drive**: Upload or record audio to drive the digital human to speak the audio content
   - **WebSocket Drive**: Drive the digital human in real-time via WebSocket protocol (TTS audio stream), suitable for real-time conversation scenarios

4. **Task Management**: You can stop the current task, interrupt ongoing playback, etc.

<Note title="Description">

- First-time use requires the server to be running properly
- If the page cannot load digital human information, please check server configuration and network connection
- Digital human rendering requires certain network bandwidth, it is recommended to use in a stable network environment

</Note>

## FAQ

**1. When running the sample source code, it shows "Network Error" or "Server Connection Failed". How to handle it?**

Please troubleshoot with the following steps:
- Check if the `VITE_API_BASE_URL` configured in the `.env` file is correct
- Ensure the server is correctly deployed and running. You can test if the API is accessible via browser or Postman
- Check if there are any CORS (Cross-Origin Resource Sharing) errors in the browser console
- Ensure network connection is normal. If testing locally, check if proxy configuration is correct

**2. Task creation failed with "AppID Not Set" or "Token Empty". How to handle it?**

The sample application automatically gets AppID and Token from the server API without manual configuration. Please check:
- Ensure the server API correctly returns task configuration information
- Use browser developer tools to view network requests and confirm the data format returned by the server is correct
- Check if server environment variables are correctly configured (AppID and ServerSecret)
