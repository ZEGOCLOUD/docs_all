## 准备环境

在运行示例源码前，请确保开发环境满足以下要求：

### 开发环境要求

- **Node.js**: 16.0 或更高版本
- **包管理器**: npm、yarn 或 pnpm
- **浏览器**: Chrome、Firefox、Safari、Edge 等现代浏览器

### 浏览器兼容性

|浏览器|Windows|Mac|Android|iOS|
| --- | --- | --- | --- | --- |
| Chrome 58 或以上| ✔️ | ✔️ | ✔️ | ✔️ |
| FireFox 56 或以上 | ✔️ | ✔️ | ✔️ | ✖ |
| Safari 11 或以上 | - | ✔️ | - | ✔️ |
| Edge 79 或以上 | ✔️ | ✔️ | ✔️ | ✔️ |

<Warning title="注意">

仅支持 SSL 的 Web 服务器（HTTPS）。本地开发环境可使用 localhost 或 127.0.0.1。

</Warning>

## 运行示例源码

<Steps>
<Step title="克隆源码">

打开终端，执行以下命令克隆仓库到本地：

```bash
git clone https://github.com/ZEGOCLOUD/digital_human_paas_quick_start.git
cd digital_human_paas_quick_start/Web
```

</Step>
<Step title="安装依赖">

在 Web 目录下，使用以下命令之一安装依赖：

<CodeGroup>
  ```bash npm
  npm install
  ```
  ```bash yarn
  yarn install
  ```
  ```bash pnpm
  pnpm install
  ```
</CodeGroup>

</Step>
<Step title="配置环境变量">

在 Web 目录下创建 `.env` 文件（如果不存在），配置服务端地址：

```env
VITE_API_BASE_URL=http://localhost:3000/api
VITE_APP_SERVER=wss://accesshub-wss.zego.im/accesshub
```

配置说明：
- `VITE_API_BASE_URL`: 业务服务端 API 地址
- `VITE_APP_SERVER`: ZEGO Server 地址，从 [ZEGO 控制台](https://console.zego.im/) 获取

<Warning title="注意">

- 确保服务端已启动并运行在配置的地址上
- 生产环境请使用 HTTPS 协议

</Warning>

</Step>
<Step title="启动开发服务器">

运行以下命令启动开发服务器：

<CodeGroup>
```bash npm
npm run dev
```
```bash yarn
yarn dev
```
```bash pnpm
pnpm dev
```
</CodeGroup>


启动成功后，浏览器会自动打开 `http://localhost:5173`（或控制台显示的地址）。

</Step>
</Steps>

## 体验数字人功能

在浏览器中打开示例应用后，您可以体验以下数字人功能：

1. **获取数字人信息**：页面加载后，系统会自动从服务端获取可用的数字人资源信息。

2. **创建任务**：点击"创建任务"按钮，创建数字人任务。任务创建成功后，数字人画面会自动显示在页面上。

3. **驱动数字人**：在控制面板中，可以体验以下驱动方式：
   - **文本驱动**：输入文本内容，驱动数字人播报该文本
   - **音频驱动**：上传或录制音频，驱动数字人播报音频内容
   - **WebSocket 驱动**：通过 WebSocket 协议实时流式（TTS 音频流）驱动数字人，适用于实时对话场景

4. **任务管理**：可以停止当前任务、打断正在进行的播报等。

<Note title="说明">

- 首次使用需要确保服务端正常运行
- 如果页面无法加载数字人信息，请检查服务端配置和网络连接
- 数字人渲染需要一定的网络带宽，建议在稳定的网络环境下使用

</Note>

## 常见问题

**1. 运行示例源码时提示"网络错误"或"服务端连接失败"，如何处理？**

请按以下步骤排查：
- 检查 `.env` 文件中配置的 `VITE_API_BASE_URL` 是否正确
- 确保服务端已正确部署并运行，可以通过浏览器或 Postman 测试 API 是否可访问
- 检查浏览器控制台是否有跨域（CORS）错误
- 确保网络连接正常，如果是本地测试，检查代理配置是否正确

**2. 创建任务失败，提示"AppID 未设置"或"Token 为空"，如何处理？**

示例应用会从服务端 API 自动获取 AppID 和 Token，无需手动配置。请检查：
- 确保服务端 API 正常返回了任务配置信息
- 使用浏览器开发者工具查看网络请求，确认服务端返回的数据格式正确
- 检查服务端环境变量配置是否正确（AppID 和 ServerSecret）
