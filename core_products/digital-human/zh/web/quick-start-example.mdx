# 示例源码运行指引

---

本文介绍如何快速跑通数字人 Web 示例源码，体验数字人渲染和驱动功能。

## 准备环境

### 开发环境要求

- **Node.js**: 16.0 或更高版本
- **包管理器**: npm、yarn 或 pnpm
- **浏览器**: Chrome、Firefox、Safari、Edge 等现代浏览器

### 浏览器兼容性

|浏览器|Windows|Mac|Android|iOS|
| --- | --- | --- | --- | --- |
| Chrome 58 或以上| ✔️ | ✔️ | ✔️ | ✔️ |
| FireFox 56 或以上 | ✔️ | ✔️ | ✔️ | ✖ |
| Safari 11 或以上 | - | ✔️ | - | ✔️ |
| Edge 79 或以上 | ✔️ | ✔️ | ✔️ | ✔️ |

<Warning title="注意">

仅支持 SSL 的 Web 服务器（HTTPS）。本地开发环境可使用 localhost 或 127.0.0.1。

</Warning>

## 前提条件

1. 已在 [ZEGO 控制台](https://console.zego.im/) 创建项目，申请有效的 AppID 和 ServerSecret，详情请参考 [控制台 - 项目管理](/console/project-info) 中的"项目信息"。

2. 已参考 [服务端示例源码运行指引](/aigc-digital-human-server/quick-start/quick-start-example) 启动服务端，并确保服务端已配置以下环境变量：
   - `NEXT_PUBLIC_ZEGO_APP_ID`: 您的 ZEGO AppID
   - `ZEGO_SERVER_SECRET`: 您的 ZEGO ServerSecret
   - `ZEGO_API_HOST`: ZEGO API 地址（默认：`aigc-digital-human-api.zegotech.cn`）
   - `PORT`: 服务端端口（默认：3000）

## 运行示例源码

### 1. 安装依赖

在 Web 目录下，使用以下命令之一安装依赖：

```bash
npm install
# 或
yarn install
# 或
pnpm install
```

### 2. 配置环境变量

在 Web 目录下创建 `.env` 文件（如果不存在），配置服务端地址：

```env
VITE_API_BASE_URL=http://localhost:3000
VITE_APP_SERVER=https://wss-l3-api.zego.im
```

其中：
- `VITE_API_BASE_URL`: 服务端 API 地址（需要与启动的服务端地址一致）
- `VITE_APP_SERVER`: ZEGO RTC 服务器地址

<Warning title="注意">

- 确保服务端已启动并运行在配置的地址上
- 如果服务端运行在其他地址或端口，请相应修改 `VITE_API_BASE_URL`
- 生产环境请使用 HTTPS 协议

</Warning>

### 3. 启动开发服务器

运行以下命令启动开发服务器：

```bash
npm run dev
# 或
yarn dev
# 或
pnpm dev
```

启动成功后，浏览器会自动打开 `http://localhost:5173`（或控制台显示的地址）。

### 4. 体验数字人功能

1. 页面加载后，系统会自动从服务端获取数字人信息。

2. 点击"创建任务"按钮，创建数字人视频流任务。

3. 任务创建成功后，数字人画面会自动显示在页面上。

4. 在控制面板中，可以体验以下功能：
   - **文本驱动**: 驱动数字人播报文本内容。
   - **音频驱动**: 驱动数字人播报音频内容。
   - **WebSocket 驱动**: 通过 WebSocket 协议实时流式驱动数字人。
   - **停止任务**: 停止当前的数字人视频流任务。

<Warning title="注意">

- 首次使用需要确保服务端正常运行
- 如果页面无法加载数字人信息，请检查服务端配置和网络连接
- 数字人渲染需要一定的网络带宽，建议在稳定的网络环境下使用

</Warning>
