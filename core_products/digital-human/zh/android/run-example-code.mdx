## 准备环境

在运行示例源码前，请确保开发环境满足以下要求：

* Android Studio 2020.3.1 或以上版本
* Gradle 8.13
* Android Gradle 插件 8.12.3
* Android 8.0（API 26）或以上版本，且支持音视频的 Android 设备或模拟器（推荐使用真机）
* Android 设备、开发电脑已经连接到 Internet

<Note title="说明">

- 首次编译示例源码时，如果没有对应的依赖库或者构建工具，Android Studio 会自动下载依赖，可能需要较长时间，请耐心等待。
- 建议提前下载特定的 Gradle，请参考 [Gradle Services](https://services.gradle.org) 下载。
- 示例源码中使用的 Gradle 相关版本在 `gradle/wrapper/gradle-wrapper.properties` 和 `gradle/libs.versions.toml` 文件中查看。
- 关于 Android Gradle 插件、Gradle、SDK Tool 之间的版本依赖关系，请查看：[Android Gradle 插件版本说明](https://developer.android.com/studio/releases/gradle-plugin)

</Note>

## 示例源码目录结构

下载示例源码后，进入 `Android` 目录，下文所涉及的文件路径均为相对于此目录的路径。

```tree
.
├─ app/
│   ├─ src/
│   │   └─ main/
│   │       ├─ java/com/example/zegodigitalhumanquickstart/
│   │       │   ├─ ZegoQuickStartConstants.java    # 密钥配置文件
│   │       │   ├─ activity/                       # 主界面 Activity
│   │       │   ├─ callback/                       # 回调接口
│   │       │   ├─ model/                          # 数据模型
│   │       │   ├─ network/                        # 网络请求
│   │       │   ├─ util/                           # 工具类
│   │       │   └─ view/                           # 自定义视图
│   │       ├─ res/                                # 资源文件
│   │       └─ AndroidManifest.xml                 # 应用清单（含权限声明）
│   └─ build.gradle                                # 应用构建配置
├─ gradle/
│   ├─ libs.versions.toml                          # 版本目录管理
│   └─ wrapper/                                    # Gradle 包装器
├─ build.gradle                                    # 项目构建配置
├─ settings.gradle                                 # 项目设置
└─ gradle.properties                               # Gradle 属性配置
```


## 运行示例源码

<Steps>
<Step title="克隆源码">

打开终端，执行以下命令克隆仓库到本地：

```bash
git clone https://github.com/ZEGOCLOUD/digital_human_paas_quick_start.git
```

</Step>
<Step title="配置服务器 URL">

打开 `app/src/main/java/com/example/zegodigitalhumanquickstart/ZegoQuickStartConstants.java` 文件，将 `DEFAULT_SERVER_URL` 修改为您的业务服务端 API 地址。

```java title="ZegoQuickStartConstants.java"
public static final String DEFAULT_SERVER_URL = "http://您的服务端地址/api";
```

<Warning title="注意">

请确保服务端地址可正常访问，否则示例源码无法正常运行。

</Warning>

</Step>
<Step title="配置权限">

示例源码已在 `AndroidManifest.xml` 中声明了所需权限（网络、摄像头、麦克风等），首次运行应用时会自动请求运行时权限，请允许相关权限。

</Step>
<Step title="运行项目">

1. 使用 Android Studio 打开 `Android` 目录
2. 连接 Android 设备或启动模拟器
3. 点击 Android Studio 的 **Run** 按钮，将应用安装到设备上

</Step>
</Steps>

## 体验数字人功能

在真机中运行项目，运行成功后，可以看到数字人界面。示例应用提供以下功能：

1. **创建任务**：点击"创建任务"按钮，创建数字人流任务
2. **驱动数字人**：
   - **文本驱动**：输入文本内容，驱动数字人播报该文本
   - **音频驱动**：上传或录制音频，驱动数字人播报音频内容
   - **WebSocket 驱动**：通过 WebSocket 协议实时流式（TTS 音频流）驱动数字人，适用于实时对话场景
3. **任务管理**：可以停止当前任务、打断驱动、销毁所有任务等

<Note title="说明">

- 首次运行应用时，需要授予摄像头和麦克风权限，否则部分功能可能无法正常使用。
- 确保服务端已正确部署并运行，否则无法创建任务和驱动数字人。
- 数字人资源会在首次使用时自动预加载，请耐心等待。

</Note>

## 常见问题

**1. 运行示例源码时提示"网络错误"或"服务端连接失败"，如何处理？**

请按以下步骤排查：
- 检查 `ZegoQuickStartConstants.java` 中配置的服务器 URL 是否正确
- 确保服务端已正确部署并运行，可以通过浏览器或 Postman 测试 API 是否可访问
- 确保 Android 设备与服务端网络连通，如果是本地测试，确保设备和电脑在同一局域网
- 检查 AndroidManifest.xml 中是否已声明网络权限

**2. 创建任务失败，提示"AppID 未设置"或"Token 为空"，如何处理？**

示例应用会从服务端 API 自动获取 AppID 和 Token，无需手动配置。请检查：
- 确保服务端 API 正常返回了任务配置信息
- 使用 Android Studio 的 Logcat 查看网络请求日志，确认服务端返回的数据格式正确

**3. 数字人无法显示或黑屏，如何处理？**

请按以下步骤排查：
- 确保已成功创建任务并登录 [实时音视频房间](/real-time-video-android-java/quick-start/implementing-video-call#createroom)
- 确保服务端返回的 [Base64Config](/aigc-digital-human-android/quick-start) 配置正确
- 在 Logcat 中过滤 `Zego` 标签查看 SDK 日志，确认数字人 SDK 是否正常启动
