# Restrictions and Limitations
---

This document describes the usage restrictions of the ZEGO Express SDK Room service, including rate limits, string size limits, etc.

## Room

The ZEGO Room service supports millions of users online simultaneously in a single room. By default, the total login QPS for all users in each room is 200, which means a maximum of 200 users can login to the same room per second.


<Note title="Note">If you need to increase the limit, please contact ZEGOCLOUD Technical Support for evaluation.</Note>

## In-room Messaging

| SDK API   | Size Limit | Send Frequency Limit (Within a single room in the SDK)  | Description |
| :---------- | :----------- | :-------- | :-- |
| Send broadcast message  [sendBroadcastMessage](https://doc-zh.zego.im/unique-api/express-video-sdk/zh/dart_flutter/zego_express_engine/ZegoExpressEngineIM/sendBroadcastMessage.html)    | The default message length limit is 1 KB. To send messages larger than 1 KB, please contact ZEGOCLOUD Technical Support for configuration.    |  10 times/s  | <ul><li>Messages are reliable.</li><li>The maximum QPS for a single user calling this API on the client is 2.</li></ul> |
| Send barrage message [sendBarrageMessage](https://doc-zh.zego.im/unique-api/express-video-sdk/zh/dart_flutter/zego_express_engine/ZegoExpressEngineIM/sendBarrageMessage.html)  | The default message length limit is 1 KB. To send messages larger than 1 KB, please contact ZEGOCLOUD Technical Support for configuration.  | 20 times/s | <p>Messages are not guaranteed to be reliable.</p> |
| Send custom signaling message [sendCustomCommand](https://doc-zh.zego.im/unique-api/express-video-sdk/zh/dart_flutter/zego_express_engine/ZegoExpressEngineIM/sendCustomCommand.html)   | The message length cannot exceed 1 KB. | <ul><li>When sending to a single user: 200 messages/s</li><li>When sending to multiple users: 10 messages/s</li></ul> | <ul><li>Messages are reliable.</li></ul> |

<Note title="Note">`Message reliability` here refers to whether the remote user can successfully receive the message when the network connection is normal.</Note>
