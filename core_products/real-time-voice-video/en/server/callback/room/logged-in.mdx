# Room Login Callback

When the SDK logs in to a room, it triggers the `room_login` callback.

## Callback Description

This callback is triggered when the user successfully logs in to the room. Developers can use this callback to get information about the user's login status and perform subsequent operations.

## Callback Trigger Timing

Triggered after the user successfully logs in to the room.

## API Structure

```javascript
callback = (response) => {
    // callback logic
}
```

## Precautions

- The callback is triggered after the user successfully logs in to the room.
- The `custom_info` field is the custom information passed during login, which can be used to carry user-specific information.
- The `term_type` field indicates the type of terminal the user is using, which can be used to implement terminal-specific logic.

## Code Example

```javascript
// Room login callback
ZegoUIKit.prebuilt.createKitContext().on('room_login', (response) => {
    console.log('Room login callback:', response);
    
    if (response.code === 200) {
        console.log('User logged in successfully:', response.uid);
        console.log('Room ID:', response.room_id);
        console.log('Terminal type:', response.term_type);
    } else {
        console.error('Room login failed:', response.message);
    }
});
```
