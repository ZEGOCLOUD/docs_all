# 即构实时音视频Express SDK 合规使用指南

为帮助使用即构实时音视频Express SDK的移动应用开发者（以下简称"您"）在符合个人信息保护相关法律法规、政策及标准的规定下合规接入、使用第三方 SDK，深圳市即构科技有限公司（以下简称"我们"）特制定本《即构实时音视频 Express SDK 合规使用指南》（以下简称"指南"），便于您在使用实时音视频 Express SDK 过程中符合相应的合规要求。请您在接入、使用实时音视频 Express SDK 前，充分阅读和了解本指南内容。

## 一、接入/升级至满足监管新规的最新 SDK 版本

我们始终高度重视 SDK 的功能优化及个人信息安全防护工作，将根据实际业务需求，适时推进 SDK 的版本升级与迭代，持续提升产品的安全防护能力与运行稳定性，确保全面契合法律法规、监管政策及行业标准的最新合规要求。在此强烈建议您及时升级至 SDK 最新版本，这样既能让您顺利使用 SDK 的各项新增功能，也能有效规避因版本更新不及时可能带来的不良后果（如 App 被监管通报、下架等情况）。

每次SDK 完成更新后，我们会第一时间通过官网公告或其他合适的渠道向您同步更新详情，帮助您及时掌握 SDK 最新版本的相关信息。与此同时，您也可以通过我们的官方指定渠道获取 SDK 最新版本的下载链接。

## 二、App 隐私政策中应披露的本 SDK 相关情况

为充分保障用户个人信息权益，请您务必确保所开发或运营的 App 已配备符合监管要求的《隐私政策》。您须明确向终端用户告知您的 App 已集成本 SDK 服务，并在《隐私政策》中补充载明本 SDK 收集、使用个人信息的目的、方式、范围，同时标注本 SDK 开发运营主体名称及官方隐私政策链接。您应于 App 登录注册页面及首次运行时，以弹窗、文本链接、附件等简洁醒目且便于访问的形式，使用清晰易懂的语言向用户告知《隐私政策》内容，确保用户在充分知情的基础上，作出自愿、明确的意思表示。

以下第三方SDK披露文案示例供您参考，您可通过文字或表格形式向用户履行告知义务。敬请知悉，不同版本 SDK 所提供的功能服务及所需字段信息，可能因开发者的选择或配置不同存在差异。请您结合 SDK 官方隐私政策及实际接入使用的 SDK 运行情况，向用户充分履行告知义务并依法获得用户同意。

## 三、获得用户同意后再初始化 SDK

为满足法律法规及监管要求，您应确保在获得App用户的同意后再初始化 SDK，并在用户触发 SDK 具体功能服务后通过配置 SDK 的相关参数完成发送请求的调用，此时 SDK 才会按照您设置的配置方式采集功能所需的个人信息或申请功能所需的权限。为了避免您在获取用户同意前，提前启动 SDK 收集使用用户个人信息，SDK 提供了延迟 SDK 初始化调用的相关技术配置方案，[点击这里](https://doc-zh.zego.im/real-time-video-android-java/quick-start/integrating-sdk) 查看向此操作指引（以Android平台为例）。

1. 确保在用户阅读 App 隐私政策并取得用户授权之后，按 App 功能需要在合适时机调用 SDK 正式初始化函数初始化 SDK。反之，如果用户不同意《隐私政策》授权，则不能调用正式初始化函数。该接口仅进行初始化，不会获取个人信息。
2. 请勿在用户同意隐私政策之前动态申请涉及用户个人信息的敏感设备权限；请勿在用户同意隐私政策前私自采集和上报个人信息（尤其注意 Android_ID、OAID、IMEI、MAC 地址、硬件序列号、应用安装列表等用户信息）。请勿在 App 处于未激活状态时（例如 App 在后台运行），请求 SDK 相关服务。

## 四、可选信息配置开关

SDK 向您提供了可选个人信息及权限的控制开关，您可以根据 App 所需的 SDK 功能服务自行配置打开或关闭隐私信息请求开关。

**基本功能**：本 SDK 的基本功能是为开发者提供稳定的音视频功能，包括音视频通话、语音通话、在线会议、在线课堂、互动直播等基础音视频功能。

**扩展功能**：本SDK不包含额外的扩展功能。

1. 未经用户同意不得收集任何个人信息。

2. 非服务所必需或无合理应用场景下，用户拒绝相关授权申请后，应用不得自动退出或关闭。

3. 在用户明确拒绝权限申请后，App 不应向用户频繁弹窗或反复申请开启与当前服务场景无关的权限、影响用户正常使用，建议掌握合适时机申请敏感权限，不得影响其他功能可用。

4. 应明确告知用户索取权限的目的和用途。

5. App 首次打开或运行中，未见使用权限对应的相关功能或服务时，不应提前向用户弹窗申请开启敏感权限。

6. 不得超出业务功能实际需要过度收集个人信息。

## 六、用户权利保障机制

本 SDK 提供相应支持，以便您帮助终端用户实现个人信息主体权利请求。

1. 终端用户撤销同意处理其个人信息的授权时，您应停止使用 SDK 相关功能并停止采集与该功能相应的用户数据，确保不再对用户的个人信息进行后续处理。
2. 由于终端用户并非我们的直接用户，与我们并无直接的交互对话界面，为保障用户权利实现，您应作为个人信息处理者，为终端用户提供便于操作的用户权利实现方式，及时响应终端用户个人信息管理请求。
3. 若终端用户提出的个人信息管理请求涉及我们 SDK 处理的个人信息、需要我们协助处理时，您应及时与我们联系，并附上必要的终端用户请求的书面材料。我们将及时核验相关材料，并按照相关法律法规及《实时音视频 Express SDK 个人信息保护规则》，为终端用户行使个人信息权利的请求提供相应的支持与配合。
4. 如我们停止运营，将及时停止收集用户个人信息的活动，停止运营的通知将以公告或其他适当的方式通知您及终端用户，并对所持有的用户个人信息进行删除或匿名化处理。

## 七、联系方式

如果您和/或终端用户对本指南、《实时音视频 Express SDK 个人信息保护规则》或个人信息保护相关事宜有任何疑问、意见或建议，可以通过在线客服或者联系销售的方式反馈给我们。

一般情况下，我们将尽快审核所涉问题，并在验证用户身份后的 15 个工作日内予以回复并处理。如果您对我们的答复不满意，特别是我们的个人信息处理行为损害了您的合法权益，您还可以向网信、公安、工商等监管部门进行投诉或举报，或向有管辖权法院提起诉讼。

## 八、接入实时音视频 Express SDK 前的合规自查

为确保您就本 SDK 的使用获得终端用户的授权，且遵守个人信息保护要求和合规流程，我们建议您在接入实时音视频 Express SDK 前进行合规自查：

1. 请仔细阅读并按本指南提示对您 App 的《隐私政策》进行合规自查，确保完整、准确披露 SDK 相关信息。

2. 请务必做延迟初始化配置，确保获得用户同意后再初始化 SDK。

3. 当实时音视频 Express SDK 基于最新的法律法规或监管要求进行更新后，请您在收到版本更新通知时及时将您 App 集成的实时音视频 Express SDK 升级到最新版本。

4. 遵守其他国家相关法律法规、监管政策及标准的要求。
