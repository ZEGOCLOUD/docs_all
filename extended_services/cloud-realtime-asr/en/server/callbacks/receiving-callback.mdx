# Receiving Callbacks


<Warning title="Note">Callback service cannot be guaranteed to be completely reliable. Please carefully consider the risks of using callback solutions to build core business processes.</Warning>

Through this callback, you can monitor events that occur during real-time speech recognition, including ASR results and exception events.

<Warning title="Note">Please ensure that you have successfully started the real-time speech recognition task by referring to the [Start Task](./../api-reference/start.mdx) interface.</Warning>

## Callback Description

- Request method: POST.
    <Note title="Note">The callback data format is JSON. You need to perform UrlDecode decoding on it.</Note>
- Request address: Please provide the address for your business server to receive callbacks and contact ZEGOCLOUD Technical Support to configure it.
- Transport protocol: HTTPS/HTTP, HTTPS is recommended.

## Callback Parameters

<Note title="Note">ZEGO will continue to optimize and update the relevant callback parameters in future iteration plans (for example: adding new fields or new parameter values for certain fields). When integrating, developers should avoid hard-coding to ensure compatibility with new versions after updates.</Note>

| Parameter | Type | Description |
|------|------|------|
| AppId | Number | The unique identifier of the developer's APP assigned by ZEGO. |
| Event | String | Event notification type.<ul><li>ASRResult: ASR result.</li><li>TranslationResult: Translation result.</li><li>Exception: Exception event.</li></ul> |
| Nonce | String | Random number, used for signature string calculation. |
| Timestamp | Number | Unix timestamp (milliseconds) when the callback is sent, used for signature string calculation. |
| Signature | String | Signature string, used to verify the identity of the callback sender. |
| TaskId | String | The unique identifier of the task, globally unique. |
| RoomId | String | Room ID. |
| Data | Object | Event details. Depending on the Event value, the members of this callback parameter are different. |

### Data

**Depending on the `Event` value, the parameters included in Data are different.**

## Return Response

When you receive a callback, please return HTTP status code 2XX (for example, 200) to indicate successful reception. Returning anything else indicates reception failure.


## Callback Retry Policy

If the ZEGO server does not receive a response, or receives an HTTP status code other than 2XX (for example, 200), it will attempt to retry, **up to 5 retries**. The interval between each retry request and the **previous request** is 2s, 4s, 8s, 16s, 32s respectively. If it still fails after the 5th retry, no more retries will be made, and the callback will be lost.
