# Configure ASR

---

## Feature Overview

To improve the recognition accuracy of speech recognition (or speech-to-text) in different scenarios, you can achieve this through the following methods:
- Choose the appropriate vendor/recognition model: Supports Tencent ASR, Alibaba Bailian Paraformer, Alibaba Bailian Gummy, Microsoft, etc.
- Choose the appropriate language: By default, Tencent and Alibaba Bailian models are for Chinese recognition, and Microsoft is for English recognition.
- Set recognition hotwords: In specific scenarios, there are usually some specialized vocabulary, such as character names, usernames, feature names, etc. You can set temporary hotwords when creating an agent instance to improve speech recognition accuracy.

## Prerequisites

Currently, Tencent is the default enabled and supported speech recognition vendor. If you need Alibaba, Microsoft, or other recognition vendors, please contact ZEGOCLOUD business support to enable them.

## Usage

When creating a real-time speech recognition task ([StartRealtimeASRTask](./api-reference/start.mdx)), you can set the vendor, language, hotwords, and other parameters through the ASR parameter.

## ASR Parameter Description <a id="asr-params" />

| Parameter             | Type   | Required | Description                                                                                                                                                                                                                          |
| --------------------- | ------ | -------- | ------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------ |
| Vendor                | String | No       | ASR vendor, defaults to Tencent:<ul><li>Tencent: Tencent</li><li>AliyunParaformer: Alibaba Cloud Paraformer</li><li>AliyunGummy: Alibaba Cloud Gummy</li><li>Microsoft: Microsoft ASR</li></ul>                                      |
| ~HotWord~             | String | No       | This parameter is deprecated.<br/>Please set it through Params extension parameters. For specific usage, refer to the hotword setting instructions for each vendor below.                                                            |
| Params                | Object | No       | Vendor parameters. For specific usage, refer to the parameter setting instructions for each vendor below.                                                                                                                            |
| VADSilenceSegmentation | number | No      | Used to set how many milliseconds after the user stops speaking, two sentences will no longer be considered as one. Range [200, 2000], default is 500. For detailed explanation, refer to [Speech Segmentation Control](#speech-segmentation-control). |

#### Set Hotwords

Only supports setting parameters through temporary hotword lists. For details, refer to [Tencent Cloud - Real-Time Speech Recognition](https://cloud.tencent.com/document/product/1093/48982#835304ab-3b0b-487e-9daa-08f7086b4fdf) `hotword_list` parameter detailed description.

Single hotword limit: "hotword|weight", single hotword no more than 30 characters (maximum 10 Chinese characters), weight [1-11], such as: "ZEGOCLOUD|5" or "ASR|11";
Temporary hotword list limit: Multiple hotwords separated by English commas, up to 128 hotwords supported, such as: "ZEGOCLOUD|10,Real-time Interaction|5,ASR|11";

Notes:
1. When the hotword weight is set to 11, the current hotword will be upgraded to a super hotword. It is recommended to only set important and mandatory hotwords to 11. Setting too many hotwords with weight 11 will affect the overall character accuracy rate.
2. Hotwords cannot contain spaces, such as: `ZEGOCLOUD Real-time Interaction`

#### Usage Example

The following is an example of setting "zego" as a hotword and language to English.

```json StartRealtimeASRTask
// !mark(6:7)
{
  "RoomId": "rtc_room_id",
  "ASR": {
    "Vendor": "Tencent",
    "Params": {
      "engine_model_type": "16k_zh",
      "hotword_list": "zego|10"
    }
  }
}
```

<Warning title="Note">Microsoft currently only supports setting `phrase_list` and `language` parameters.</Warning>

#### Set Phrase List

A phrase list is a pre-provided list of words or phrases that helps improve recognition capabilities. Phrases added to the phrase list have higher importance and are more likely to be recognized.
You can set the phrase list through the `phrase_list` parameter. For detailed explanation, refer to [Microsoft - Improve Recognition Accuracy with Phrase Lists](https://learn.microsoft.com/en-us/azure/ai-services/speech-service/improve-accuracy-phrase-list?tabs=terminal&pivots=programming-language-python).

<Note title="Note">Allowed characters include locale-specific letters and numbers, space characters, and special characters such as +, -, $, :, (, ), \{, \}, _, ,, ?, @, \\, ', &, #, %, ^, *, \`, \<, \>, ;, /. Other special characters will be removed from within phrases.</Note>

#### Set Language
You can set the language through the `language` parameter. For detailed explanation, refer to [Microsoft - Language and Voice Support for Speech Service](https://learn.microsoft.com/en-us/azure/ai-services/speech-service/language-support?tabs=stt).

#### Usage Example

The following is an example of setting "zego" and "microsoft" as phrase list and language to English.

```json RegisterAgent
// !mark(6:10)
{
  "RoomId": "rtc_room_id",
  "ASR": {
    "Vendor": "Microsoft",
    "Params": {
        "language": "en-US",
        "phrase_list": [
            "zego",
            "microsoft"
        ]
    }
  }
}
```

## Speech Segmentation Control <a id="speech-segmentation-control" />
Determining whether the user has finished speaking can be influenced by the `VADSilenceSegmentation` parameter.

|Configuration|Q&A Result|
|---|---|
|VADSilenceSegmentation = 500ms|User is determined to have said 2 sentences:<br/>1st sentence: The weather is really nice today. I want to go out and play<br/>2nd sentence: How about you?<Note title="Note">Since 400ms \< VADSilenceSegmentation, the first two segments are counted as the 1st sentence; 800ms > VADSilenceSegmentation, so the third segment is counted as an independent 2nd sentence.</Note>|

### Best Practice Configuration

<Note title="Note">If you don't know which configuration works better, we recommend using Scenario 1 configuration.</Note>

| Scenario | VADSilenceSegmentation |
| --- | --- |
| Scenario 1: Need to get recognition results as soon as possible. Used for displaying subtitles, etc. | 500ms |
| Scenario 2: Want the most accurate recognition results possible, can accept some delay. For example, real-time summarization. | 1000ms |
