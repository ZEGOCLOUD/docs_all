## 2023-11-14

|Release Item-30% | Description-50% | Related Documents-20% |
| -------- | -------- | -------- |
| Server Update | When creating a cloud player, a new parameter `RoomUserUpdate` is added.When this parameter is set to `1`, the <a href="/article/api?doc=Express_Video_SDK_API~java_android~class~IZegoEventHandler#on-room-user-update" target="_blank">onRoomUserUpdate</a> callback will be triggered on the client side for rooms created by the cloud player. This callback notifies the developer when other users come online or go offline in the room, causing changes in the user list. | [Create a cloud player](/cloud-player-server/create-player) |
| Server Update | A new callback event type (the `EventType` parameter adds a new value `4`) is added, which represents abnormal events detected by the cloud player during runtime.After detecting such abnormal events, the cloud player will automatically retry internally. If multiple callbacks of this type of exception are received within a short period of time, developers need to check if the resource server access is normal or perform other fault-tolerant processing. | [Related callbacks and verification instructions](/cloud-player-server/verification-instructions) |
