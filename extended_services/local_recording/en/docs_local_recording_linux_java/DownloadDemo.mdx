## Running the demo

This guide describes how to quickly run the demo source code to experience the basic audio and video stream recording features.

### Prerequisites

- You have created a project in the [ZEGOCLOUD Console](https://console.zegocloud.com) and obtained a valid AppID. For details, see "Project Information" in [Console - Project Management](/console/project-info).

### Run the demo

Assuming the on-premise server recording Demo is placed in the "/root" directory, the Demo directory structure is as follows:

<Frame width="512" height="auto" caption=""><img src="https://doc-media.zego.im/sdk-doc/Pics/Linux/ServerRecording/Java/4.jpeg" /></Frame>

```tree
.
├─ libs
    ├─ commons-cli-1.4.jar        # Command-line argument parsing jar
    ├─ commons-codec-1.15.jar     # Common utility jar
    ├─ libzegoliveroomrecorder.so # Recording SDK so
    ├─ ZegoLiveRoom.jar           # Recording SDK jar
├─ LiveRoomDemo.iml
├─ out                            # Demo output directory
├─ src                            # Source code
```

1. Run `cd ~/ZegoServerRecordingDemo` to navigate to the "ZegoServerRecordingDemo" directory.

<Frame width="512" height="auto" caption="">
  <img src="https://doc-media.zego.im/sdk-doc/Pics/Linux/ServerRecording/use_step1.jpg" />
</Frame>

2. Run `cd out/artifacts/playrecorder_demo_jar` to navigate to the "playrecorder_demo_jar" directory.

<Frame width="512" height="auto" caption="">
  <img src="https://doc-media.zego.im/Pics/Linux/ServerRecording/Java/2.jpeg" />
</Frame>

3. Run `java -Djava.library.path=./../../../libs -cp playrecorder-demo.jar com.zego.record.LiveRoomRecorder -a {app_id} -k {token} -r {room_id} -u {user_id}` to run the demo and start single-stream recording. Where:

- "app_id" refers to the AppID obtained in the [Prerequisites](#prerequisites).
- "room_id" refers to the room ID of the stream you want to record.
- "user_id" refers to the user ID used to log in to the room.
- "token" refers to the authentication Token for logging in to the room where the stream resides.
- If no additional parameters are set, the default recording duration is 20s, recording single streams, generating one .mp4 audio and video file.

<Frame width="512" height="auto" caption="">
  <img src="https://doc-media.zego.im/Pics/Linux/ServerRecording/Java/3.jpeg" />
</Frame>

<Warning title="Note">

- The recorded files are saved in the `./zego/log_[roomID]_[timestamp]` directory, where:
    - roomID is the room ID specified when running the Demo.
    - timestamp is the timestamp when running the Demo.
- Customizing the output directory for recorded files is not supported.
- If there are no streams in the specified roomID, recording will not start.
</Warning>

## Command-line parameters

Optional command-line parameters are as follows:

- `-a`: AppID of the project.
- `-r`: Room ID where the stream to be recorded resides.
- `-u`: User ID used to log in to the room.
- `-k`: Authentication Token for logging in to the room.
- `-d`: Recording duration, in minutes.
- `-m`: Recording mode.
    - 1: Record single streams only.
    - 2: Record mix stream only.
    - 3: Record both single streams and mix stream simultaneously.
- `-s`: Single-stream recording type.
    - 1: Record audio only.
    - 2: Record video only.
    - 3: Record both audio and video.
- `-x`: Mix-stream recording type.
    - 1: Record audio only.
    - 2: Record video only.
    - 3: Record both audio and video.
- `-o`: Recording output mode.
    - 1: Generate recording files only.
    - 2: Return audio and video data only.
- `-t`: Output recording file type.
- `-g`: Generate an mp3 file in addition to the mp4 file.
- `-b`: Cache size, ranging from 64KB to 1 MB, default is 1 MB.
