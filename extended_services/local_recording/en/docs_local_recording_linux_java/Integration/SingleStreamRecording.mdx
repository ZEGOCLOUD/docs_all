# Record Single Stream

- - - 

## Overview

The on-premise server recording SDK supports recording individual audio and video streams in a room. Once integrated, you can start using the single-stream recording feature.

## Prerequisites

- You have integrated the ZEGO on-premise server recording SDK into your project. For details, see [Quick Start - Integration](./SDK%20Integration.mdx).
- You have created a project in the [ZEGOCLOUD Console](https://console.zegocloud.com) and obtained a valid AppID. For details, see "Project Information" in [Console - Project Management](/console/project-info).

## Procedure

###  Set up event callbacks

Before initializing the SDK, use the [setZegoRoomCallback](https://doc-zh.zego.im/API/ServerRecord-java/com/zego/zegoliveroom/ZegoLiveRoom.html#setZegoRoomCallback-com.zego.zegoliveroom.callback.IZegoRoomCallback-) and [setZegoStreamRecordCallback](https://doc-zh.zego.im/API/ServerRecord-java/com/zego/zegoliveroom/ZegoLiveRoom.html#setZegoStreamRecordCallback-com.zego.zegoliveroom.callback.IZegoStreamRecordCallback-) interfaces to set up room and recording callbacks respectively, so you can receive event notifications during the audio/video stream recording process.

```java
private static IZegoRoomCallback mRoomCallback = new IZegoRoomCallback() {
    // Handle room callbacks
};

private static IZegoStreamRecordCallback mStreamRecordCallback = new IZegoStreamRecordCallback() {
    // Handle recording event callbacks
};

public void setZegoRoomCallback(IZegoRoomCallback callback)

public void setZegoStreamRecordCallback(IZegoStreamRecordCallback callback)
```
  
Before initializing the SDK, you can also call the [setLogDirAndSize](https://doc-zh.zego.im/API/ServerRecord-java/com/zego/zegoliveroom/ZegoLiveRoom.html#setLogDirAndSize-java.lang.String-long-) interface to set the log file path and size for troubleshooting and analysis.
    
###  Initialize the SDK

Call the [initSDK](https://doc-zh.zego.im/API/ServerRecord-java/com/zego/zegoliveroom/ZegoLiveRoom.html#initSDK-long-byte:A-com.zego.zegoliveroom.callback.IZegoInitSDKCompletionCallback-) interface to initialize the SDK, passing your obtained AppID in the "appID" parameter.

```java
public boolean initSDK(long appID, IZegoInitSDKCompletionCallback callback)
```

Register the [onInitSDK](https://doc-zh.zego.im/API/ServerRecord-java/com/zego/zegoliveroom/callback/IZegoInitSDKCompletionCallback.html#onInitSDK-int-) async callback to receive event notifications during SDK initialization.

```java
public interface IZegoInitSDKCompletionCallback {
    void onInitSDK(int errorCode);
}
```

###  Set the Token

After initializing the SDK and before logging in to the room, call the [setCustomToken](https://doc-zh.zego.im/API/ServerRecord-java/com/zego/zegoliveroom/ZegoLiveRoom.html#setCustomToken-java.lang.String-) interface with your obtained Token for authentication.

For how to obtain a Token, see [Use Token for Authentication](/local-recording-linux-java/integration/user-access-control).

```java
public void setCustomToken(String thirdPartyToken);
```

###  Log in to a room

After authentication succeeds, call the [loginRoom](https://doc-zh.zego.im/API/ServerRecord-java/com/zego/zegoliveroom/ZegoLiveRoom.html#loginRoom-java.lang.String-com.zego.zegoliveroom.callback.IZegoLoginCompletionCallback-) interface to log in to a room.

```java
public boolean loginRoom(String roomID, IZegoLoginCompletionCallback callback)
```

Register the [onLoginCompletion](https://doc-zh.zego.im/API/ServerRecord-java/com/zego/zegoliveroom/callback/IZegoLoginCompletionCallback.html#onLoginCompletion-int-java.lang.String-) async callback to receive event notifications during room login.

```java
public interface IZegoLoginCompletionCallback {
    void onLoginCompletion(int errorCode, String roomID);
}
```

###  Start recording

After successfully logging in to the room, call the [startRecordSingleStream](https://doc-zh.zego.im/API/ServerRecord-java/com/zego/zegoliveroom/ZegoLiveRoom.html#startRecordSingleStream-java.lang.String-java.lang.String-int-int-com.zego.zegoliveroom.entity.ZegoRecordSingleStreamConfig-) interface to start recording audio and video streams.

<Note title="Note">

- It is recommended to wait until you have successfully logged in to the room and received the [onStreamUpdate](https://doc-zh.zego.im/API/ServerRecord-java/com/zego/zegoliveroom/callback/IZegoRoomCallback.html#onStreamUpdate-int-com.zego.zegoliveroom.entity.ZegoStreamInfo:A-java.lang.String-) callback with stream information updates, confirming new streams have been added, before calling the [startRecordSingleStream](https://doc-zh.zego.im/API/ServerRecord-java/com/zego/zegoliveroom/ZegoLiveRoom.html#startRecordSingleStream-java.lang.String-java.lang.String-int-int-com.zego.zegoliveroom.entity.ZegoRecordSingleStreamConfig-) interface to start recording.
- Before starting recording, you can call the [getMaxRecordCount](https://doc-zh.zego.im/API/ServerRecord-java/com/zego/zegoliveroom/ZegoLiveRoom.html#getMaxRecordCount--) interface to get the maximum number of concurrent recording streams supported by the SDK, and manage the number of simultaneous recordings accordingly.
</Note>

```java
public boolean startRecordSingleStream(String streamID, String pathAndName, int muxerStreamType, int fragmentSeconds, ZegoRecordSingleStreamConfig config)
```

Where:

- streamID: The ID of the stream to be recorded.
- pathAndName: The storage path (including file name) for the generated recording file.
    - Must end with `.mp3`/`.mp4`/`.flv`/`.m3u8`.
    - The SDK automatically creates the corresponding path. If a file with the same name exists at the path, it will be overwritten.
- muxerStreamType: The recording stream type. For details, see [ZegoMuxerStreamType](https://doc-zh.zego.im/API/ServerRecord-java/com/zego/zegoliveroom/constants/ZegoMuxerStreamType.html). By default, both audio and video are recorded.
- fragmentSeconds: The recording file fragmentation interval, in seconds.
    - For m3u8 format, the range is [2, 60]. For other formats, the range is [0, 10].
    - A value of 0 means no fragmentation. A value greater than 0 means fragmentation is enabled. The default value is 2.
    - Fragmentation can prevent issues where saved recording files cannot be played properly if an exception occurs during recording and causes an interruption.
- config: Configuration for single-stream recording. For details, see [ZegoRecordSingleStreamConfig](https://doc-preview-zh.zego.im/API/ServerRecord-java/com/zego/zegoliveroom/entity/ZegoRecordSingleStreamConfig.html).
  
<Warning title="Note">

"Fragmentation" is an internal SDK concept for processing recording files. It does not split the recorded file into multiple smaller files stored separately.
</Warning>

Register the [onStreamRecordBegin](https://doc-zh.zego.im/API/ServerRecord-java/com/zego/zegoliveroom/callback/IZegoStreamRecordCallback.html#onStreamRecordBegin-java.lang.String-java.lang.String-) async callback to receive event notifications during recording.

```java
public interface IZegoStreamRecordCallback {

    void onStreamRecordBegin(String streamID, String pathAndName);
}
```

After recording starts successfully, you will receive the [onStreamRecordBegin](https://doc-zh.zego.im/API/ServerRecord-java/com/zego/zegoliveroom/callback/IZegoStreamRecordCallback.html#onStreamRecordBegin-java.lang.String-java.lang.String-) callback.
          
###  Stop recording

When recording is no longer needed or is complete, call the [stopRecordSingleStream](https://doc-zh.zego.im/API/ServerRecord-java/com/zego/zegoliveroom/ZegoLiveRoom.html#stopRecordSingleStream-java.lang.String-) interface to stop recording.

```java
public boolean stopRecordSingleStream(String streamID)
```

Register the [onStreamRecordEnd](https://doc-zh.zego.im/API/ServerRecord-java/com/zego/zegoliveroom/callback/IZegoStreamRecordCallback.html#onStreamRecordEnd-java.lang.String-java.lang.String-int-) async callback to receive event notifications when recording stops.
 
```java
public interface IZegoStreamRecordCallback {
    void onStreamRecordEnd(String streamID, String pathAndName, int reason);
}
```
 
Then:

1. Call the [logoutRoom](https://doc-zh.zego.im/API/ServerRecord-java/com/zego/zegoliveroom/ZegoLiveRoom.html#logoutRoom--) interface to log out of the room.
2. Call [setZegoRoomCallback](https://doc-zh.zego.im/API/ServerRecord-java/com/zego/zegoliveroom/ZegoLiveRoom.html#setZegoRoomCallback-com.zego.zegoliveroom.callback.IZegoRoomCallback-)(null) and [setZegoStreamRecordCallback](https://doc-zh.zego.im/API/ServerRecord-java/com/zego/zegoliveroom/ZegoLiveRoom.html#setZegoStreamRecordCallback-com.zego.zegoliveroom.callback.IZegoStreamRecordCallback-)(null) to stop callback listeners.
3. Call [unInitSDK](https://doc-zh.zego.im/API/ServerRecord-java/com/zego/zegoliveroom/ZegoLiveRoom.html#unInitSDK--) to deinitialize the SDK.
