# Record Mix Stream

- - - 

## Overview

The on-premise server recording SDK supports mixing all users' audio and video streams in a room into a single view for recording. Once integrated, you can start using the mix-stream recording feature.

## Prerequisites

- You have integrated the ZEGO on-premise server recording SDK into your project. For details, see [Quick Start - Integration](./SDK%20Integration.mdx).
- You have created a project in the [ZEGOCLOUD Console](https://console.zegocloud.com) and obtained a valid AppID. For details, see "Project Information" in [Console - Project Management](/console/project-info).

## Procedure

### Set up event callbacks

Before initializing the SDK, use the [setZegoRoomCallback](https://doc-zh.zego.im/API/ServerRecord-java/com/zego/zegoliveroom/ZegoLiveRoom.html#setZegoRoomCallback-com.zego.zegoliveroom.callback.IZegoRoomCallback-), [setZegoStreamRecordCallback](https://doc-zh.zego.im/API/ServerRecord-java/com/zego/zegoliveroom/ZegoLiveRoom.html#setZegoStreamRecordCallback-com.zego.zegoliveroom.callback.IZegoStreamRecordCallback-), and [setZegoExternalVideoDataCallback](https://doc-preview-zh.zego.im/API/ServerRecord-java/com/zego/zegoliveroom/ZegoLiveRoom.html#setZegoExternalVideoDataCallback-com.zego.zegoliveroom.callback.IZegoExternalVideoDataCallback-) interfaces to set up room, recording, and video data callbacks respectively, so you can receive event notifications during the audio/video stream recording process.

```java
private static IZegoRoomCallback mRoomCallback = new IZegoRoomCallback() {
    // Handle room callbacks
};

private static IZegoStreamRecordCallback mStreamRecordCallback = new IZegoStreamRecordCallback() {
    // Handle recording event callbacks
};
   
private static IZegoExternalVideoDataCallback mVideoDataCallback = new IZegoExternalVideoDataCallback() {
    // Handle video frame data callback. The callback outputs the decoded frame data for each single stream. Currently only supported under mix-stream recording.
};

public void setZegoRoomCallback(IZegoRoomCallback callback)

public void setZegoStreamRecordCallback(IZegoStreamRecordCallback callback)
 
public void setZegoExternalVideoDataCallback(IZegoExternalVideoDataCallback callback)
```

Before initializing the SDK, you can also call the [setLogDirAndSize](https://doc-zh.zego.im/API/ServerRecord-java/com/zego/zegoliveroom/ZegoLiveRoom.html#setLogDirAndSize-java.lang.String-long-) interface to set the log file path and size for troubleshooting and analysis.
    
### Initialize the SDK

Call the [initSDK](https://doc-zh.zego.im/API/ServerRecord-java/com/zego/zegoliveroom/ZegoLiveRoom.html#initSDK-long-byte:A-com.zego.zegoliveroom.callback.IZegoInitSDKCompletionCallback-) interface to initialize the SDK, passing your obtained AppID in the "appID" parameter.

```java
public boolean initSDK(long appID, IZegoInitSDKCompletionCallback callback)
```

Register the [onInitSDK](https://doc-zh.zego.im/API/ServerRecord-java/com/zego/zegoliveroom/callback/IZegoInitSDKCompletionCallback.html#onInitSDK-int-) async callback to receive event notifications during SDK initialization.

```java
public interface IZegoInitSDKCompletionCallback {
    void onInitSDK(int errorCode);
}
```

###  Set the Token

After initializing the SDK and before logging in to the room, call the [setCustomToken](https://doc-zh.zego.im/API/ServerRecord-java/com/zego/zegoliveroom/ZegoLiveRoom.html#setCustomToken-java.lang.String-) interface with your obtained Token for authentication.

For how to obtain a Token, see [Use Token for Authentication](/local-recording-linux-java/integration/user-access-control).

```java
public void setCustomToken(String thirdPartyToken);
```

###  Log in to a room

After authentication succeeds, call the [loginRoom](https://doc-zh.zego.im/API/ServerRecord-java/com/zego/zegoliveroom/ZegoLiveRoom.html#loginRoom-java.lang.String-com.zego.zegoliveroom.callback.IZegoLoginCompletionCallback-) interface to log in to a room.

```java
public boolean loginRoom(String roomID, IZegoLoginCompletionCallback callback)
```

Register the [onLoginCompletion](https://doc-zh.zego.im/API/ServerRecord-java/com/zego/zegoliveroom/callback/IZegoLoginCompletionCallback.html#onLoginCompletion-int-java.lang.String-) async callback to receive event notifications during room login.

```java
public interface IZegoLoginCompletionCallback {
    void onLoginCompletion(int errorCode, String roomID);
}
```

###  Start recording

After successfully logging in to the room, call the [startRecordMixStream](https://doc-zh.zego.im/API/ServerRecord-java/com/zego/zegoliveroom/ZegoLiveRoom.html#startRecordMixStream-com.zego.zegoliveroom.entity.ZegoMixStreamRecordConfig-) interface to start recording audio and video streams.

<Note title="Note">

- It is recommended to wait until you have successfully logged in to the room and received the [onStreamUpdate](https://doc-zh.zego.im/API/ServerRecord-java/com/zego/zegoliveroom/callback/IZegoRoomCallback.html#onStreamUpdate-int-com.zego.zegoliveroom.entity.ZegoStreamInfo:A-java.lang.String-) callback with stream information updates, confirming new streams have been added, before calling the [startRecordMixStream](https://doc-zh.zego.im/API/ServerRecord-java/com/zego/zegoliveroom/ZegoLiveRoom.html#startRecordMixStream-com.zego.zegoliveroom.entity.ZegoMixStreamRecordConfig-) interface to start mix-stream recording.
- Before starting recording, you can call the [getMaxRecordCount](https://doc-zh.zego.im/API/ServerRecord-java/com/zego/zegoliveroom/ZegoLiveRoom.html#getMaxRecordCount--) interface to get the maximum number of concurrent recording streams supported by the SDK, and manage the number of simultaneous recordings accordingly.
</Note>

```java
public boolean startRecordMixStream(ZegoMixStreamRecordConfig config)
```

Where: config refers to the mix-stream recording configuration. For details, see [ZegoMixStreamRecordConfig](https://doc-zh.zego.im/API/ServerRecord-java/com/zego/zegoliveroom/entity/ZegoMixStreamRecordConfig.html).
  
Register the [onStreamRecordBegin](https://doc-zh.zego.im/API/ServerRecord-java/com/zego/zegoliveroom/callback/IZegoStreamRecordCallback.html#onStreamRecordBegin-java.lang.String-java.lang.String-) and [onMixStreamRecordUpdate](https://doc-zh.zego.im/API/ServerRecord-java/com/zego/zegoliveroom/callback/IZegoStreamRecordCallback.html#onMixStreamRecordUpdate-java.lang.String:A-) async callbacks to receive event notifications during mix-stream recording.

```java
public interface IZegoStreamRecordCallback {

    void onStreamRecordBegin(String streamID, String pathAndName);
    void onMixStreamRecordUpdate(String[] listStreamID);
}
```

After recording starts successfully, you will receive the [onStreamRecordBegin](https://doc-zh.zego.im/API/ServerRecord-java/com/zego/zegoliveroom/callback/IZegoStreamRecordCallback.html#onStreamRecordBegin-java.lang.String-java.lang.String-) callback. When streams with audio/video data are successfully pulled, you will receive the [onMixStreamRecordUpdate](https://doc-zh.zego.im/API/ServerRecord-java/com/zego/zegoliveroom/callback/IZegoStreamRecordCallback.html#onMixStreamRecordUpdate-java.lang.String:A-) callback.
    
<Warning title="Note">

If you need to record both single streams and mix streams simultaneously, call the [enableSingleAndMixRecordMode](https://doc-zh.zego.im/API/ServerRecord-java/com/zego/zegoliveroom/ZegoLiveRoom.html#enableSingleAndMixRecordMode-boolean-) interface before starting recording to enable `simultaneous single-stream and mix-stream recording mode`:

- In this mode, "mix-stream recording" will include all streams being recorded by "single-stream recording". Even if you have not specified a single stream as an input for mix-stream recording, its audio (if present) will still be recorded in the mix stream.
- This mode consumes more system resources. You must start mix-stream recording first, then start single-stream recording. When mix-stream recording is stopped, all single-stream recordings will also be stopped.

Use this mode with caution.
</Warning>
      
###  Update mix-stream configuration

When you need to add input streams, update input streams, or update mix-stream output configuration, reconstruct the mix-stream configuration [ZegoStreamConfig](https://doc-zh.zego.im/API/ServerRecord-java/com/zego/zegoliveroom/entity/ZegoStreamConfig.html) and call the [updateInputStreamConfig](https://doc-zh.zego.im/API/ServerRecord-java/com/zego/zegoliveroom/ZegoLiveRoom.html#updateInputStreamConfig-com.zego.zegoliveroom.entity.ZegoStreamConfig:A-) interface to update the mix-stream configuration.

```java
public boolean updateInputStreamConfig(ZegoStreamConfig[] listStreamConfig)
```
    
Register the [onMixStreamRecordUpdate](https://doc-zh.zego.im/API/ServerRecord-java/com/zego/zegoliveroom/callback/IZegoStreamRecordCallback.html#onMixStreamRecordUpdate-java.lang.String:A-) async callback to receive event notifications during mix-stream configuration updates.

```java
public interface IZegoStreamRecordCallback {
    void onMixStreamRecordUpdate(String[] listStreamID);
}
```

<Note title="Note">

- After successful mixing, the list of mix-stream input stream IDs will be returned (containing only stream IDs with successfully pulled data).
- Whenever the mix-stream recording input streams change (e.g., starting recording, stream exceptions during recording), you will receive the [onMixStreamRecordUpdate](https://doc-zh.zego.im/API/ServerRecord-java/com/zego/zegoliveroom/callback/IZegoStreamRecordCallback.html#onMixStreamRecordUpdate-java.lang.String:A-) callback.
</Note>  

###  Stop recording

When recording is no longer needed or is complete, call the [stopRecordMixStream](https://doc-zh.zego.im/API/ServerRecord-java/com/zego/zegoliveroom/ZegoLiveRoom.html#stopRecordMixStream--) interface to stop recording.

```java
public boolean stopRecordMixStream()
```

Register the [onStreamRecordEnd](https://doc-zh.zego.im/API/ServerRecord-java/com/zego/zegoliveroom/callback/IZegoStreamRecordCallback.html#onStreamRecordEnd-java.lang.String-java.lang.String-int-) async callback to receive event notifications when recording stops.
 
```java
public interface IZegoStreamRecordCallback {
    void onStreamRecordEnd(String streamID, String pathAndName, int reason);
}
```

Then:

1. Call the [logoutRoom](https://doc-zh.zego.im/API/ServerRecord-java/com/zego/zegoliveroom/ZegoLiveRoom.html#logoutRoom--) interface to log out of the room.
2. Call [setZegoRoomCallback](https://doc-zh.zego.im/API/ServerRecord-java/com/zego/zegoliveroom/ZegoLiveRoom.html#setZegoRoomCallback-com.zego.zegoliveroom.callback.IZegoRoomCallback-)(null), [setZegoStreamRecordCallback](https://doc-zh.zego.im/API/ServerRecord-java/com/zego/zegoliveroom/ZegoLiveRoom.html#setZegoStreamRecordCallback-com.zego.zegoliveroom.callback.IZegoStreamRecordCallback-)(null), and [setZegoExternalVideoDataCallback](https://doc-preview-zh.zego.im/API/ServerRecord-java/com/zego/zegoliveroom/ZegoLiveRoom.html#setZegoExternalVideoDataCallback-com.zego.zegoliveroom.callback.IZegoExternalVideoDataCallback-)(null) to stop callback listeners.
3. Call [unInitSDK](https://doc-zh.zego.im/API/ServerRecord-java/com/zego/zegoliveroom/ZegoLiveRoom.html#unInitSDK--) to deinitialize the SDK.
