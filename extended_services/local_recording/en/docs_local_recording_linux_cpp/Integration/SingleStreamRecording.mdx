# Single-Stream Recording
- - - 

##   Feature Overview

The Local Server Recording SDK supports recording individual audio/video streams in a room. After integrating the SDK, you can start using the single-stream recording feature.

##   Prerequisites


- You have integrated the ZEGOCLOUD Local Server Recording SDK in your project. For details, see [Quick Start - Integration](./SDK%20Integration.mdx).
- You have created a project in the [ZEGOCLOUD Console](https://console.zegocloud.com) and obtained a valid AppID. For details, see "Project Information" in [Console - Project Management](/console/project-info).

<Warning title="Note">

ZEGOCLOUD Local Server Recording SDK versions before ga01596a1c8 (released before 2022-03-23) require obtaining a valid AppID from the [ZEGOCLOUD Console](https://console.zegocloud.com) and contacting ZEGOCLOUD Technical Support to obtain an AppSign. If you need to upgrade the authentication method, see [How to Upgrade from AppSign Authentication to Token Authentication](http://doc-zh.zego.im/faq/token_upgrade?product=ExpressVideo&platform=all).
</Warning>


##   Steps

### Set Up Callback Listener

Before initialization, you need to implement ICallback and set up the callback listener.

- API prototype:

  ```cpp
  ZEGO_API bool SetCallback(ICallback* pCB)
  ```
    
- Parameters:

  `pCB`: Callback object pointer.
  
- Note:

  Before initializing the SDK, you can optionally call the following API as needed:

  [SetLogDirAndSize](https://doc-zh.zego.im/API/ServerRecord/namespace_z_e_g_o_1_1_l_i_v_e_r_o_o_m.html#a3b3fbef85f70e1796df124616ad3eb5e) to set the log file path and size.
    
### Initialize the SDK

Call the [InitSDK](https://doc-zh.zego.im/API/ServerRecord/namespace_z_e_g_o_1_1_l_i_v_e_r_o_o_m.html#af8fcc2e5a69e83075637631d68fcf4ea) API to initialize the SDK, passing the obtained AppID to the "uiAppID" parameter.


- API prototype:

  ```cpp
  ZEGO_API bool InitSDK(unsigned int uiAppID)
  ```
    
- Parameters:

  `uiAppID`: The numeric ID assigned by ZEGO, the unique identifier for each developer.
  

- Async callback:

  ```cpp
  class ICallback
  {
  public:
      virtual void OnInitSDK(int errorCode) {}
  }
  ```
  errorCode == 0 indicates successful initialization.


### Set Token

Before logging in to a room, call the [SetCustomToken](https://doc-zh.zego.im/API/ServerRecord/namespace_z_e_g_o_1_1_l_i_v_e_r_o_o_m.html#a68bdea0d66d62c8bf4d671284707b3ae) API to pass the Token to the SDK.

- API prototype:

  ```cpp
  ZEGO_API void SetCustomToken(const char *thirdPartyToken)
  ```

- Parameters:

  `thirdPartyToken`: Authentication Token.

### Log In to a Room

You must log in to a room before recording.

- API prototype:

  ```cpp
  ZEGO_API bool LoginRoom(const char* pszRoomID)
  ```
    
- Parameters:  

  `pszRoomID`: The room ID of the room where streams are being published.
  
- Async callback:

  ```cpp
  class ICallback
  {
  public:
      virtual void OnLoginRoom(int errorCode, const char *pszRoomID) = 0;
  }
  ```
    
### Start Recording

1. This API must be called after successful initialization.
2. It is recommended to start recording after logging in to a room and receiving the [OnStreamUpdate](https://doc-zh.zego.im/API/ServerRecord/class_z_e_g_o_1_1_l_i_v_e_r_o_o_m_1_1_i_callback.html#a6d0fe744187badddf744918869c59dfd) stream update callback, based on the added stream information.
3. Before starting recording, you can call [GetMaxRecordCount](https://doc-zh.zego.im/API/ServerRecord/namespace_z_e_g_o_1_1_l_i_v_e_r_o_o_m.html#ac0a46bc40d185af50d12dc3b210c7bc7) to get the maximum number of concurrent recording streams supported by the SDK, and use this value to manage the number of simultaneously recorded streams.

- API prototype:

  ```cpp
  ZEGO_API bool StartRecordSingleStream(const char* pszStreamID, const char *pszPathAndName, MuxerStreamType nMuxerStreamType = MuxerStreamTypeBoth, int nFragmentSeconds = 2)
  ```
    
- Parameters:  

  pszStreamID: The stream ID to be recorded.

  pszPathAndName: The full path of the recording file (including file name), and it must end with .mp4, .m3u8, .flv, .mp3, or .aac (where .mp3 and .aac only support audio-only recording). The SDK will automatically create the corresponding path. If a file with the same name exists, it will be overwritten.

  nMuxerStreamType: The recording stream type. See [MuxerStreamType](https://doc-zh.zego.im/API/ServerRecord/namespace_z_e_g_o_1_1_l_i_v_e_r_o_o_m.html#aa3bdd4c85ace109d95354ff574454b96). Default records both audio and video.

  nFragmentSeconds: Recording file fragment interval (0–10s). 0 means no fragmentation; greater than 0 means fragmentation. Default interval is 2s. Enabling fragmentation ensures that saved recording files can be played normally even if an abnormal interruption occurs during recording. Note that this fragmentation is an internal SDK logical concept for processing recording files, not splitting the recording file into multiple small files stored by time interval.
  
- Async callback:

  ```cpp
  class ICallback
  {
  public:
      virtual void OnStreamRecordBegin(const char* pszStreamID, const char *pszPathAndName) = 0;
  }
  ```
A recording started callback will be received upon successful recording start.
      
### Stop Recording

- API prototype:

  ```cpp
  ZEGO_API bool StopRecordSingleStream(const char* pszStreamID)
  ```
    
- Parameters:  

  pszStreamID: The stream ID.

- Async callback:

  ```cpp
  class ICallback
  {
  public:
      virtual void OnStreamRecordEnd(const char* pszStreamID, const char *pszPathAndName, RecordEndReason reason) = 0;
  }
  ```
    
- Note:

  When recording is no longer needed, it is recommended to perform the following steps:
  1. Call [LogoutRoom](https://doc-zh.zego.im/API/ServerRecord/namespace_z_e_g_o_1_1_l_i_v_e_r_o_o_m.html#a02b98c29388acfea615e98580dbf37be) to log out of the room.
  2. Call `LIVEROOM::SetCallback(nullptr);` to remove the callback listener.
  3. Call [UnInitSDK](https://doc-zh.zego.im/API/ServerRecord/namespace_z_e_g_o_1_1_l_i_v_e_r_o_o_m.html#ad22d5b7f5f22a4f2339d7a09fbc8e149) to deinitialize the SDK.
