<Accordion title="2021-12-02 Version：gd2ef35d43f">

**New Features**

1. H.265 overall solution support
    
    The complete H.265 encoding/decoding solution is now available for single-host live streaming and multi-user interactive live streaming scenarios. This plugin primarily adapts to the RTC product's H.265 automatic fallback (H.265 falling back to H.264) capability, generating two files during recording (one for H.265 and one for H.264).

2. Support for taking screenshots of specified streams.
3. Support for capturing audio input sources during external capture.
4. The following features have been added to mix-stream recording:

    - Support for setting image watermarks.
    - Support for pausing/resuming recording during mix-stream recording.
    - Support for setting background images for specified streams during mix-stream recording.

**Deprecations**

To simplify environment understanding for developers, the test environment has been deprecated and unified to use the production environment. In version `gd2ef35d43f` and above, the original [SetUseTestEnv] API for setting the test environment has been deprecated. Developers can use it directly without setting environment information.

</Accordion>

<Accordion title="2020-04-30 Version：g709410abf">
 
**New Features**

1. Login authentication  
Added the [SetCustomToken](https://doc-zh.zego.im/API/ServerRecord/namespace_z_e_g_o_1_1_l_i_v_e_r_o_o_m.html#a68bdea0d66d62c8bf4d671284707b3ae) API for setting custom authentication information to verify user legitimacy during login. 
2. Set user information  
Added the [SetUser](https://doc-zh.zego.im/API/ServerRecord/namespace_z_e_g_o_1_1_l_i_v_e_r_o_o_m.html#a7ac98e20f63c4fae7266214be860a089) API for setting user name and related information.  
3. Get absolute timestamp of recording start  
In online education scenarios, teachers may simultaneously live stream and edit whiteboards. Developers can use the ZEGO recording SDK to record the teacher's live audio/video stream and obtain the absolute timestamp of the recording start relative to the teacher's end, used for aligning the teacher's audio/video recording with whiteboard data.   
(1) Added the [onRecvMediaSideInfo](https://doc-zh.zego.im/API/ServerRecord/class_z_e_g_o_1_1_l_i_v_e_r_o_o_m_1_1_i_media_side_callback.html#a1ebd6831721ce03e71db1c52a904e008) callback to receive media side information sent by the streaming end.  
(2) Added the [GetRecordLastMediaSideTime](https://doc-zh.zego.im/API/ServerRecord/namespace_z_e_g_o_1_1_l_i_v_e_r_o_o_m.html#af11acf7b7af97126a6cbe2fca2e7cca9) API to get the time interval between the most recently received media side information packet and the first recorded packet.  

  After the streaming end starts streaming, it periodically sends media side information (containing the current absolute timestamp). When the recording end receives this, it subtracts the time returned by `GetRecordLastMediaSideTime` from the parsed timestamp to obtain the absolute timestamp of the recording start. 

4. Get recording file information  
Added the [GetRecordStatus](https://doc-zh.zego.im/API/ServerRecord/namespace_z_e_g_o_1_1_l_i_v_e_r_o_o_m.html#a1c2d983f3d3cef1ef0b54288e9e2eac3) API for getting the recording file storage path, recorded duration, and file size.  
5. Added support for setting watermark position, font color, and background color attributes.

**Bug Fixes**

1. Fixed a bug where no error callback was triggered when flexible configuration fetch failed.
2. Fixed a bug where [LoginRoom](https://doc-zh.zego.im/API/ServerRecord/namespace_z_e_g_o_1_1_l_i_v_e_r_o_o_m.html#ac13869a794f456b739a5c9240e8f752c) would trigger a success callback even when [InitSDK](https://doc-zh.zego.im/API/ServerRecord/namespace_z_e_g_o_1_1_l_i_v_e_r_o_o_m.html#a0b05e2d88f37fe107e55c6f01809674b) failed.
3. When a stream is removed from the mix input, [OnMixStreamRecordUpdate](https://doc-zh.zego.im/API/ServerRecord/class_z_e_g_o_1_1_l_i_v_e_r_o_o_m_1_1_i_callback.html#ad5838ef43c3ec75de7f556c085fba7f3) callback is triggered.
</Accordion>
