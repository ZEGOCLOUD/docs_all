## Running the Demo Source Code

This document describes how to quickly run the demo source code to experience basic audio/video stream recording functionality.

### Prerequisites

You have created a project in the [ZEGOCLOUD Console](https://console.zegocloud.com) and obtained a valid AppID. For details, see "Project Information" in [Console - Project Management](/console/project-info).

### Running the Demo Source Code

Assume the Local Server Recording Demo is placed in the "/root" directory.

The directory contents of the Local Server Recording Demo are as follows:

<Frame width="512" height="auto" caption="">
  <img src="https://doc-media.zego.im/sdk-doc/Pics/Linux/ServerRecording/demoMenu.png" />
</Frame>

1. Run the `cd ~/ZegoServerRecordingDemo` command to navigate to the "ZegoServerRecordingDemo" directory.

<Frame width="512" height="auto" caption="">
  <img src="https://doc-media.zego.im/sdk-doc/Pics/Linux/ServerRecording/use_step1.jpg" />
</Frame>

2. Run the `chmod 777 playrecorder` command to add executable permissions to playrecorder.

<Frame width="512" height="auto" caption="">
  <img src="https://doc-media.zego.im/sdk-doc/Pics/Linux/ServerRecording/use_step2.jpg" />
</Frame>

3. Run the `export LD_LIBRARY_PATH=./libs/:$LD_LIBRARY_PATH` command to add the "libs" folder to the library search path.

<Frame width="512" height="auto" caption="">
  <img src="https://doc-media.zego.im/sdk-doc/Pics/Linux/ServerRecording/use_step3.jpg" />
</Frame>

4. Run the `./playrecorder -a appID -r roomID -u userID -k token` command to start recording. 

- "appID": The AppID obtained in the [Prerequisites](#prerequisites).
- "roomID": The room ID of the room where data streams are to be recorded.
- "userID": The user ID used to log in to the room.
- "token": The authentication Token for logging in to the room containing the streams to be recorded.
- If no additional parameters are set, the default recording duration is 20s, recording a single stream, generating one .mp4 audio/video file.

<Frame width="512" height="auto" caption="">
  <img src="https://doc-media.zego.im/sdk-doc/Pics/Linux/ServerRecording/use_step4.jpg" />
</Frame>

<Warning title="Note">

- The generated recording files are saved in the `~/zego/log_[roomID]_[timestamp]` folder, where:
    - roomID: The room ID specified when running the Demo.
    - timestamp: The timestamp when running the Demo.
- Custom directory for generated recording files is not supported.
- If there are no streams in the specified roomID, recording will not start.
</Warning>

## Command Line Parameters

Optional command line parameters are as follows:

- `-a`: The AppID of the project.
- `-r`: The room ID containing the streams to be recorded.
- `-u`: The user ID for logging in to the room.
- `-k`: The authentication Token for logging in to the room.
- `-d`: Recording duration in minutes.
- `-m`: Recording mode.
    - 1: Single-stream recording only.
    - 2: Mix-stream recording only.
    - 3: Both single-stream and mix-stream recording simultaneously.
- `-s`: Single-stream recording type.
    - 1: Audio only.
    - 2: Video only.
    - 3: Both audio and video.
- `-x`: Mix-stream recording type.
    - 1: Audio only.
    - 2: Video only.
    - 3: Both audio and video.
- `-o`: Recording output mode.
    - 1: Generate recording files only.
    - 2: Return audio/video data only.
- `-t`: Output recording file type.
- `-g`: Generate mp3 files along with mp4 files.
- `-b`: Cache size, ranging from 64KB to 1 MB, default is 1 MB.
