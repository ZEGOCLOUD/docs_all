## 1 Development Environment Requirements
Ensure your development environment meets the following conditions:

* CentOS 6.5+ 64-bit
* Ubuntu 14.04+ 64-bit
* cmake is installed on Linux

## 2 Download SDK
Local Server Recording SDK download link: [ZegoLiveRoom-ServerRecording-Linux](https://artifact-sdk.zego.im/downloads/ZegoLiveRoom-ServerRecording-Linux.zip)

After downloading and extracting the SDK, the contents are as follows:

As shown in the diagram marked with "2", after extraction the package contains the include and libs directories:
* include directory: Contains the SDK header file — `LiveRoom.h`.
* libs directory: Contains the SDK shared library file — `libzegoliveroomrecorder.so`.

## 3 Integrate the SDK

### 3.1 Add Header Files
Add the header file (LiveRoom.h) from the include directory of the SDK package to your project's header file directory. Your project needs to add the link path to this header file directory.

### 3.2 Link Dynamic Libraries
Add the shared library file (libzegoliveroomrecorder.so) from the libs directory of the SDK package to your project's library file directory or a custom directory. Your project needs to add the link path to this shared library file.

### 3.3 (Optional) Block SIGPIPE Signal
(Optional) The Linux SDK does not block the SIGPIPE signal. During actual development, you may choose whether to block the SIGPIPE signal based on your needs. In most cases, you need to block this signal; otherwise, the client process will exit by default upon receiving this signal. You can also ignore the signal directly: `signal(SIGPIPE,SIG_IGN);`.

```cpp
    static void ExceptionSigHandler(int signum)
     {
         char exitSignalStr[1024] = {0};
         sprintf(exitSignalStr,"exception signal : %d",signum);
         log("------------------------------");
         log(exitSignalStr);
         log("------------------------------");
     }
     int main(int argc, const char * argv[]) {    
         signal(SIGPIPE, ExceptionSigHandler);
         // your code
      
         return 0;
     }
```

## 4 Integration Example
The following example uses the integration method of the Local Server Recording Demo.
### 4.1 Create a New Project
Assume the project code is placed in the `~/zegoServerRecordingDemo` directory.
* $ mkdir ~/zegoServerRecordingDemo

### 4.2 Copy the SDK
Copy the `include/` and `libs/` folders to the `~/zegoServerRecordingDemo` directory.
* $ cp -r include libs ~/zegoServerRecordingDemo 
* $ cd ~/zegoServerRecordingDemo

### 4.3 Create main.cpp
Create a main.cpp file and print the SDK version number.
* $ touch main.cpp
* $ vim main.cpp
* Enter the following content:

  ```cpp
  #include <stdlib.h>
  #include <stdio.h>
  #include "LiveRoom.h"

  using namespace ZEGO;

  int main(int argc, const char * argv[])
  {
     printf("Zego SDK Version: %s\n", LIVEROOM::GetSDKVersion());
     return 0;
  }
  ``` 

### 4.4 Create CMakeLists.txt
* $ touch CMakeLists.txt
* $ vim CMakeLists.txt
* Enter the following content:

  ```cpp
  cmake_minimum_required(VERSION 3.5.2)
  project (zegoServerRecordingDemo)

  include_directories("./include")

  link_directories("./libs")

  link_libraries(zegoliveroomrecorder rt)

  add_compile_options(
    -std=c++11
    )

  aux_source_directory(. SRC_LIST)

  add_executable(zegoServerRecordingDemo ${SRC_LIST})
  ```

### 4.5 Build
* $ mkdir build & cd build
* $ cmake .. & make

A successful build will generate an executable file.

### 4.6 Run 
* $ ./zegoServerRecordingDemo

If the integration is successful, it will print "Zego SDK Version: xxx".
