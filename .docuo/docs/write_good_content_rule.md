# 规范

## 表达规范

### 目录和标题

- 句式结构建议保持一致。
- 简明扼要概括文章/章节中心内容。

### 词汇
- 避免使用表示程度、强调语气的词，建议用具体的数据或实例代替
    > 例如：较多、较好、完全地、基本地、决定性的、最后的、 仅仅、事实上、值得注意的。
- 避免使用表示转折的词
    > 例如：当然、然而、但是
- 避免使用含义模糊的词
    > 例如：有些、非常、大量、一些、少许、部分，一段时间，几乎。
- 避免使用过于口语化的词
    > 例如：接下来、总的来说、点击一下按钮。

### 专业术语
- 文档中出现的技术术语时，需要在文档最前面加上说明和解释，尽可能附带详细说明链接

### 缩略语

正文中第一次出现的缩略语必须解释
格式如下：
- 中文格式为：缩略语（英文全称，中文解释）
    > 例如 VPN （Virtual Private Network，虚拟专用网络 ）
- 英文格式为：英文全称（缩略语）
    > 例如 Virtual Private Network (VPN)

### 注意和警告

- 需要提供给开发者的说明信息或提示时使用 `<Note>` 组件
- 若用户忽略注意标志，可能会因误操作而带来不良后果或者无法操作成功时说明，需要提供给开发者的警告信息时使用 `<Warning>` 组件

## 格式规范

### 目录和标题
- 不带任何标点符号

### 表格

- 表格中必须有表头，并使用名词或名词词组结构，同一表格中有且仅有一行表头。
- 表格中不出现空白的单元格，无内容单元填写特定符号 "-"，或者使用 `!mu` 或 `!md` 或 `!ml` 或 `!mr` 标记合并单元格。

### 标点符号

- 当路径存在多个层级时，使用斜杆隔开（/），且最后一级目录后面不加（/）
    > 例如：/ZegoExpressExample/Mixer
- 中文文档中只用全角中文标点，英文文档中用半角英文标点。（代码和引用内容除外）
- 所有陈述句都要以句号结尾。
- 中文文档中的英文和数字前后都各空一格。比如：可以设置 FPS 为 15 。
- 链接、单行代码块前后都各空一格。
- 菜单间用 ">" 符号连接，">" 前后各空一格，整个菜单需要加引号
    > 例如：选择 “文件 > 新建 > 新建工程” 菜单。
- 用"～"符号连接，且前后空格表示数据范围
    > 例如：300 ～ 600
- 用开闭区间 [xx, xx] 表示参数取值范围
    > 例如：volume 取值范围为 [1, 100]
- 数字范围前后两个数字同时带单位或者不带单位
    > 例如：10 ms ～ 100 ms

### 时间

- 年、月、日格式为"YYYY-MM-DD"，当月和日为个位数时，需要加"0"补充。
- 时、分、秒格式为"hh：mm：ss"，当时、分、秒为个位数时，需要加"0"补充
- 年月日 + 时分秒一起写，需要空格，例如：2023-07-20 14:42:35
